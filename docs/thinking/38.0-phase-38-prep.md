### D-3. Wikilink extension framework shape anticipated (Unit 38.1)

D-A. **Plugin shape** = `rehypeResolveWikilinks` rehype plugin
that walks the HAST tree post-sanitize, finds text nodes
matching `[[slug]]` pattern, replaces with `<a>` element nodes:

```ts
const WIKILINK_PATTERN = /\[\[([a-z0-9-]+)\]\]/g;

const rehypeResolveWikilinks: Plugin<[ResolveOptions], Root> =
  (options) => (tree) => {
    visit(tree, "text", (node, index, parent) => {
      // Split text on [[slug]] matches; replace each match
      // with an <a href="/problems/{slug}">{slug}</a> element.
      // Defer to Unit 38.1 for exact split-and-replace logic.
    });
  };
```

D-B. **URL resolution convention**: `[[slug]]` →
`<a href="/problems/{slug}">{slug}</a>` (locale-less; relies on
Next.js i18n middleware to forward to `/{locale}/problems/{slug}`
at request time). Decision deferred to Unit 38.1 implementation
time:

- **Path A**: locale-less hrefs (`/problems/{slug}`); middleware
  redirects to current-locale.
- **Path B**: locale-aware via plugin options
  (`{ locale: string }`) — plugin needs locale-context-per-render-
  call; requires Phase-37 framework's `getExtensions(surface)`
  return values to be parameterized per-render-call OR a thread-
  local locale state.
- **Path C**: locale-less hrefs with a default-locale anchor at
  `/en/problems/{slug}` matching the project's i18n strategy
  (ADR-0011 — default locale = `en`; Next.js middleware handles
  redirect/rewrite to current locale on request).

D-C. **Schema overrides**: wikilinks generate `<a>` elements
with `href`. Both are already in `bioSchema.tagNames` +
`bioSchema.attributes.a` + `bioSchema.protocols.href` allow-list
— **NO `schemaOverrides` required** for wikilink elements
themselves (they reuse the existing markdown-link sanitization
path). The wikilink-generated href starts with `/` (relative
URL), which the `rehypeStripUnsafeHrefs` plugin would strip per
ADR-0018 D-D (relative URLs denied). **Wikilinks must therefore
fold AFTER `rehypeStripUnsafeHrefs` in the pipeline** — exactly
matching the Phase-37 framework's APPEND-D-D plugin-order-after-
default discipline. The relative `/problems/{slug}` URL bypasses
the existing strip step because it's added by the extension AFTER
that step runs.

This is a **subtle architectural property**: the Phase-37
framework's "extensions fold after default plugins" design
enables wikilinks-style extensions that emit relative URLs the
default sanitization stack would otherwise reject. Documented
explicitly in Phase-38 ADR-0018 D-G APPEND.

D-D. **`WikilinkExtensionRegistry` class shape**:

```ts
export class WikilinkExtensionRegistry implements MarkdownExtensionRegistry {
  constructor(private readonly enabledSurfaces: ReadonlySet<MarkdownSurface>) {}

  getExtensions(surface: MarkdownSurface): MarkdownExtensionSet {
    if (this.enabledSurfaces.has(surface)) {
      return { rehypePlugins: [rehypeResolveWikilinks] };
    }
    return {};
  }
}
```

Constructor takes the set of enabled surfaces; non-enabled
surfaces return empty extension sets (= default behavior).
Phase-38 default enables wikilinks on `actionRationale` only:

```ts
const PHASE_38_DEFAULT_ENABLED = new Set<MarkdownSurface>(["actionRationale"]);
```

Future Phase 39+ may expand the enabled set if cross-surface
wikilink demand surfaces (bio user might want
`[[problem-slug]]`-style internal references too).

D-E. **Factory dispatch arm** = `MARKDOWN_EXTENSIONS=wikilinks`
in `getExtensionRegistry()`. First env-var dispatch arm under
Phase-37 framework (Phase 37 had only the implicit default
arm). Mirrors `getModerator()` D-E throw-on-unknown discipline.

D-F. **Integration points** = ALL FOUR markdown helpers continue

per default-enabled-surfaces-set).
