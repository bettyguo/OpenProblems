### `truncateRationale()` source-truncation incompatibility (anticipated for Unit 18.3)

Current `/profile` rendering uses
`truncateRationale(challenge.reviewNotes)` to limit preview to
~200 chars. Truncating markdown SOURCE mid-string risks breaking
formatting:

- Truncating `**bold text more bold**` to `**bold text mor` →
  unclosed `**` → renders as raw asterisks at best, malformed
  AST at worst.
- Truncating `[link text](https://example.com)` to `[link te` →
  unclosed `[` → renders as raw bracket.
- Multiple line collapses (`...`) lose track of which block-level
  element they're in.

Unit 18.3 resolution: **drop `truncateRationale(challenge.reviewNotes)`
entirely**; render FULL markdown; apply CSS `line-clamp-3` (Tailwind
v4 built-in utility) for visual truncation. The `truncateRationale()`
helper itself stays for the rationale field (Phase-11 plain-text
field; not markdown Phase 18; defer to Phase 19+ if rationale
markdown promotes).
