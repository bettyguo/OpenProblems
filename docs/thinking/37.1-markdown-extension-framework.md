`renderReviewNotesMarkdown` + `renderRationaleMarkdown` and

### Alternatives considered

**Option 1: Mirror `lib/moderation/` shape verbatim (chosen)** —
5 files: `types.ts` + `default.ts` + `index.ts` + 2 test files.
Lazy-singleton factory + (currently noop) env-var dispatch +
`__resetRegistryForTests` hook. **Why chosen**: Phase 35
established the framework-only shape; reusing it verbatim
validates the pattern's reusability per Phase-37-prep D-1
rationale 1.

**Option 2: Class-per-registry vs interface + plain-object
registry** — `lib/moderation/` uses class (`NoopModerator`
implements `ContentModerator`). Markdown framework could
mirror exactly OR use a plain-object literal
(`DEFAULT_EXTENSION_REGISTRY: MarkdownExtensionRegistry = {
getExtensions: () => ({}) }`).
- **Lean: class** — preserves shape-identity with
  `lib/moderation/`. The framework is more "behaves-like-a-
  service" than "is-data"; class signals that intent and
  matches the per-process singleton semantics.

**Option 3: Per-surface methods vs single `getExtensions(surface)`
method** — D-10 lean is single method. Sticking with the lean:
adding a new surface (Phase 38+) means adding a new entry to
`MarkdownSurface` union and one new arm in
`DefaultExtensionRegistry.getExtensions`; per-surface methods
would require 5 method additions per surface.

**Option 4: Mutable extension registry vs immutable** — A
`registry.registerExtension(surface, set)` mutator would let
extensions register at module load. **Rejected**: violates
explicit-control discipline (any module could mutate the
registry as a side effect); harder to test; harder to audit
which extensions are active. Framework stays read-only with
all wiring happening via the factory dispatch arm at
`getExtensionRegistry()` time.

### Edge cases & failure modes

1. **Empty extension set must NOT regress pipeline performance**.
   Folding zero plugins / zero schema-overrides into the existing
   processor MUST resolve to a no-op. Unit 37.2 verifies this
   with the existing 50+ markdown tests passing verbatim.
2. **`schemaOverrides` deep-merge surprise**: `Partial<Schema>`
   covers `tagNames` / `attributes` / `protocols` / etc. as
   top-level keys, but each is itself an object/array. A future
   override with `tagNames: ["table"]` should NOT replace the
   entire base `tagNames` allow-list. Unit 37.2 implements
   shallow-merge at the top level (`{ ...base, ...override }`)
   per D-11 lean **override-replace** — callers supply the
   complete replacement for any field they override. Documented
   in the ADR-0018 D-G APPEND so future curators know to
   include the full base list when overriding.
3. **Factory called before env var read**: Phase 37 factory has
   only one dispatch arm (the default). Future Phase 38+ env-
   var dispatch must throw on unknown values (mirrors the
   `getModerator()` D-E throw-on-unknown discipline). Phase 37
   framework defers throw-on-unknown to Phase 38+ when a
   second arm is added.
4. **Singleton survives across tests** = test pollution. The
   `__resetRegistryForTests` hook clears the singleton per-test;
   factory test file uses `beforeEach`/`afterEach` to reset.
5. **Type errors when calling `getExtensions("unknownSurface")`**:
   `MarkdownSurface` is a TS literal union; misspelled surface
   names fail at compile time. Defensive: `DefaultExtensionRegistry.getExtensions`
   uses an exhaustive switch (no `default` arm), which TypeScript
   verifies exhaustively against `MarkdownSurface`.

## DESIGN

### `MarkdownSurface` literal union

```ts
export type MarkdownSurface = "bio" | "reviewNotes" | "rationale" | "actionRationale";
```

Order mirrors the chronological introduction of each surface
(Phase 17 → 18 → 27 → 29). Future Phase 38+ additions append.

### `MarkdownExtensionSet` shape

```ts
export interface MarkdownExtensionSet {
  remarkPlugins?: ReadonlyArray<Pluggable>;
  rehypePlugins?: ReadonlyArray<Pluggable>;
  schemaOverrides?: Partial<Schema>;
}
```

Uses `Pluggable` from `unified` (re-export of `PluggableList`
member type) for plugin entries; uses `Partial<Schema>` from
`rehype-sanitize` (the existing `Options as Schema` alias) for
schema diffs.

All three fields are optional; an empty `{}` is a valid
`MarkdownExtensionSet` (= no extensions for that surface). The
`DefaultExtensionRegistry` returns `{}` for every surface.

### `MarkdownExtensionRegistry` interface

```ts
export interface MarkdownExtensionRegistry {
  getExtensions(surface: MarkdownSurface): MarkdownExtensionSet;
}
```

Single method per D-10 lean. Implementations return the
extension set for the given surface; concrete implementations
(future Phase 38+) compose their per-surface logic inside this
single method.

### `DefaultExtensionRegistry` class

```ts
export class DefaultExtensionRegistry implements MarkdownExtensionRegistry {
  getExtensions(surface: MarkdownSurface): MarkdownExtensionSet {
    switch (surface) {
      case "bio":
      case "reviewNotes":
      case "rationale":
      case "actionRationale":
        return {};
    }
  }
}
```

Exhaustive switch over `MarkdownSurface`; no `default` arm so
TypeScript catches missing surfaces at compile time when the
union expands Phase 38+. Returns the same empty-object literal
shape every call — Day-1 behavior is verbatim Phase-18/27/29
baseline.

### `getExtensionRegistry()` factory

```ts
let registryInstance: MarkdownExtensionRegistry | null = null;

export function getExtensionRegistry(): MarkdownExtensionRegistry {
  if (registryInstance) return registryInstance;
  registryInstance = new DefaultExtensionRegistry();
  return registryInstance;
}

export function __resetRegistryForTests(): void {
  registryInstance = null;
}
```

Single-arm dispatch Phase 37. Future Phase 38+ env-var
dispatch arms add after the current `if` per ADR-0018 D-G
APPEND D-F deferral list (e.g., `MARKDOWN_EXTENSIONS=wikilinks`
→ a `WikilinkExtensionRegistry`).

`__resetRegistryForTests` mirrors `__resetModeratorForTests`
from `lib/moderation/index.ts` Phase 35 + `__resetResendClientForTests`
from `lib/email/index.ts` Phase 30 — project's established
pattern for vitest per-suite singleton reset.
