# 7.8a — `app/robots.ts` (sister convention to `app/sitemap.ts`)

Sister-unit to Unit 7.8 (sitemap). Picks up the Class B follow-on flagged in [Unit 7.9](7.9-phase-7-hygiene.md): "`app/robots.ts` — sister convention to `app/sitemap.ts`". Permissive default (allow all crawlers) + sitemap pointer. No User-Agent / Disallow policy yet — punted on a future ADR if the project decides on an AI-scraping posture.

## THINK

### Why now

- Unit 7.8 added `/sitemap.xml`. Crawlers find it via two channels: (1) explicit `Sitemap:` directive in `/robots.txt`; (2) heuristic discovery (Google checks `/sitemap.xml` directly). Channel 1 is canonical; without `/robots.txt`, crawler-side discovery relies on heuristics or webmaster-tool registration.
- Adding `/robots.txt` with a `Sitemap:` line completes the sitemap-discoverability story in one tiny unit.
- Cost: ~10 lines of code. Blast radius: zero.

### Inputs

- **[Unit 7.8](7.8-sitemap.md)** — created `app/sitemap.ts` + `lib/sitemap/build-sitemap.ts` (exports `SITE`).
- **[Unit 7.9](7.9-phase-7-hygiene.md) Class B** — `app/robots.ts` flagged as deferred, unlock condition = "a User-Agent / Disallow policy decision (AI-scraping posture; no Q surfaced yet)". This unit ships the **permissive baseline** without making a policy decision. Stricter rules can be layered in via a future ADR.

### Scope (this unit)

Two new files + one edit:

1. **`app/robots.ts`** (new) — Next.js convention entry-point. Returns `MetadataRoute.Robots` with:
   - `rules: [{ userAgent: "*", allow: "/" }]` — permissive default; allows every crawler everywhere.
   - `sitemap: ${SITE}/sitemap.xml` — points at the Unit 7.8 sitemap.
   - `host: SITE` — canonical host (helps crawlers deduplicate www/non-www; harmless if redundant).
   - Reuses `SITE` from `lib/sitemap/build-sitemap.ts` (no hardcoded duplication; matches the Q-flag in Unit 7.8's scope-NOT-in-this-unit list).
2. **`app/robots.test.ts`** (new) — vitest case: calls the default export, asserts shape (`rules` array present; `sitemap` URL ends with `/sitemap.xml`; uses `SITE` prefix).
3. **`CHANGELOG.md`** — Phase-7 Unit 7.8a entry.
4. **`docs/thinking/7.8a-robots.md`** (this doc).

### Scope NOT in this unit

- **No AI-scraping posture** — no `User-agent: GPTBot` / `Disallow: /` block (or any other AI bot). A deliberate policy ADR is the right shape; a knee-jerk block-all is reactionary.
- **No `Disallow` for any path** — `/api/v1/*` could theoretically be excluded (machine routes), but the JSON / RSS endpoints are exactly what we WANT crawlers to discover. `/drafts/` is gitignored (never reaches the server). Nothing to disallow.
- **No `Crawl-delay`** — site has no rate-limit issues; no `429s` observed.
- **No alternate-language directives** — `robots.txt` doesn't standardize this; hreflang is handled via `<link rel="alternate" hreflang>` (deferred to a future SEO-meta unit) + the sitemap's `alternates.languages` block (Unit 7.8 ships this for `/about`).

### Constraints

- **`app/robots.ts` is a Next.js convention** — same special-file pattern as `app/sitemap.ts`. Default export returns `MetadataRoute.Robots`; arbitrary helper exports forbidden.
- **`exactOptionalPropertyTypes: true`** — `host` is optional on the Robots type; only assign when defined (always defined here, but follow the pattern).
- **No new dependencies** — `MetadataRoute.Robots` ships with `next`.
- **Reuses `SITE` from `lib/sitemap/build-sitemap.ts`** — second consumer; centralizing is not yet justified (Unit 7.8's threshold was 5+ call sites). Keep import path.

### Alternatives considered

- **Block AI bots (`User-agent: GPTBot Disallow: /`)** — Rejected. Project policy on AI scraping isn't decided; defaulting to "block" pre-empts a substantive decision; defaulting to "allow" matches the permissive open-content posture (CC-BY-4.0 content license) baseline.
- **Inline `SITE` rather than import from `lib/sitemap/build-sitemap.ts`** — Rejected. Two call sites in `app/` already use it (via the sitemap); importing keeps the constant single-source even though Unit 7.8 flagged the centralization-threshold as 5+.
- **Use `MetadataRoute.Robots`'s `disallow` to exclude `/_next/` or `/api/`** — Rejected. Next.js serves these as machine routes; crawlers naturally don't surface them in indexing (no inbound links from indexed pages). Adding explicit disallows is noise.

### Tradeoffs / defensible defaults flagged

1. **Permissive baseline** — every crawler allowed. If the project later wants stricter rules (block specific bots, throttle), it's an additive change. Defaulting open is reversible; defaulting closed is harder to soften without breaking SEO discoverability.
2. **`host` set explicitly** — minor positive: crawlers don't have to infer; downside: if `SITE` changes, this needs to update in the same PR. Coupled by import, so safe.
3. **No `User-agent` blocks for AI bots** — defaulting permissive matches the project's open-content + open-research thesis. Curator can ADR-grade reject crawler classes later.
