# Unit 1.5 — RatingRadar v1

> Stage 1 (THINK) artifact. Critical-path.

## Problem restatement

§11 lists `RatingRadar` as viz catalog entry #1 — "5-axis radar, animated entry, dimension-tooltips, confidence shading. (Problem detail.)". This unit ships v1: a self-contained SVG radar that renders the five rating dimensions for one rating action, sized for the Problem-detail page's header card (Unit 1.6). No data history (that comes with `RatingHistoryStream` in Phase 3).

## Inputs

- §3.1 — five dimensions: Difficulty (S–E), Saturation (0–100), Urgency / Value / Industry Call (0–5 ⭐).
- §8.3 — composite formula maps each dimension into the `[0, 5]` space; (100 − Saturation) / 20 is the saturation contribution. Reusing that normalization for the radar plot.
- §10.1 — palette: chart hues 1–5 per the five dimensions, all WCAG-AA on background.
- §11 — animated entry, dimension tooltips, confidence shading.
- §14.1 — component folder structure: `index.tsx`, `index.stories.tsx`, `index.test.tsx`, optional `README.md`.
- §14.2 — viz needs Storybook story + Playwright screenshot. Phase 1.5 ships the story; Phase 1.12 baselines the screenshot.
- §10.3 — every chart is also a table.
- §10.4 — perf budget; viz code-splitting via `next/dynamic` only if size demands. v1 has no client JS — it's pure SVG; will inline.

## Constraints

- **RSC-safe.** No `"use client"`. Pure SVG with `<title>` elements for tooltips (native browser behaviour, also screen-reader-friendly).
- **No external chart library.** D3 / Visx come later when the algorithm needs polyline simplification, area generation, brushing. A 5-axis polygon is straightforward trigonometry.
- **Confidence shading.** Fill opacity = mean confidence across the five dimensions, mapped to `[0.15, 0.55]`. Stroke is full opacity. This makes a low-confidence radar visually distinguishable without disappearing.
- **Animated entry.** CSS `@keyframes` on the polygon: scale from 0 → 1, 250 ms `ease-out`. Honours `prefers-reduced-motion` via the global guard in `globals.css`.
- **Themeable.** All colors via CSS variables — `--color-chart-1..5`, `--color-border`, `--color-muted-foreground`, `--color-background`. Dark mode works automatically through `.dark`.
- **Per-dimension normalization** lives in a sibling helper `lib/ratings/normalize.ts` (single source of truth used by the radar v1 *and* by the composite calculation that arrives in Unit 1.6 or 1.8).
- **Accessibility.** `role="img"`, `aria-label`, `aria-describedby` pointing at a hidden `<desc>` block summarising the values. The companion table fallback ships in Unit 1.6 (problem detail page); v1 just gets the chart-only state right.

## Alternatives considered

1. **Pure SVG + RSC + CSS animation** (chosen). Smallest bundle; RSC ships zero JS to the client.
2. **D3-based polygon.** Overkill for 5 points; pulls D3 client-side.
3. **`<canvas>`.** Smaller per-pixel but breaks accessibility primitives and dark-mode CSS-variable consumption.
4. **Recharts.** A reasonable choice for many charts but doesn't allow the CSS-variable theming the project wants on every chart.

## Edge cases & failure modes

1. **Missing dimension.** A v1 rating action *must* include all five (RatingActionSchema is strict). If a future schema version relaxes that, the radar code currently assumes five — surface as a bug in the consumer rather than silently defaulting.
2. **Confidence = 0.** Fill opacity floor is 0.15 so the polygon doesn't disappear; stroke stays full.
3. **All-zeros polygon.** Degenerates to a single point at center. Acceptable; the radar still renders the axes and labels.
4. **`prefers-reduced-motion`.** Global guard sets animation-duration to 0.01 ms — entry effectively snaps.
5. **Chart hue collision with brand accent.** Unit 1.0 already nudged `--chart-saturation` to hue 140°; safe.
6. **Tooltip behaviour on touch devices.** Native `<title>` doesn't show on touch. Acceptable for v1; richer tooltip is Phase 2 work.

## Open questions surfaced

- **Q35. Companion table fallback.** §10.2 mandates "every chart is also a table". v1 ships the chart only; the table-fallback toggle lives on the problem-detail page (Unit 1.6) so it can be ergonomic across the page. Surface for traceability.

Stage 2 (DESIGN): geometry —

- viewBox 0 0 200 200, center at (100, 100), max radius 80.
- 5 axes at angles `-90° + i * 72°` (first axis points up).
- 5 grid rings at radii 16, 32, 48, 64, 80 (= 1/5, 2/5, … of max). Plus center dot.
- Value polygon: 5 vertices at `(cx + cos(angle) * r, cy + sin(angle) * r)` where `r = (value / 5) * 80`.
- Axis labels at radius 92 along each axis.
- Each axis colored with its `--color-chart-N`; polygon stroke `--color-foreground` with fill `--color-foreground / α(confidence)`.
