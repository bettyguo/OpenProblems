## THINK

### UI shape decision (resolves Phase-36-prep D-10 + D-11 + D-12)

Per Phase-36-prep D-10 lean: **labeled checkbox in a `<form>` with
submit-on-change**. Mirrors Phase-15 displayName/bio form pattern
(no new UI primitive; no client-side state; no shadcn `Switch`).
Checkbox state becomes the form value via the standard HTML form
semantics: checked → `formData.get("isPublic") === "on"`; unchecked
→ `formData.get("isPublic") === null`.

Per Phase-36-prep D-11 lean: **NO confirmation modal**. Toggle is
reversible; single-click; no destructive action. User can re-toggle
immediately if they change their mind.

Per Phase-36-prep D-12 lean: **status surface placement = within
form description text**. Renders `"Your profile is currently public."`
or `"Your profile is currently private."` above the checkbox so the
user can see current state without needing to recall what the
checkbox represents.

### Server action shape

```ts
const updateProfilePrivacyAction = async (formData: FormData) => {
  "use server";
  const actionSession = await auth();
  if (!actionSession?.user?.id) {
    redirect(`/api/auth/signin/github?callbackUrl=/${locale}/profile`);
  }
  const isPublic = formData.get("isPublic") === "on";
  await setProfilePublic(actionSession.user.id, isPublic);
  revalidatePath(`/[locale]/profile`, "page");
  revalidatePath(`/[locale]/u/[handle]`, "page");
  redirect(`/${locale}/profile?saved=privacy`);
};
```

Mirrors Phase-15 `updateProfileAction` shape verbatim: auth check
+ redirect-if-unauth + extract field + call helper + revalidate +
redirect with success param. The double `revalidatePath` ensures
both the user's own `/profile` and any cached `/u/{handle}` for
their handle re-render with the new visibility.
