# 5.13b — `/contributing` v1.1 bump (LLM-assisted ingest path + COI disclosure)

Phase-5 post-acceptance hygiene unit. Mirrors Unit 5.13a's "Phase-5 hygiene that
belongs to Phase 5" framing. Documents the LLM-assisted curation path that
shipped in Units 5.3 / 5.5 (CLIs) + 5.7 / 5.8 / 5.9 (digest pipeline) + ADRs
0008 / 0009 (provider + diff contract), and lands the conflict-of-interest
disclosure that ADR-0008 D-F explicitly defers to a "future content-side
unit".

## THINK

### Inputs

- **`content/contributing/v1.mdx`** (Unit 4.5, `1c83f61`) at `version: "1.0.0"`.
  Six sections; tone matches MASTER_PROMPT.md / methodology v1.mdx; written for
  external curators.
- **ADR-0008 D-F** (`docs/adr/0008-llm-provider-anthropic.md`) explicitly
  defers the COI disclosure to "a future content-side unit (likely a
  `content/contributing/v1.x` bump or a `/methodology` § appendix)" and pins
  the disclosure verbatim:
  > "This encyclopedia uses Anthropic Claude to draft curator suggestions,
  > including for papers from Anthropic itself. Curators are expected to apply
  > equal scrutiny regardless of authorship."
- **ADR-0009 D-D / D-E** (`docs/adr/0009-human-review-diff.md`): the no-auto-
  merge diff contract + `drafts/` lifecycle. Curator runs `git apply
  drafts/<file>.diff` after review; CLIs never write to `content/`. This is
  the workflow external curators need to know about if they pick up the LLM
  CLIs themselves.
- **`SESSION_HANDOFF_phase5_close.md`** + **`5.13-phase-5-acceptance-gate.md`**
  list this bump as a non-blocking Phase-5 follow-on.
- **`app/contributing/page.tsx`** sorts the `contributing` collection by
  `version` frontmatter and renders the latest at `/contributing`. The
  `[version]` route emits one SSG page per `version` in the collection. This
  means a new `v1.1.mdx` with `version: "1.1.0"` adds one new SSG route
  (`/contributing/v1.1.0`) and flips `/contributing` to render v1.1 — the
  v1.0 audit URL `/contributing/v1.0.0` stays live from the existing
  `v1.mdx`.
- **Velite `contributing` collection** (`velite.config.ts` L75-95) globs
  `contributing/*.mdx`. New file is auto-discovered; no config change.

### Constraints

- **Tone parity with v1.0**. Direct, technical, no marketing register.
  Same heading structure; same link style (full GitHub URLs to MASTER_PROMPT
  and ADRs; relative links for in-site routes).
- **No fabricated URLs** per §15.6. CLI script names verified against
  `package.json` (`ingest-arxiv` + `extract-leaderboard` confirmed at
  L`pnpm ingest-arxiv` / L`pnpm extract-leaderboard`).
- **Frontmatter compatibility** with the `contributing` Velite collection
  (mirrors `methodology` 1:1: `version` / `title` / `summary` / `date` /
  optional `supersedes`).
- **`exactOptionalPropertyTypes`** — `supersedes: "1.0.0"` (string), no
  `undefined` issue.
- **Forward-compatible with v1.2+**. Section numbering uses the existing
  `§N.M` register so future v1.2 (e.g. `§3.7` something or `§8` something)
  slots in without breaking inbound links.
- **§13 contributor pact preserved.** The four contribution types
  (§2's table) remain the canonical front-door for outside contributors;
  the new §7 explicitly frames LLM CLIs as curator-side, not contributor-
  side, so contributors aren't asked to set up an `ANTHROPIC_API_KEY`.

### Alternatives considered

- **Edit `v1.mdx` in place + bump version to 1.1.0**. Rejected — loses the
  v1.0 audit trail. The §5 versioning paragraph in v1.0 explicitly promises
  "every committed contributing-version stays browsable"; the loader's
  `[version]` route depends on each version having its own MDX file.
- **Put the COI disclosure on `/methodology` instead of `/contributing`**.
  ADR-0008 D-F lists both as candidates. Rejected — the disclosure is
  about curator workflow (LLM-drafted suggestions reviewed by curators);
  /contributing is the curator-workflow page. Methodology is about rating
  dimensions. /contributing is the right home; /methodology mirror would
  be redundant.
- **Two separate units (5.13b for COI; 5.13c for §7 LLM workflow)**.
  Rejected — both surfaces are the same content shape (additive sections
  in v1.1.mdx); splitting doubles the commit overhead without splitting
  the diff in any reviewable way.
- **Skip §7 entirely; only land §3.6**. Rejected — the LLM workflow IS
  shipped Phase-5 surface that contributors see (drafts on PRs reference
  the unit numbering 5.3/5.5; reviewers need to know what they're looking
  at). The handoff explicitly lists "documenting the LLM-assisted ingest
  path" as the bump's primary scope.
- **Inline the full diff format spec from ADR-0009 in §7**. Rejected —
  the ADR is the canonical spec; /contributing should link to it, not
  duplicate it. Same pattern as v1.0's §3.3 linking to ADR-0005 instead
  of inlining the immutability rules.

### Tradeoffs / defensible defaults flagged

1. **Two MDX files now (v1.mdx + v1.1.mdx).** Acceptable per the loader
   contract — collection is small (will accumulate v1.2, v1.3, … over the
   project's lifetime). The version-history nav in
   `app/contributing/page.tsx` L34-49 already renders all versions as
   inline links. No UX degradation.
2. **`supersedes: "1.0.0"`** uses the methodology precedent's anticipated
   shape (v1.mdx had `supersedes: undefined`; v2.mdx would set
   `supersedes: "1.0.0"`). v1.1 is the first to actually exercise this
   field. Velite schema accepts it as `s.string().optional()`.
3. **Date stamp = 2026-05-16** (today; the `currentDate` from session
   context). Matches the curator-rhythm convention of dating the bump
   to the day the curator wrote it.
4. **Page count goes 322 → 323** (the new `/contributing/v1.1.0` SSG
   route). First Load JS shared chunk should remain 103 kB unchanged
   (MDX renders to HTML server-side; no new client deps).
5. **No `lighthouserc.json` change**. The `/contributing` URL is already
   enrolled (Phase-4 Unit 4.6); the new `/contributing/v1.1.0` versioned
   snapshot inherits the same template and would have identical a11y/perf
   characteristics. Adding the versioned snapshot to the LHCI cohort would
   2× the audit cost on every PR for marginal coverage. Skip.
