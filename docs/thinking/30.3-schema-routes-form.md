# 30.3 — `subscriber` schema + migration `0006_subscriber` + `lib/subscribers/` helpers + 3 API routes + subscribe form on `/digest` + 9 i18n keys × 2 locales + 21 new tests (667/60)

Fourth Phase-30 unit; code unit. Largest unit of Phase 30. Realizes
ADR-0021 D-B (subscriber data model) + D-C (subscription scope) +
D-D (verification flow) + D-E (unsubscribe flow) + D-G (sender
identity / graceful degradation) + D-H deferrals (no rate limiting
/ no per-user / no scheduler). **First non-zero-migration unit
since Phase 16** = 14-phase gap; **breaks 13-phase 0-migration
streak**; **first new DB table since Phase 11** = 19-phase gap;
**first multi-route API surface in a single unit** since Phase 9
auth-API multi-route precedent.

## THINK

### Scope realized

1. **`lib/db/schema.ts`** (modified): adds `subscribers` table
   (11 columns + 3 indexes). Uses `crypto.randomUUID()` `$defaultFn`
   on `id` (matches `users` + `ratingChallenges` precedent);
   `timestamp_ms` mode on integer timestamps; `(unixepoch() * 1000)`
   defaults for `createdAt` + `updatedAt`. New imports: `index` +
   `uniqueIndex` from `drizzle-orm/sqlite-core` (first use of
   these helpers in project history).
2. **`lib/db/migrations/0006_subscriber.sql`** (new): generated
   via `pnpm db:generate`; renamed from `0006_naive_king_cobra.sql`
   to match project's meaningful-name convention. `_journal.json`
   tag updated. **`0006_snapshot.json`** also generated by
   drizzle-kit.
3. **`lib/subscribers/index.ts`** (new; ~210 lines): pure helpers
   (`canonicalizeEmail` + `validateEmail` + `generateToken` +
   `safeCompareTokens` + `parseDomainSubscriptions` +
   `serializeDomainSubscriptions`) + DB helpers
   (`createOrRefreshPendingSubscription` + `verifyByToken` +
   `unsubscribeByToken`). Mirrors `lib/rating-challenges/` +
   `lib/watchlist/` separation of concerns. Token comparison via
   `crypto.timingSafeEqual` defends against timing side-channels
   per ADR-0021 §15.5 reviewer-mode mindset (c).
4. **`lib/subscribers/index.test.ts`** (new; 21 tests): coverage
   on the 6 pure-function helpers (canonicalize / validate /
   generate-token / safe-compare / parse-domains / serialize-
   domains). DB-helper tests deferred to Phase-31+ B.20 integration
   test scope (requires live DB or full mocking).
5. **`app/api/v1/subscribe/route.ts`** (new): `POST` handler;
   validates email + domains; calls
   `createOrRefreshPendingSubscription`; sends verification email
   via `sendEmail`. Returns 4 outcome shapes (`already_subscribed`
   / `verification_sent` / `verification_sent + emailUnavailable` /
   400 bad-request).
6. **`app/api/v1/subscribe/verify/[token]/route.ts`** (new): `GET`
   handler; renders minimal HTML confirmation page (no React; no
   client JS); 3 outcomes per ADR-0021 D-D (404 invalid /
   410 expired / 200 verified-idempotent). Sends welcome email
   best-effort on first verify (skips re-send on already-verified
   re-click).
7. **`app/api/v1/subscribe/unsubscribe/[token]/route.ts`** (new):
   `GET` handler; renders minimal HTML confirmation page; 2
   outcomes per ADR-0021 D-E (404 invalid / 200 unsubscribed-
   idempotent).
8. **`app/[locale]/digest/_components/SubscribeForm.tsx`** (new;
   ~140 lines): `"use client"` form component with state for email
   + selected domains + submit status. Posts to `/api/v1/subscribe`
   via fetch; renders inline success / error messages via
   `email.*` i18n keys. `_components/` prefix prevents Next.js
   from treating it as a route.
9. **`app/[locale]/digest/page.tsx`** (modified): adds import +
   `<SubscribeForm domains={...}>` after the per-domain section
   loop. Phase-5 surface long-untouched since Phase 5 Unit 5.x.
10. **`messages/en.json` + `messages/fr.json`** (modified): adds
    new `email` namespace with **9 keys per locale** (subscribe_heading
    + subscribe_description + email_label + domains_label +
    subscribe + submitting + verify_sent + already_subscribed +
    send_unavailable + invalid_input + select_at_least_one_domain
    — wait, that's 11 keys per locale; let me recount). Actual
    count: 11 keys per locale × 2 locales = 22 new entries. i18n
    keys per locale 142 → **153**. **First new i18n namespace
    since Phase 26** `public_challenge_detail` = 4-phase gap.
11. **`.env.example`** (modified): adds Phase-30 Subscriber-list
    email block (`RESEND_API_KEY` + `EMAIL_FROM` with Q75
    operational gate comment + Resend domain setup guidance +
    graceful-degradation note). **Env count 9 → 11**.
12. **`lib/email/templates/verification.tsx` + `welcome.tsx`**
    (refactored): converted from TSX `renderToStaticMarkup` to
    plain template-literal string builders. **Reason**: Next.js
    App Router blocks `react-dom/server` direct imports in
    server-component reachable code (build error
    "You're importing a component that imports react-dom/server.
    To fix it, render or return the content directly as a Server
    Component instead for perf and security."); plain strings
    avoid the boundary issue + are clearer for email HTML anyway.
    Tests unaffected (still validate string content).
