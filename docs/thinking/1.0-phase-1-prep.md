# Unit 1.0 — Phase 1 prep: license, brand, accent

> Stage 1 (THINK) artifact for the Phase 1 opener. Small, no code changes beyond branding + license boilerplate. Resolves the three OPEN_QUESTIONS items the Unit 0.12 triage flagged as Phase-1-blocking.

## Problem restatement

Land three blocking decisions before any Phase 1 visual / content work begins:

1. **Q1 — Brand name.** Confirmed as **LLM OpenProblems** (the working title). No alias needed.
2. **Q4 — License.** Code under **Apache-2.0** (explicit patent grant; defensible for a research-adjacent project that intends a citable methodology paper). Content under **CC-BY-4.0** (attribution required, derivatives allowed, commercial use allowed — standard for academic-adjacent published content).
3. **Q5 — Brand accent.** **Deep cyan, HKU-green-leaning.** OKLCH hue 170° (between pure cyan ~195° and pure green ~145°, matching the "HKU green" digital register the user referenced). Replaces the Unit 0.4 placeholder at hue 215°.

After this unit, every Phase 1+ deliverable can proceed without "wait, what is the accent" or "can we publish this yet" overhead.

## Constraints

- **Apache-2.0 full text** committed at `LICENSE` (the canonical project-root location; SPDX expects this for license-aware tooling).
- **CC-BY-4.0 explanation** at `content/LICENSE.md` (path makes scope obvious — content/ is the thing licensed CC-BY).
- **`package.json#license`** set to `Apache-2.0` (SPDX identifier).
- **One-line accent swap.** Per §10.1's "variable names stable, values free" rule from Unit 0.4. `--accent` and `--ring` change values only; component code consumes the tokens.
- **WCAG AA check.** At OKLCH `0.45 0.10 170` on `--background` `oklch(0.984 0.003 96)`, contrast ratio is ~4.7:1 — clears AA for normal text (4.5:1). Dark variant at `0.68 0.13 170` on dark background `0.149 0.006 254` clears 4.5:1 too.
- **Chart-hue collision.** `--chart-saturation` is at hue 160°, accent now at 170°. Δ = 10° hue. Mitigation: nudge `--chart-saturation` from 160° → 140° (warmer-leaning sage green) so chart-2 and the accent are ≥ 30° apart. This is a one-line edit, well within Unit 0.4's "values free" rule.

## Alternatives considered

1. **Apache-2.0 + CC-BY-4.0** (chosen). Defensible for citable research artifact + patent grant.
2. **MIT + CC-BY-4.0.** Simpler license boilerplate; widely adopted in academic web tooling; *no* patent grant — slightly weaker downstream protection. Defensible but I lean Apache for this project.
3. **MIT + CC-BY-SA-4.0.** Share-alike pressures derivative content authors. Rejected — friction for citation-as-derivative use cases.
4. **Single LICENSE with both texts.** Cleaner search hits, but tooling that reads `LICENSE` expects one license. Rejected.

For Q5:

1. **OKLCH hue 170° (deep teal / HKU-green register)** (chosen).
2. **OKLCH hue 195° (pure deep cyan).** Closer to the original "deep cyan" wording in §10.1 but further from the HKU reference the user added.
3. **OKLCH hue 160° (more green-leaning).** Reads more "forest" than "academic teal"; collides with `--chart-saturation` even without the chart-2 nudge.
4. **Vermilion.** Q5's stated alternative; user closed it by choosing cyan.

## Edge cases & failure modes

1. **Chart-2 (saturation) collision.** Mitigated by nudging chart-2 from 160° → 140° in both light and dark blocks.
2. **Apache-2.0 NOTICE file.** Apache-2.0 *requires* a NOTICE file only if the project receives one from a dependency or wants to add attribution metadata. We don't — skip for now. Add when needed (e.g., bundled assets with attribution).
3. **`package.json#license`** at `"Apache-2.0"` (SPDX). Some npm tools also want `licenses` array; SPDX `license` is canonical.
4. **CC-BY-4.0 on YAML and MDX both.** `content/` covers `problems/`, `papers/`, `authors/`, `institutions/`, `methodology/`, `taxonomy.yaml`. Scope cleanly delimited.
5. **README license section.** Update from "Pending — see Q4" to "Code: Apache-2.0 · Content (`content/`): CC-BY-4.0".
6. **Visual diff on `/`.** The chart-1..5 legend will visibly shift chart-2's hue. Acceptable — this is the brand finalization, not a regression.
