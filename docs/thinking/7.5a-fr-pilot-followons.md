   - **`/methodology` alternates assertion** — mirrors the `/about` test (line 45).

- **Sitemap entries for `/<locale>/methodology` themselves**. Sitemap convention: the canonical bare URL carries `alternates.languages`; locale-prefixed URLs are NOT duplicated as top-level entries (they're discoverable via the alternates map). - **`/methodology/v1` alternates on the alternate-locale URLs themselves**. Per Google sitemap-i18n spec, every alternate URL points to itself + every other alternate. The `MetadataRoute.Sitemap` shape from Next.js attaches alternates to a single entry; we attach to the canonical (bare) URL only. Mirrors Unit 7.8's `/about` handling.

### Constraints

- **Sitemap test invariant** (line 56 of `build-sitemap.test.ts`): `does NOT attach locale alternates to routes without a [locale]/ shadow`. Currently checks `/problems` and `/digest`. Adding `/methodology` alternates wouldn't break this test (it asserts only on `/problems` + `/digest`), but it's worth noting that `/methodology` is now in the "has-a-shadow" set.
- **EN-canonical URL only**. Alternates target `/<locale>/<route>`; bare canonical URL holds the alternates block. Don't add per-locale top-level entries (would duplicate URLs in the sitemap).
- **Per-version alternates**: each bare `/methodology/v<X>` entry gets its own alternates block pointing to `/<locale>/methodology/v<X>`. Currently there's only `/methodology/v1`; if a future v1.0.1 lands, the loop adds it automatically.
- **LHCI URL count drift**: now 18; previously 16; Unit 7.7's CHANGELOG mentioned "14 → 16"; this unit takes 16 → 18. Unit 7.11 acceptance gate's URL count summary will report 18.

### Alternatives considered

- **Land LHCI + sitemap as two separate units (7.7a + 7.8b)** — Rejected. Both are ≤10-line edits in completely different files; bundling avoids two pre-commit-hook test cycles (each takes ~15s) for trivial work.
- **Add `/methodology/v1` to LHCI alongside the latest-page enrol** — Rejected. The versioned-snapshot page mirrors the latest-page rendering 1:1; double-canarying offers no signal. LHCI's job is regression detection on canonical landing surfaces.
- **Use a `LOCALE_ALTERNATE_ROUTES` table at the top of `build-sitemap.ts` instead of inline `if`-branches** — Rejected for now. With 2 entries (`/about`, `/methodology`) the inline if-chain is 4 lines; a table is overhead. Extract when the count reaches 5+ (matches the SITE-constant extraction trigger flagged in Unit 7.8).
- **Add `/contributing` alternates speculatively** — Rejected. The locale-aware `/[locale]/contributing/` routes don't exist; alternates pointing to 404s violate the sitemap-test invariant + would mislead crawlers. Defer until the FR contributing pilot (a curator-track item per Q51 lean).

### Tradeoffs / defensible defaults flagged

1. **LHCI canaries only the methodology landing page, not the versioned URLs**. Matches 7.7's about-page choice. If a future regression breaks `/<locale>/methodology/v1.0.0` rendering specifically (vs the landing page), LHCI would miss it. Acceptable; the version page shares ~95% rendering pipeline with the landing page.
2. **Sitemap alternates attached only to the bare canonical URL**. Per-locale URLs (e.g., `/en/methodology`) are NOT enrolled as separate sitemap entries. Google's i18n spec recommends bidirectional cross-references; we follow Next.js's `MetadataRoute.Sitemap` single-entry-with-alternates pattern (matches Unit 7.8's `/about` treatment). Crawlers receive alternates via the canonical URL's metadata block.
3. **No `methodology/v1.0.0` deep-link LHCI enrolment**. Same rationale as point 1.
4. **Defensive ordering**: this unit lands AFTER 7.5 (the FR pilot) but its `/en/methodology` + `/fr/methodology` URLs were minted by 7.5. Without this unit, the URLs render correctly but lack LHCI canary + sitemap-alternate discoverability. Acceptable intermediate state for ~2 commits; closes here.
