### D-3. `MarkdownExtensionRegistry` framework shape anticipated (Unit 37.1)

D-A. **Registry shape** = per-surface configuration object:

```ts
export interface MarkdownExtensionSet {
  remarkPlugins?: ReadonlyArray<unknown>; // PluggableList shape from unified
  rehypePlugins?: ReadonlyArray<unknown>;
  schemaOverrides?: Partial<Schema>; // rehype-sanitize Schema diff
}

export type MarkdownSurface = "bio" | "reviewNotes" | "rationale";

export interface MarkdownExtensionRegistry {
  getExtensions(surface: MarkdownSurface): MarkdownExtensionSet;
}
```

D-B. **Default registry** = `DefaultExtensionRegistry` returns empty
extension set for all 3 surfaces. Zero behavioral change vs Phase
18 + 27 baseline.

D-C. **Factory** = `getExtensionRegistry()` mirrors Phase-35
`getModerator()` shape: lazy singleton + env-var dispatch (currently
just `default`; future values per Phase 38+ candidate extensions).

D-D. **Integration points** (Unit 37.2 wires all 3 markdown helpers):

1. `renderBioMarkdown` reads `registry.getExtensions("bio")` + folds
   `schemaOverrides` into `bioSchema` + folds `remarkPlugins` /
   `rehypePlugins` into the unified pipeline.
2. `renderReviewNotesMarkdown` reads `registry.getExtensions("reviewNotes")`.
3. `renderRationaleMarkdown` reads `registry.getExtensions("rationale")`.

D-E. **Test surface** = framework tests verify default-returns-empty
+ singleton-caching + future-extension-override-merge correctness.
Existing 50+ markdown tests pass verbatim (default empty extension
set = zero behavioral change).
