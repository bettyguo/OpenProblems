### D-3. arXiv extension shape anticipated (Unit 41.1)

D-A. **Plugin shape** = `remarkLinkArxivIds` remark plugin that
walks the mdast tree, finds text-node substrings matching the
arXiv-ID pattern, splice-replaces each match with a `link`
mdast node:

```ts
import type { Link, Paragraph, Root, Text } from "mdast";
import type { Plugin } from "unified";
import { visit } from "unist-util-visit";

// Modern arXiv ID format: YYMM.NNNNN[vN]
// Older-style category-prefixed IDs (e.g., math/0211159) are
// out of scope Phase 41 (pre-2007; not encountered in the
// project's existing paper-evidence URLs).
const ARXIV_PATTERN = /\barxiv:(\d{4}\.\d{4,5})(v\d+)?\b/gi;

export const remarkLinkArxivIds: Plugin<[], Root> = () => (tree) => {
  visit(tree, "text", (node, index, parent) => {
    if (typeof node.value !== "string") return;
    if (!parent || index === undefined) return;
    if (!node.value.toLowerCase().includes("arxiv:")) return;
    // Split text on arxiv:NNNN.NNNNN[vN] matches; replace each
    // match with a `link` mdast node:
    //   { type: "link", url: "https://arxiv.org/abs/NNNN.NNNNN[vN]",
    //     children: [{ type: "text", value: "arxiv:NNNN.NNNNN[vN]" }] }
    // Defer to Unit 41.1 for the exact split-and-replace logic.
  });
};
```

The plugin runs at the mdast stage (after `remark-parse` +
`remark-gfm`; before `remark-rehype`), so the emitted `link`
nodes flow through the standard mdast→hast bridge and end up
as `<a href="https://arxiv.org/abs/...">arxiv:...</a>` in
HAST. The default `rehype-sanitize` step then validates the
output against the schema; `<a>` + `href` + `https:` scheme
are all already in the Phase-17 base allow-list — **no
`schemaOverrides` needed**.

D-B. **URL emission convention**: `arxiv:NNNN.NNNNN[vN]` →
`<a href="https://arxiv.org/abs/NNNN.NNNNN[vN]">arxiv:NNNN.NNNNN[vN]</a>`.
The URL fragment preserves the version suffix if matched (e.g.,
`arxiv:2024.01234v3` → `.../abs/2024.01234v3`). Case-insensitive
match (`arxiv:` / `ArXiv:` / `ARXIV:`); displayed text
preserves source casing of the prefix.

D-C. **Plugin-order placement under Phase-37 framework discipline**:
The Phase-37 framework folds extension `remarkPlugins` AFTER the
default `remark-parse` + `remark-gfm` per APPEND-D-D plugin-
order-after-default discipline. arXiv autolinks fold after GFM
so a GFM-table cell containing `arxiv:NNNN.NNNNN` also gets
auto-linked (composes cleanly with Phase 39 tables consumer
when both surfaces overlap; not currently the case — see D-G
composition below).

D-D. **Schema overrides**: NONE required. The emitted `<a>`
element with `href="https://arxiv.org/abs/..."` is fully covered
by the Phase-17 base allow-list (`bioSchema.tagNames` includes
`"a"`; `bioSchema.attributes.a` includes `"href"`;
`bioSchema.protocols.href` includes `"https"`). The
`rehypeStripUnsafeHrefs` defense-in-depth step does NOT strip
absolute `https://` URLs (it strips `javascript:` / `data:` /
`file:` / relative URLs). arXiv autolinks therefore differ
architecturally from Phase-38 wikilinks: wikilinks emit
relative URLs that bypass `rehypeStripUnsafeHrefs` only because
the plugin folds AFTER that step; arxiv autolinks emit absolute
URLs that pass that step naturally.

D-E. **`ArxivExtensionRegistry` class shape**:

```ts
export class ArxivExtensionRegistry implements MarkdownExtensionRegistry {
  private readonly enabledSurfaces: ReadonlySet<MarkdownSurface>;

  constructor(enabledSurfaces: ReadonlySet<MarkdownSurface>) {
    this.enabledSurfaces = enabledSurfaces;
  }

  getExtensions(surface: MarkdownSurface): MarkdownExtensionSet {
    if (this.enabledSurfaces.has(surface)) {
      return { remarkPlugins: [remarkLinkArxivIds] };
    }
    return {};
  }
}

export const PHASE_41_DEFAULT_ENABLED_SURFACES: ReadonlySet<MarkdownSurface> =
  new Set(["rationale"]);
```

Constructor takes the set of enabled surfaces; non-enabled
surfaces return empty extension sets (= default behavior).

### Phase-41-blocking decisions deferred to per-unit implementation

- **D-10** (Unit 41.1): Pattern strictness — `\d{4}\.\d{4,5}` (4-5
  digit suffix; matches modern arXiv ID format) vs broader
  `\d+\.\d+` (would also match e.g. `arxiv:1.0` which is invalid
  but harmless — produces a 404). Lean = **strict `\d{4}\.\d{4,5}`**
  (the regex IS validation per Phase-38 APPEND-D-I XSS-safety
  contract; clearer rejection than silent 404).
- **D-11** (Unit 41.1): Case sensitivity — case-insensitive prefix
  match (`arxiv:` / `ArXiv:` / `ARXIV:` all match) vs case-
  sensitive (`arxiv:` only). Lean = **case-insensitive** (curator
  prose varies; matches common arXiv documentation style).
- **D-12** (Unit 41.1): Version-suffix preservation in display
  text — `arxiv:2024.01234v3` → display `arxiv:2024.01234v3` and
  href `.../abs/2024.01234v3` vs strip `v3` from both. Lean =
  **preserve verbatim** (curator may cite a specific version
  intentionally; arXiv URL with version suffix resolves to that
  version).
- **D-13** (Unit 41.2): Default-enabled-surfaces — `Set(["rationale"])`
  Phase-41 vs configurable via env-var (e.g.,
  `MARKDOWN_EXTENSIONS_ARXIV_SURFACES=rationale,reviewNotes`).
  Lean = **hardcoded Phase 41** (mirrors Phase 38 D-12 + Phase
  39 hardcoded-default precedent; surface-list-as-env-var
  deferred to Phase 42+).
