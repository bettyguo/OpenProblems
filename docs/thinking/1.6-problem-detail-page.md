# Unit 1.6 — Problem detail page `/problems/[slug]` — 10-block layout

> Stage 1 (THINK) artifact. Critical-path.

## Problem restatement

Render the canonical Problem URL. §9 fixes the 10 blocks in strict order — Header / Rating Radar / TL;DR / Background / Definition / Benchmarks / History / Recent rating actions / Related problems / Citation. Every block ships at v1 fidelity. SSG-prerendered per slug via `generateStaticParams`; 404 for unknown slugs via `notFound()`.

## Inputs

- §9 — block list + order.
- Unit 1.4 — `hallucination-reduction` content (one full seed; will exercise every block).
- Unit 1.5 — `RatingRadar` component + `lib/ratings/normalize.ts`.
- Velite collections `problems`, `problemPages`, `ratings` (Unit 1.4 velite extensions).
- `lib/mdx/mdx-content.tsx` (Unit 1.3) — renders MDX `body`.
- §10.2 "every chart is also a table" — closes Q35 with a "View as table" disclosure under the radar.
- §10.4 — problem detail page JS budget < 180 KB gz (excluding viz route chunks). RatingRadar is RSC-only, so the only client JS on this page is the `<details>` element (native, no JS) and the optional dark-mode toggle if we add it to the layout.

## Constraints

- **RSC-only by default.** No client components in Unit 1.6. Tooltip / accordion behaviour uses native `<details><summary>` so screen-readers and keyboard work out of the box.
- **`generateStaticParams`** returns every problem slug from Velite at build time. New problems become pages on the next build.
- **Unknown slug → `notFound()`** triggers the segment-local `not-found.tsx` (Q26) when present; for v1, accept the default Next 404.
- **Citation BibTeX uses the canonical site URL.** Phase 1 site URL is unresolved (Q2). Use `https://llm-openproblems.org/problems/<slug>` placeholder; pull from `NEXT_PUBLIC_SITE_URL` env var with a default.
- **Status pill colors map directly.** open → border, partially-solved → chart-3 (urgency amber-ish), converging → chart-4 (purple), solved → chart-2 (sage), retired → muted.
- **Rating-action sort** by `date` descending; show top 3; link to full `/problems/[slug]/ratings`.
- **Related problems** are slugs only; v1 shows them as bare links. The link is not asserted to resolve (cross-FK is Phase 2's cross-link audit job; missing related problems just render as a TODO marker).
- **Benchmarks table.** Columns: name, dataset, metric (direction marker), `upper_bound` if set, "view protocol" link if `protocol_url` set. No scores yet (Q34 entries.json is Phase 2).

## Alternatives considered

1. **Single page route renders all 10 blocks; helpers in `lib/content/load-problem.ts`** (chosen).
2. Split blocks into separate sub-components first; this unit just composes them. Slightly better for Storybook coverage, but premature componentisation — every block is bespoke to this page.
3. Defer Block 7 (history) viz; show only the MDX. Chosen for v1 — the §11 `TimelineRibbon` viz lands in Phase 2.

## Edge cases & failure modes

1. **Missing MDX page.** If `background.mdx` is absent for some problem, render a `<p>` placeholder with `TODO(curate)` framing. Same for `definition.mdx`, `history.mdx`.
2. **No rating actions yet.** Block 8 shows a placeholder; the page still renders.
3. **Slug not in taxonomy.** A Problem may reference an FK that's been removed; v1 shows the raw `domain` / `subdomain` ids without taxonomy lookup. Taxonomy resolution happens in Unit 1.7.
4. **Long benchmark list.** Vertical table with overflow-x: auto wrapper.
5. **Print stylesheet / BibTeX copyability.** v1 uses a `<pre>` block with the BibTeX text plus a hint to use the browser's native copy. Phase 2 may add a copy-button.
6. **MDX in citation block.** Citation is plain text — no MDX.
7. **`status` may be `"open"` (only one in Phase 1).** Pill component handles all five statuses but only one path is exercised by content.
8. **Q11 collision URLs.** `/domains/[domain]/[subdomain]` — the problem's `domain` and `subdomain` are scoped under the canonical pair. Block 1 links the breadcrumb as `/domains/deep-learning` and `/domains/deep-learning/large-language-models`.
