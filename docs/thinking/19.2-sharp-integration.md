## THINK

### What this unit ships

Two file edits + 1 dep install:

1. **`pnpm add sharp`** ‚Äî installs `sharp@0.34.5` as direct
   runtime dep. `sharp` was previously transitively available via
   `next/image` per the Phase-0 `pnpm-workspace.yaml` `allowBuilds.sharp:
   true` configuration; explicit direct dep makes the import
   stable (future `next` updates can't break the chain).

2. **`lib/storage/index.ts` (edit)** ‚Äî `putAvatar` extended with
   `sharp` pipeline per ADR-0019 D-C + D-D + D-B (in pipeline
   order):
   - **D-C input stage**: `Buffer.from(await file.arrayBuffer())`
     converts the `File` to a Node Buffer.
   - **D-D auto-rotation**: `sharp(inputBuffer).rotate()` ‚Äî reads
     EXIF Orientation tag (1-8), applies rotation/flip to pixel
     data, resets Orientation to 1. **iOS portrait shots render
     correctly across all clients including those that ignore
     EXIF Orientation.**
   - **D-B EXIF strip**: `.toBuffer()` re-encodes WITHOUT EXIF
     metadata (sharp's default behavior on `.toBuffer()` strips
     ALL EXIF tags). **No `.withMetadata()` call Phase 19** ‚Äî
     strip everything (conservative privacy default). Color
     profile + dimensions + encoding settings preserved by sharp
     internally.
   - **D-C upload stage**: `put(key, strippedBuffer, ...)` ‚Äî
     existing Vercel Blob `put()` call, but now receives the
     transcoded buffer instead of the raw `File`. Output format
     preserved (sharp keeps input format by default when no
     explicit `.jpeg()/.png()/.webp()` is called); `contentType:
     file.type` remains correct.

### Why mock `sharp` (not run real sharp + fixture)

Per ADR-0019 D-E backwards-compat note + Phase-19 scope-cap
discipline: unit tests verify the INTEGRATION shape (sharp is
invoked correctly + output flows to Vercel Blob). Real EXIF-strip
behavior is sharp's responsibility; verified by sharp's own test
suite + integration tests against real fixtures Phase 20+.

Risks of running real sharp in unit tests:
- **Native bindings**: vitest may not have sharp's libvips
  bindings ready in all CI environments (cross-arch CI; cold
  cache); test could fail on infrastructure quirks unrelated to
  our integration.
- **Test asset complexity**: checked-in JPEG fixture (~20 KB) +
  assertion harness for EXIF probe via `sharp.metadata()` ‚Äî adds
  fixture management burden.
- **Test runtime cost**: real sharp cold-start ~100 ms per test;
  vitest's parallel runner may serialize; suite slows.

Mocking covers the integration boundary; cost ‚âà 0; flake risk ‚âà 0.

### Backwards compatibility (per ADR-0019 D-E)

Phase 19 only strips on NEW uploads via `updateProfileImageAction`.
Existing Phase 16-18 avatars NOT retroactively processed.
Phase-20+ backfill script candidate (`scripts/backfill-exif-strip.ts`;
manual curator invocation; CLI dry-run flag; ~1-2 unit scope when
promoted).

### Phase 20+ inheritance contract (per ADR-0019 D-F)

The Phase-19 pipeline `sharp(buffer).rotate().toBuffer()`
establishes the integration point. Phase 20+ image-processing
surfaces insert their transforms between `.rotate()` and
`.toBuffer()`:

```ts
// Phase 20+ Q68 expansion content moderation:
sharp(buf).rotate()
  .pipe(moderationApi)
  .toBuffer()

// Phase 20+ cropping UI:
sharp(buf).rotate()
  .extract({ left, top, width, height })
  .toBuffer()

// Phase 20+ resizing + WebP transcoding:
sharp(buf).rotate()
  .resize(256, 256, { fit: "cover" })
  .webp({ quality: 85 })
  .toBuffer()
```

Multiple Phase-20+ transforms can stack. Audit boundary stays
explicit.
