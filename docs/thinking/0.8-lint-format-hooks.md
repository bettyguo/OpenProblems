# Unit 0.8 — ESLint, Prettier, Husky, lint-staged, commitlint

> Stage 1 (THINK) artifact.

## Problem restatement

Wire the tooling that enforces code style and commit discipline per MASTER_PROMPT §5.9 and §14.4. Once landed, every PR is gated on lint, format, conventional-commit message, and the ADR-0005 rating-action-immutability hook (per ADR-0005 §enforcement, which explicitly names Unit 0.8 as the hook's home).

## Inputs

- ADR-0005 (rating-action immutability — pre-commit hook lives here).
- ADR-0001 (Next.js 15 + React 19 — eslint-config-next applies).
- §5.9 (TS strict + ESLint with `@typescript-eslint`, `eslint-plugin-react`, `eslint-plugin-tailwindcss`, `eslint-plugin-jsx-a11y`; Prettier; Husky + lint-staged).
- §14.4 (Conventional Commits via commitlint).
- §14.1 (no barrel files — can be enforced via an ESLint rule on `index.ts` files; deferred unless feasible cheaply).

## Constraints

- **ESLint 9 flat config** (`eslint.config.mjs`). Legacy `.eslintrc` is deprecated.
- **`eslint-config-next` for Next 15** carries `react`, `react-hooks`, and Next's own rules.
- **Tailwind v4 plugin caveat.** `eslint-plugin-tailwindcss` historically lagged Tailwind major bumps; v4 support has shipped in late 2025 but the plugin may emit false positives on v4 idioms (`@theme inline`, OKLCH literals). Install it; if it produces noise on our files, downgrade specific rules to warn or disable narrowly with `// eslint-disable-next-line` and a reason comment per §14.1 (no untagged disables).
- **Prettier as the formatter.** ESLint does not format; that's Prettier's job. Use `eslint-config-prettier` to disable any ESLint rules that conflict with Prettier output.
- **Husky v9.** `pnpm dlx husky init` lays down `.husky/_/` infrastructure; we hand-author `pre-commit` and `commit-msg` hooks.
- **lint-staged for incremental hygiene.** Only changed files are linted/formatted on commit.
- **commitlint with `@commitlint/config-conventional`.** Standard Angular convention; aligns with the commit messages already in our git history (`chore(phase-0): unit X.Y — ...`).
- **ADR-0005 rating-immutability hook.** Pre-commit bash that scans `git diff --cached --name-only --diff-filter=MDR` for `content/problems/<slug>/ratings/*.yaml` paths and blocks the commit if any are present. (The CI gate that catches `--no-verify` bypass lives in Unit 0.11.)
- **No `--no-verify` instructions in docs.** §14.4's PR template will not say "you can skip hooks with `--no-verify`."

## Alternatives considered

1. **ESLint 9 flat config + Prettier + Husky 9 + lint-staged 16 + commitlint 19** (chosen — current best-of-breed in 2026).
2. **Biome** as the single tool (lint + format + import sort). Rejected: lacks Next.js / React 19 specialty rules; the ecosystem coverage of `eslint-config-next` and `eslint-plugin-jsx-a11y` is still ESLint-only.
3. **`oxlint`** for speed. Same gap as Biome on Next-specific rules and a11y depth.
4. **Skip `eslint-plugin-tailwindcss`** entirely until v4 support solidifies. Counter-argument: §5.9 names it explicitly. Try installing; if it noisy-fails on our v4 files, narrow rules rather than skip entirely.

## Edge cases & failure modes

1. **ESLint flag conflicts with `next.config.ts` `eslint.ignoreDuringBuilds: true`.** That setting only suppresses Next's _inline_ lint pass during `next build`. Our `pnpm lint` script invokes ESLint directly and is the canonical gate. Add a CI step in Unit 0.11 separate from `pnpm build`.
2. **Husky v9 hook path on Windows.** Husky writes shell scripts; Git Bash (or Git's bundled `bash.exe`) runs them on Windows. Confirmed working in earlier units (the `git commit -m "$(cat <<'EOF' ... EOF)"` heredoc pattern we used relies on the same bash). No special handling needed.
3. **lint-staged + Prettier + ESLint --fix race.** Run ESLint --fix _before_ Prettier so the formatter has the final say on whitespace. lint-staged supports chained commands per glob.
4. **`commitlint` failing on a previously-merged commit message** (e.g., Unit 0.0's "Unit 0.0 summary"). `commitlint` only checks the _current_ commit-msg, not history. Fine.
5. **ADR-0005 hook bypass risk.** A contributor running `git commit --no-verify` skips the hook. The CI gate in Unit 0.11 is the backstop. Document both in `docs/contributing.md` (deferred but eventual home).
6. **MDX / YAML formatting by Prettier.** Prettier formats MDX and YAML; ensure `.prettierignore` excludes `pnpm-lock.yaml` (auto-generated), `.next/`, `node_modules/`.
7. **`no-console` rule.** Disabled in `scripts/` (CLI needs to print). Enabled elsewhere.
8. **`@typescript-eslint/no-unused-vars`.** Configure to ignore vars prefixed with `_` (our test destructuring `const { verified: _v, ...rest } = VALID` pattern).
9. **Test files.** Loosen `no-explicit-any` and a few others in `*.test.ts` so fixtures can be authored ergonomically.
10. **Next 15 + ESLint 9 + flat config.** `eslint-config-next` has shipped a flat-config variant (`eslint-config-next/flat` or via the `nextPlugin` import). Use whichever the current Next 15 docs prescribe — pin the version and ride the upstream API.

## Decision

Files in this unit:

1. `eslint.config.mjs` — flat config: TypeScript-ESLint, eslint-config-next, react, jsx-a11y, tailwindcss. Test-file overrides loosen a couple rules. `scripts/` allows `no-console`.

6. `.husky/pre-commit` — runs `lint-staged` and the ADR-0005 rating-immutability check.
7. `.husky/commit-msg` — runs `commitlint --edit "$1"`.
8. `package.json` scripts: `lint`, `lint:fix`, `format`, `format:check`, `prepare` (for husky install).
