### D-3. Prior-action auto-fill scope

| Concern | Phase 23 scope | Notes |
|---|---|---|
| **Source** | `content/problems/<problemSlug>/ratings/*.yaml` filesystem read. | NOT Velite `#site/content` (couples to build step; script doesn't run velite first). Direct `fs.readdir` + per-file `fs.readFile` + `parse(yaml)`. |

| **Copy 4 OTHER dimensions** | When `priorAction` provided: for the 4 dimensions that are NOT the PROPOSED dimension, copy `grade` / `value` / `stars` + `confidence` + `rationale` verbatim from prior. | Curator can still edit; placeholder REVIEW header comment annotates "values copied from prior — adjust if signal demands." |
| **`prior_action` field** | `<problemSlug>/<priorFilename-without-.yaml>`. | Matches existing rating-actions' format (per `2026-12-15-q4-revision.yaml` example: `prior_action: compute-optimal-test-time-reasoning/2026-09-01-q3-revision`). |

| **`watchlist`** | Copy from prior. | Inherits prior watchlist state. |
| **Schema-mismatch fallback** | If prior parses but is missing fields (legacy data), fall back to Phase-22 placeholder behavior for the missing fields. | Defense-in-depth. |

### D-4. `readPriorRatingAction` API

New pure-ish async helper in `lib/rating-challenges/scaffold.ts`:

```ts
export interface PriorActionInfo {
  /** Filename without .yaml extension (e.g., "2026-12-15-q4-revision"). */
  filename: string;
  /** Schema-validated rating-action payload. */
  action: RatingAction;
}

export async function readPriorRatingAction(opts: {
  contentRoot: string;        // e.g., "content/problems" or test fixture root
  problemSlug: string;
}): Promise<PriorActionInfo | null>;
```

- Pure-ish: takes `contentRoot` as parameter (testable via fixture
  dir).
- Returns `null` when no prior actions exist OR all malformed.
- Uses `fs/promises.readdir` + `readFile`.
- Parses via `RatingActionSchema.parse()`.

### D-5. `buildRatingActionYaml` extension

Existing `ScaffoldInput` gets a new optional field:

```ts
export interface ScaffoldInput {
  // ... existing fields
  priorAction?: PriorActionInfo | null;
}
```

When `priorAction` provided:
- 4 OTHER dimensions filled from `priorAction.action.dimensions`.
- `prior_action` field set to `<problemSlug>/<priorAction.filename>`.
- `signals_considered` starter set from prior + placeholder TODO.
- `watchlist` from prior.
- Header comment block updated to note "OTHER 4 dimensions
  COPIED FROM PRIOR — adjust if signal demands" (different
  wording vs Phase-22's "PLACEHOLDERS — copy from prior").

When `priorAction` is null/undefined:
- Phase-22 behavior preserved verbatim (placeholders + TODO
  rationale on OTHER 4 dimensions).
