### Q45 framing

Next.js 15 App Router supports dynamic-segment folders (`[slug]/`) and concatenated dynamic-plus-literal segments (`[slug].extension/`). The dotted-suffix convention turned out fragile in practice:

- **Windows path quoting** + git's handling of `.` in directory names is inconsistent across Bash, PowerShell, and git-bash workflows.
- **`[domain].xml`** as a folder name technically works in Next.js + NTFS, but the tooling overhead (test fixtures, CI path handling, pnpm scripts) made it unattractive vs. the plain `[domain]/` shape.
- **`content-type` headers** + RSS-reader auto-discovery via `<link rel="alternate">` (Unit 5.9) compensate for the URL-side loss of the `.xml` suffix.

**Decision**: future dynamic API routes use the plain `[<param>]/route.ts` convention. Concatenated dotted-suffix routes are forbidden by convention (no ADR; this is a routing-style choice, not a load-bearing architectural decision).

### Tradeoffs / defensible defaults flagged

1. **Q38's refinement is interpretive, not new framing**: the original lean targeted multi-select-dimming-with-URL-state; Phase 4 shipped a thinner chip-as-link. Phase 6+ may revisit; the lean still applies if/when full filtering ships.
2. **Q42 promotion to `decided-as-lean`**: the "no default cap" position has been tested by 2 LLM CLIs (Units 5.3, 5.5) without user-facing cost surprises. The re-eval gate is "100 real ingest runs" — soft, observational, not enforced.
3. **Q45 as a "decided" without an ADR**: routing-style choice. If a future contributor wonders why dotted-suffix routes aren't used, the OPEN_QUESTIONS entry is the canonical doc.
