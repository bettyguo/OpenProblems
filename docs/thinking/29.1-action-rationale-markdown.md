### Wikilink rendering deferred (Class B.14; not promoted to Q)

Existing rating-action YAML content contains `[[problem-slug]]`
wikilink syntax — e.g., `hallucination-reduction/2026-05-14-initial.yaml`
value rationale references `[[scalable-oversight]]` +
`[[long-context-rag]]`. Markdown promotion does **NOT** resolve
wikilinks because:

- `remark-parse` lacks native wikilink support.
- `remark-gfm` does not add wikilink extension.
- ADR-0018 D-G tag allow-list does not include any custom
  wikilink element.
- No `remark-wiki-link` plugin in the pipeline.

**No regression**: literal-text display is preserved Phase 28 →
Phase 29 (`whitespace-pre-line` renderer pre-29 rendered them as
literal text; markdown renderer post-29 also renders them as
literal text). One test (`preserves wikilink syntax as literal
text`) documents this behavior as a regression guard for
Phase-30+ wikilink-resolution work.

**Active wikilink resolution is Phase-30+ candidate** (Phase-29
Class **B.14** hygiene item to be surfaced Unit 29.2). Promotion
gated on: (1) curator demand signal; (2) cross-surface wikilink
consistency decision (challenge rationale + review notes + bio
could also use wikilinks — needs scope decision first); (3)
ADR-0018 D-G amendment shape or new ADR if semantics evolve
beyond simple slug-to-href mapping.

### Heading-demotion collision risk accepted

Per Phase-29 prep D-12: `<h3>` demotion clamp coexists with the
existing page-level `<h3>` "Signals considered" heading. ADR-0018
D-C semantics permit same-level coexistence; the clamp behavior
(>= h4 source → h6 output) is the actual failure mode, not
co-existence with peers. Curator authoring guidance (informal):
prefer `<h2>`-or-deeper headings in rationales (which become
`<h4>+` after demotion). No current rating-action rationale
content uses headings — all 14 rating-action YAMLs ship
plain-prose rationales — so the risk is theoretical at Phase-29
close.
