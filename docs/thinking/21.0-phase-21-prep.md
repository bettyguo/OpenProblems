### D-3. Cleanup scope

| Cleanup concern | Phase 21 scope | Notes |
|---|---|---|
| **Affected blobs** | Vercel Blob entries with `pathname` matching `avatars/<userId>-<timestamp>(-backfill)?.<ext>` shape AND whose URL is NOT referenced by any `users.imageOverride` row. | **DB is source of truth**; storage is the side-effect. Orphan = blob in store NOT referenced in DB. |
| **Enumeration phase** | `list({ prefix: "avatars/", limit: 1000, cursor })` paginated until `hasMore === false`. | Vercel Blob `list()` API v2; pagination handled in a while-loop. |
| **DB-snapshot phase** | `SELECT id, imageOverride FROM user WHERE imageOverride IS NOT NULL` → build `Set<string>` of in-use URLs (one query; cached for the duration of the run). | Snapshot-at-start posture; if a user uploads during the run, their NEW blob may show as orphan. **Mitigation**: re-run after user-upload windows; `--dry-run` curator inspection catches false positives. |
| **Orphan-detection phase** | For each listed blob: if `blob.url` is NOT in the in-use set → mark as orphan. | URL string equality (Vercel Blob URLs are deterministic given pathname). |
| **Delete phase** | `del(<orphan-url>)`; idempotent on already-deleted. | One `del()` call per orphan; per-blob try/catch (failed blob doesn't kill batch). |
| **Dry-run** | `--dry-run` CLI flag: enumerate + snapshot + detect + log orphan list WITHOUT executing `del()`. Curator inspects log first. | Mirrors Phase-20 backfill script pattern verbatim. |
| **Pathname pattern** | Cleanup limits its scope to `avatars/<userId>-<timestamp>.<ext>` + `avatars/<userId>-<timestamp>-backfill.<ext>` shapes (per ADR-0017 D-A + Phase-20 D-10 conventions). | Defends against future non-avatar Vercel Blob usage (e.g., Phase-22+ uploaded content); cleanup script is avatar-specific. |
| **Idempotency** | Safe to re-run; orphan-deleted blobs are no-op'd by Vercel Blob's `del()` (already-deleted returns successfully). | Operator-level safety: log-and-skip pattern. |
| **Error handling** | Per-blob try/catch; failed `del()` logged + counted in summary; non-zero exit if any failed. | One bad blob doesn't kill the batch. |
| **Concurrency** | Phase 21 ships serial-blob processing (for loop). | Phase 22+ if curator demands parallelism. |
| **Pagination concurrency** | Phase 21 ships serial pagination (await `list()` → process → await next page). | Phase 22+ for streaming pattern if blob count grows. |

### D-6. Backwards compatibility + safety

- **DB-snapshot-at-start** narrows blast radius — orphan
  detection uses a single SELECT at run start; the snapshot is
  immutable for the duration of the run. Mitigation against
  concurrent-write race: if a user uploads during the run,
  their NEW blob may match the orphan filter (uploaded AFTER
  snapshot; not yet seen in DB-snapshot Set). **`--dry-run`
  inspection catches false positives**; operator schedules runs
  during low-traffic windows.
- **Pathname-pattern filter** narrows to `avatars/*` only.
  Skips any non-avatar Vercel Blob entries (theoretical Phase-22+
  uploaded content; defends against future variations).
- **Per-blob try/catch**. Failed `del()` calls logged + counted;
  batch continues.
- **`--dry-run` mandatory first run**. Script header comment
  documents this; mirrors Phase-20 D-6 stance. Phase-22+ session-
  level sentinel refinement deferred.
