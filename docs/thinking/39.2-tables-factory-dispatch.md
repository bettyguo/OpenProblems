## THINK

### Problem restatement

Unit 39.1 shipped the tables registry + schema override + 10
plugin-level tests. The registry is not yet wired into the
factory dispatch. Unit 39.2 adds:

- `"tables"` case arm to the `MARKDOWN_EXTENSIONS` switch in
  `getExtensionRegistry()`.
- Updated JSDoc + error message listing all 3 recognized
  values.
- Env-var-aware dispatch tests in `extensions/index.test.ts`.
- End-to-end reviewNotes tables rendering tests in
  `lib/markdown/index.test.ts` exercising the FULL `unified`
  pipeline.

### Inputs

- HEAD `e7b55d0` (Unit 39.1 ship; tables module + 10 tests).
- `lib/markdown/extensions/index.ts` — Phase-38 factory with 2
  dispatch arms (default + wikilinks); throw-on-unknown.
- `lib/markdown/extensions/tables.ts` — `TablesExtensionRegistry`
  + `PHASE_39_DEFAULT_ENABLED_SURFACES`.

### Constraints

- **Backward compat**: existing 794 tests pass verbatim.
- **Test pollution discipline**: env-var snapshot + restore
  pattern mirrors Phase-38 wikilinks tests.
- **Mutually exclusive Phase 39**: `wikilinks` OR `tables`
  OR default; no composition.
- **Throw-on-unknown error message** updated to list 3
  recognized values + Phase-40+ deferral cross-ref.

### Alternatives considered

**Option 1: Mirror Phase-38 wikilinks dispatch shape (chosen)**
— add one `case "tables":` arm; update error message.
Single-arm-per-extension; matches existing pattern.

**Option 2: Lift dispatch into a registry-table map
(rejected)** — `const REGISTRIES = { wikilinks: () => ..., tables: () => ... }`.
**Why rejected**: premature abstraction; only 2 non-default
arms; Phase-40+ may introduce multi-value composition which
would naturally lift the dispatch then. Phase 39 stays
explicit switch.

### Edge cases & failure modes

1. **Unknown value error message must include `tables`**:
   tested.
2. **Wikilinks + tables both set**: Phase 39 takes only one
   value; no comma-parsing. `MARKDOWN_EXTENSIONS=wikilinks,tables`
   hits the `default` throw arm (not a recognized value).
3. **Mutual exclusivity per surface**: a curator switching
   `MARKDOWN_EXTENSIONS=wikilinks` → `tables` loses wikilinks
   on actionRationale and gains tables on reviewNotes. Phase
   40+ multi-value composition would enable both
   simultaneously.
