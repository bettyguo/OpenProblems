### D-3. Item F — markdown rationale scope

**Schema** (`lib/markdown/sanitize-schema.ts`):
- New `rationaleSchema: Schema` export as sibling to
  `bioSchema` + `reviewNotesSchema`.
- **Identical to `bioSchema` Phase-27** per ADR-0018 D-G
  scope-cap discipline (same precedent as Phase-18's
  `reviewNotesSchema = { ...baseSchemaConfig }`):
  audit-boundary consistency at minimum cost.
- Phase-28+ Q72-style divergence candidates: tables / footnotes
  if curator + user demand surfaces.

**Helper** (`lib/markdown/index.ts`):
- New `renderRationaleMarkdown(text: string): string` —
  rationale is `NOT NULL` per Phase-11 schema (50-2000 chars
  required; no nullable case). Returns sanitized HTML string.
- New `rationaleProcessor` singleton (identical pipeline shape
  to `bioProcessor` + `reviewNotesProcessor`; uses
  `rationaleSchema`).
- **Third call site for the ADR-0018 D-G inheritance pattern**.

**Detail page** (`app/[locale]/u/[handle]/challenges/[id]/page.tsx`):
- Replace `<p className="...whitespace-pre-wrap">{challenge.rationale}</p>`
  with `<div className="...prose styling...prose styling..." dangerouslySetInnerHTML={{ __html: renderRationaleMarkdown(challenge.rationale) }} />`.
- Reuse exact 14 arbitrary-variant prose styling utility
  classes Phase-18 established for reviewNotes.

**Listing pages** — switch from source-truncation to CSS
line-clamp per Phase-18 precedent:
- `/u/[handle]/challenges/page.tsx`: replace
  `truncateRationale(row.rationale)` with markdown render +
  `line-clamp-3` CSS class.
- `/profile/page.tsx`: replace `truncateRationale(challenge.rationale)`
  (Phase-12 source-truncation; same incompatibility with
  markdown as Phase-18 noted for reviewNotes) with markdown
  render + `line-clamp-3`.
- `/problems/[slug]/challenges/page.tsx`: same.
