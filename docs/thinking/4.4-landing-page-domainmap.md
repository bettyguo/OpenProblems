## THINK

### Inputs

- **Unit 4.3 (`e3c2623`)** shipped `lib/content/build-domain-map.ts` (the shared loader) and the brushable `/domains` DomainMap. The loader is consumer-ready for `/` as well.
- **Unit 4.0 (per-unit description for 4.4)**: "replace the existing tile-grid hero with a DomainMap teaser + filter chips that **link into /domains/[domain]**."
- **Unit 4.0 D-6 interaction #4** describes full multi-select dimming chips with URL search-param deep-link state. **Two readings of D-6 apply to two different pages**: 4.4 lands the simple "chip-as-navigation" interpretation per the per-unit description; the full multi-select dimming pattern is a Phase-5 enhancement (and may target `/domains` rather than `/` — landing is funnel surface, not filter surface).
- **`app/page.tsx` today** has 4 sections: hero, "Recently rated", "By domain" (tile-grid), "Methodology". Phase 4's update is scoped to the "By domain" section.
- **Q38 lean** ("filter chips URL persistence") was decided-as-lean for URL search params — that lean stands when full filtering ships; it doesn't apply to navigation chips which use `<Link href>` natively.

### Scope (this unit)

Two changes to `app/page.tsx`:

1. **Replace `<DomainTileGrid />`** in the "By domain" section with `<DomainMap />` + `<ChartTableSwitch>` + `<DomainMapTable />`. Mirror Unit 4.3's composition exactly.
2. **Add a chip row** above the DomainMap: 5 `<Link>` buttons, one per domain, each linking to `/domains/<id>`. Pure server-rendered navigation; no client state.

### Scope explicitly deferred

- **Multi-select dimming + URL-search-param state** (full Unit 4.0 D-6 interaction #4 / Q38 lean). Phase-5 enhancement when the brushable-filter pattern has clear user-research demand. Today's "chip-as-link" satisfies the unit description.
- **Hero-section changes**. Unit 4.0 had a passing reference to "replace the existing tile-grid hero" — the actual landing page doesn't have a tile-grid HERO (the tile grid is in the "By domain" section, not the hero). Hero stays untouched.

### Constraints

- **Server component, no `"use client"`.** Same as Unit 4.3.
- **Mirror Unit 4.3's ChartTableSwitch composition** for consistency.
- **Chips link to existing `/domains/<id>` routes.** Those have been live since Phase 1 (`app/domains/[domain]/page.tsx`).
- **No new code paths in `buildDomainMap()`.** The loader is unchanged; `/` consumes the same `{ nodes, links }` shape as `/domains`.

### Chip row design

5 chips, one per domain, in `taxonomy.domains[]` order:

```tsx
<nav aria-label="Browse by domain" className="mb-4 flex flex-wrap gap-2">
  {taxonomy.domains.map((d) => (
    <Link
      key={d.id}
      href={`/domains/${d.id}`}
      className="border-border hover:border-accent/60 hover:bg-muted/40 rounded-full border px-3 py-1 text-xs font-medium ..."
    >
      {d.title}
    </Link>
  ))}
</nav>
```

Chip styling: rounded-pill (rounded-full), border + transparent background, accent-color hover. Matches the existing design-token-driven button styles.

### Alternatives considered

- **Implement full multi-select dimming with URL search-params now.** Rejected — Q38 lean was conditional on the brushable-filter pattern being load-bearing. Landing-page chips are navigation, not filter. Defer; ship the simpler shape.
- **Skip chips entirely; just the DomainMap.** Rejected — chips are explicitly in the per-unit description, and they add navigation affordance for users who want to drill into a single domain without inspecting the map.
- **Use a different node sample on `/` vs. `/domains`** (e.g., only top-rated problems on `/` for hero-density). Rejected — extra loader complexity for a marginal visual difference. Same data, different framing.
- **Different DomainMap dimensions on `/` (smaller viz)**. Considered. Rejected — DomainMap's `viewBox` is fixed 600×420; the CSS-side responsive sizing handles smaller containers automatically. No bespoke "compact mode" needed for Phase 4.

### Tradeoffs / defensible defaults flagged

1. **Chips link out, don't filter in-place.** Phase-5 may want both behaviors (e.g., "click chip" filters, "click chip with cmd" navigates). The simpler shape suffices for Phase 4's landing-as-funnel framing.
2. **Same DomainMap viz on `/` and `/domains`** (Unit 4.0 D-12 lean). Identical viz + different page framing keeps maintenance cost low.
3. **No "Recently rated" or hero changes.** Out of scope; this unit's only target is the "By domain" section.
4. **DomainMap renders even on first paint of `/`.** Some landing-page traffic may never scroll to the map; ~30 SVG nodes is a few hundred bytes in the HTML payload. Negligible cost; not optimized away.
