**Why bioRxiv as the 7th consumer (vs OSF or other preprint
sources)**:

- bioRxiv is the most widely-cited preprint server in
  biological sciences (Cold Spring Harbor Laboratory
  initiative; >300k preprints since 2013). High curator demand
  signal.
- The bioRxiv ID format `YYYY.MM.DD.NNNNNN` is distinctive (8-
  character date prefix + 6-digit submission number) —
  pairwise-disjoint from arxiv (`\d{4}\.\d{4,5}` modern or
  category-prefixed legacy) + doi (`10.<registrant>/<suffix>`)
  + pubmed (`(?:pubmed|pmid):\d+`) + orcid (`\d{4}-\d{4}-\d{4}-\d{3}[\dX]`)
  via distinct literal prefix `biorxiv:`. **Regex-
  disjointness via prefix discriminator alone**, mirroring the
  established 4-consumer discipline.

**DOI overlap concern resolution**: bioRxiv preprints have
DOIs like `10.1101/2024.01.15.575678` (where `10.1101` is
bioRxiv's DOI registrant prefix). However, the curator
references bioRxiv preprints via `biorxiv:2024.01.15.575678`
(literal `biorxiv:` prefix + bare year.month.day.number ID
WITHOUT the `10.1101/` DOI-namespace prefix). The doi
consumer's regex requires a literal `doi:` prefix to match,
so curators writing `doi:10.1101/2024.01.15.575678` get the
generic doi.org resolver URL (which redirects to bioRxiv);
curators writing `biorxiv:2024.01.15.575678` get the direct
biorxiv.org URL form. **No regex collision**; curators choose
the citation form based on URL preference.

D-B. **Pattern strictness rationale**:
- `\d{4}\.\d{2}\.\d{2}\.\d{6}` matches the modern bioRxiv ID
  format `YYYY.MM.DD.NNNNNN` introduced ~2019 (e.g.,
  `2024.01.15.575678`).
- Optional `(v\d+)?` version suffix (e.g., `v1`, `v2`) —
  mirrors Phase-41 arxiv version suffix shape.
- Word-boundary anchors `\b...\b` prevent mid-word matches
  (mirrors arxiv/orcid).
- Case-insensitive `/i` flag for prefix matching (mirrors all
  prior consumers).
- Older numeric-only IDs (`001234` legacy format from pre-2019
  bioRxiv) — Phase 59+ deferral if curator demand surfaces (no
  precedent for legacy IDs in the project's existing paper-
  evidence URLs).

D-C. **URL emission**: `https://www.biorxiv.org/content/10.1101/${id}${version ?? ""}`.
- The `10.1101/` prefix is bioRxiv's stable DOI namespace
  (synthesized server-side from the curator's bare ID).
- Version suffix appended verbatim (`v1`, `v2`, etc.) when
  present.
- Canonical URL form per bioRxiv conventions.
