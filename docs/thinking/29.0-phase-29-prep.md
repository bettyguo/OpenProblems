### D-6. Wikilink rendering deferral (new Q-candidate flagged; NOT promoted Phase 29)

Existing rating-action YAML content already contains wikilink
syntax — e.g., `content/problems/hallucination-reduction/ratings/2026-05-14-initial.yaml`
value rationale contains `[[scalable-oversight]]` and
`[[long-context-rag]]`. The current `whitespace-pre-line` renderer
displays these as literal text. **After markdown promotion these
will still render as literal text** because:

- `remark-parse` does not natively support wikilink syntax.
- ADR-0018 D-G's tag allow-list does not include any custom
  wikilink element.
- No `remark-wiki-link` plugin in the pipeline.

**No regression**: wikilink display fidelity is unchanged Phase 28
→ Phase 29 (literal text both before and after). **Active wikilink
resolution** (rendering `[[slug]]` → `<a href="/problems/slug">slug</a>`)
is a **Phase 30+ candidate**; would require either: (a) a
`remark-wiki-link` plugin + ADR-0018 D-G amendment to allow the
emitted anchor tag with the resolved href + slug-resolution
context plumbed through the processor pipeline; (b) a new ADR if
the resolution semantics evolve beyond simple slug-to-href
mapping.

**Q-candidate (NOT promoted)**: surface as Phase-29 hygiene B.14
item ("rating-action rationale wikilink rendering") for Phase 30+
ADR consideration. Promotion gated on: (1) curator demand signal;
(2) cross-surface wikilink consistency (challenge rationale +
review notes + bio could also use wikilinks — needs scope decision
first); (3) ADR-0018 D-G amendment shape.
