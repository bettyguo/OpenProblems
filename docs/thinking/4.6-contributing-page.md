# 4.6 — /contributing page composition + Velite collection

## THINK

### Inputs

- **Unit 4.5 (`1c83f61`)** shipped `content/contributing/v1.mdx` with `methodology`-compatible frontmatter (`version` / `title` / `summary` / `date`).
- **Unit 4.0 D-9**: `/contributing` page mirrors `/methodology` MDX rendering pattern.
- **`velite.config.ts` (L55-73)** has the `methodology` collection precedent — schema with `version` / `title` / `summary` / `date` / optional `supersedes` / auto-derived `slug` / `body: s.mdx()`, plus a `.transform()` stripping the `methodology/` prefix.
- **`app/methodology/page.tsx`** + **`app/methodology/[version]/page.tsx`** are the route shape — latest renders at `/methodology`, versioned snapshots at `/methodology/v{version}`.
- **Velite Zod-3-internals + Q31** still hold — collection schemas use Velite's `s` factory, not `lib/schemas/*` Zod-4.

### Scope (this unit)

Three changes that together replace the `/contributing` stub with the live MDX-rendered page:

1. **`velite.config.ts`** — add a `contributing` collection mirroring `methodology` 1:1.
2. **`app/contributing/page.tsx`** — replace the 10-line `<StubPage>` with the latest-version MDX renderer, near-copy of `app/methodology/page.tsx`.
3. **`app/contributing/[version]/page.tsx`** — new dynamic-slug route, near-copy of `app/methodology/[version]/page.tsx`.

### Constraints

- **Don't refactor shared methodology/contributing code** into a common helper in this unit. Two near-copies is the minimum-friction shape; a third versioned-MDX surface would justify extraction. Per the §14 "no premature abstraction" rule.
- **`exactOptionalPropertyTypes`** in tsconfig — careful with optional `supersedes` in the schema (Velite's `s.string().optional()` handles this correctly; cross-checked against the methodology precedent).
- **Velite collection name**: `Contributing` (TitleCase per Velite convention; mirrors `Methodology`).
- **Slug normalization**: strip `contributing/` prefix from the path slug, same as methodology strips `methodology/`.
- **SSG routes added**: `/contributing` (static, latest version) + `/contributing/v1.0.0` (SSG, the v1 snapshot). Total route count goes 198 → 200.

### Alternatives considered

- **Skip the `[version]` route for now** since only v1 exists. Rejected — mirroring methodology exactly keeps the contributor pact's audit trail aligned (every committed contributing-version stays browsable); a future v2 wouldn't need an additional Phase-5 unit to wire that up.
- **Extract a shared `<VersionedMdxPage>` component** taking the collection + slug-base as props. Rejected — premature abstraction; defer until ≥ 3 such pages exist.
- **Put the Velite collection in Unit 4.5**. Rejected — split is already shipped; this is the natural pair to it.

### Tradeoffs / defensible defaults flagged

1. **Two near-copies of `methodology` pages.** ~70 lines of code duplication. The refactor target is `<VersionedMdxPage>` when a third versioned doc lands; meanwhile, the duplication is reviewable.
2. **`v{version}` URL convention** (e.g. `/contributing/v1.0.0`) — matches methodology. The `v` prefix is parseable; the inner format `1.0.0` is SemVer-clean.
3. **No deletion of the original stub**. `app/contributing/page.tsx` is overwritten in-place; git history preserves the stub via the diff.
4. **No `<StubPage>` usage anymore on `/contributing`.** If the StubPage component becomes unused elsewhere (it's also used on other planned Phase-4 routes that aren't shipped yet), keeping it is fine — it's still on `app/contributing/page.tsx`'s import path being removed, not the file itself.
