# 6.7 — `lighthouserc.json` enrolment for `/problems/[slug]/talk`

Sixth code unit of Phase 6. Tiny config-only unit: adds one URL to the LHCI cohort so the talk-page route gets the canonical a11y / perf / SEO / best-practices gate on every PR. Mirrors the Phase-5 Unit 5.13 precedent (added `/digest` as the 13th URL).

## THINK

### Inputs

- **Unit 6.3 (`f0dbff1`)** shipped `app/problems/[slug]/talk/page.tsx` (10 SSG routes).
- **Unit 6.4 (`f9a465e`)** wired the Giscus iframe into the slot; iframe loads lazily AFTER page shell renders (ADR-0010 D-A), so Lighthouse a11y scores the SSG shell — not the iframe contents.
- **Unit 6.0 D-6** + **Unit 6.3 THINK** anticipated this enrolment: "Unit 6.7 will enrol `/problems/<slug>/talk` for one representative problem."
- **`lighthouserc.json` at HEAD** carries 13 URLs (Phase-5 5.13 added `/digest`). Asserts a11y / perf / SEO ≥ 95 as errors; best-practices ≥ 95 as warn.
- **§14.2 testing**: every page route gets a Lighthouse check. One representative URL per route-template is sufficient — the 10 talk pages share the same template + props shape.

### Scope (this unit)

Two file changes:

1. **`lighthouserc.json`** — add `http://localhost:3000/problems/hallucination-reduction/talk` (the same problem the existing cohort uses for `/problems/<slug>` + `/ratings` + `/history`). Brings URL count from 13 → 14.
2. **THINK doc** (this file).

### Scope NOT in this unit

- **Enrolling all 10 talk pages** — overkill. One representative URL per route-template is the project convention.
- **Real CI runs of LHCI against the new URL** — happens on the first PR that contains this commit; out-of-band for this unit.
- **Lighthouse audit of the iframe contents** — the iframe is cross-origin (Giscus); Lighthouse doesn't traverse it. The page shell (header / breadcrumb / "Discussion" h1 / no-script fallback / back-link) is what's scored.

### Constraints

- **§5.5 perf budget** — talk page's First Load JS = 108 kB (vs. 103 kB shared chunk; +2 kB delta is the Giscus wrapper from Unit 6.4). Should still score perf ≥ 95 on a desktop preset — the page is mostly text, has one above-the-fold heading, no images.
- **A11y ≥ 95** — page structure: `<nav aria-label="Breadcrumb">`, `<h1>`, `<section aria-label="Discussion thread">` containing the embed, `<noscript>` with descriptive content, back-link. No interactive widgets pre-mount on the SSG shell. Should comfortably hit ≥ 95.
- **SEO ≥ 95** — `<title>` from root layout (TBD if route-specific title is needed; current inherited title is acceptable for v1; can iterate post-LHCI results). Page has h1 + descriptive content.
- **Best-practices warn-only ≥ 95** — typically the failure mode is third-party `<iframe>` security headers. The Giscus iframe is loaded lazily AFTER page-load; pre-iframe-load score should be clean.

### Alternatives considered

- **Skip this unit entirely; rely on per-route Playwright smoke for a11y**. Rejected — Lighthouse a11y is the project's canonical contract per Unit 1.12 (Q27 close) + §14.2; bypassing for talk pages would create asymmetric coverage.
- **Enrol all 10 talk pages**. Rejected — overkill; LHCI CI cost = 3 runs × 14 URLs = 42 audits per PR. Adding 9 more URLs would push that to 69. One representative URL is the convention.
- **Different problem (not `hallucination-reduction`)**. Considered. Sticking with the same problem the rest of the cohort uses keeps the LHCI traces directly comparable when surfaced (the diff between `/problems/hallucination-reduction` and `/problems/hallucination-reduction/talk` is the Phase-6 change, not the problem identity).

### Tradeoffs / defensible defaults flagged

1. **Lighthouse runs against the lazy-loaded iframe**: the iframe loads AFTER Lighthouse's primary metrics collection (FCP / LCP / TTI). The score reflects the shell. Acceptable + matches ADR-0010 D-A's bundle-isolation framing.
2. **No URL-removal for any other route**: 13 → 14 URLs adds ~7% to LHCI CI runtime. Within budget; the gate is asserting correctness, not throughput.
3. **The talk page has no above-the-fold third-party content** when scored pre-iframe-load. Lighthouse's "best-practices" check for `iframe sandbox` could flag post-mount, but mount is post-Lighthouse-window.
