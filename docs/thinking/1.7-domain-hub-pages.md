# Unit 1.7 ‚Äî Domain & subdomain hub pages

> Stage 1 (THINK) artifact.

## Problem restatement

Three routes:

- `/domains` ‚Äî All 8 domains, tile grid with subdomain count + problem count per domain. Replaces the Unit 1.1 smoke-proof list.
- `/domains/[domain]` ‚Äî Domain hub: subdomain list + featured (first 3) problems in this domain.
- `/domains/[domain]/[subdomain]` ‚Äî Subdomain page: problems table + nav back to domain.

Goal: from the home page or methodology page a reader can drill `landing ‚Üí /domains ‚Üí /domains/deep-learning ‚Üí /domains/deep-learning/large-language-models ‚Üí /problems/hallucination-reduction` and return without dead-ends (¬ß10.2 "no dead-ends").

## Inputs

- `taxonomy` (Velite) ‚Äî 8 domains √ó ~80 subdomains.
- `problems` (Velite) ‚Äî 1 in Phase 1.4; will grow to 6‚Äì10 in Unit 1.11.
- Q11 (subdomain-id collision) ‚Äî URL strategy is domain-scoped; no globally-unique subdomain assumption.
- ¬ß9 ‚Äî IA spec.
- ¬ß10.4 ‚Äî perf budget; all three routes are static (taxonomy + problems both built at build time).

## Constraints

- **All routes SSG.** `generateStaticParams` from Velite. Unknown segments ‚Üí `notFound()`.
- **No client JS.** Tile grid + table are pure RSC; filtering / sorting is Unit 1.8 (the `/problems` index).
- **Domain order = `taxonomy.yaml` order.** Stable, matches MASTER_PROMPT ¬ß4.
- **Per-domain subdomain order** likewise preserves YAML order.
- **Featured-problems heuristic.** First three problems whose `(domain, subdomain)` lands under this domain, sorted by `editorial.last_curated` desc. Trivial in Phase 1 (only one problem); deterministic at scale.
- **Empty-state copy.** Most subdomains have zero problems in Phase 1. Show a clean "No problems yet ‚Äî see contributing." link instead of empty space.

## Alternatives considered

1. **Three thin page files + one shared `lib/content/load-domain.ts` helper** (chosen).
2. Render `/domains/[domain]/[subdomain]` from a sibling segment route. Cleaner for catch-all but premature.
3. Tile grid as `DomainMap` (force-graph viz). That's ¬ß11 viz catalog #4, owned by Phase 4 per ¬ß13.

## Edge cases & failure modes

1. **`theory` is both a top-level domain id and a subdomain id under `deep-learning`** (Q11). The taxonomy schema (Unit 0.5) handles it; routes `/domains/theory` and `/domains/deep-learning/theory` are both valid and distinct.
2. **Subdomain found in taxonomy but with no problems.** Render the subdomain page with the empty-state copy.
3. **Slug typo.** `notFound()`.
4. **Domain has very many subdomains** (e.g., `general-ml` with 15). Display in 1- or 2-column grid; subdomain rows wrap.
5. **Problem listed under a non-existent (domain, subdomain) pair.** Unit 0.7's validate-content already catches this ‚Äî the page just doesn't include such a problem.

## Decision

Files:

1. `docs/thinking/1.7-domain-hub-pages.md` ‚Äî this file.
2. `lib/content/load-domain.ts` ‚Äî `loadDomain(id)`, `loadSubdomain(domainId, subId)`, `problemsInDomain(id)`, `problemsInSubdomain(domainId, subId)`. Plus `allDomainIds()` / `allDomainSubdomainPairs()` for `generateStaticParams`.
3. `app/domains/page.tsx` ‚Äî tile grid, replacing the Unit 1.1 list.
4. `app/domains/[domain]/page.tsx` ‚Äî real domain hub.
5. `app/domains/[domain]/[subdomain]/page.tsx` ‚Äî real subdomain page.
6. `CHANGELOG.md` ‚Äî Unit 1.7 entry.

Stage 2 (DESIGN): tile grid is a 2-column responsive grid (`sm:grid-cols-2 lg:grid-cols-3`). Each tile: title (font-serif), subdomain count, problem count, all wrapped in a `<Link>` that hovers to accent color. Domain page lists subdomains as compact rows with their problem counts. Subdomain page is a simple `<ul>` of problem cards (title, status pill, last-curated).

Stage 3 (CODE): write the helper + three page files. Smoke.
