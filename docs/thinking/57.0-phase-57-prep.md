**XSS audit anticipated** (Unit 57.1 finalizes per APPEND-D-O
boundary):

- **`colspan` / `rowspan`** on `<th>` and `<td>`: name-only
  allow without value restriction. HTML5 spec parses these as
  non-negative integer values; non-numeric values are treated
  as 1 by browsers. The attribute value is NOT a script-
  injection vector — it's not an event handler, URL, or
  style — so the defense-in-depth value-restriction
  discipline (per APPEND-D-Q item 3 note) reduces to "schema
  allows the attribute name; browser parses value per HTML5
  spec; no XSS surface introduced". Documented in Unit 57.1
  APPEND-D-AO XSS-audit boundary section.
- **`scope`** on `<th>`: literal enum restriction via tuple
  form `["scope", "row", "col", "rowgroup", "colgroup"]` —
  matches the 4 valid HTML5 `scope` attribute values
  verbatim. Any other value strips the attribute at
  `rehype-sanitize`. Mirrors the Phase-39 `align` value-
  restriction discipline.
