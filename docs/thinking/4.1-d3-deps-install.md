# 4.1 — D3 sub-package install

## THINK

### Inputs

- **Unit 4.0 D-1**: DomainMap renders to SVG, not Canvas. Force simulation runs in JS regardless of paint target. d3-force is the canonical force-layout library.
- **Unit 4.0 D-2**: tree-shaken sub-packages only, no umbrella `d3`. Projected client-bundle bump ~20–25 KB gz across the phase.
- **No D3 deps installed today** (verified against `package.json`). This is the first D3 unit in the project.
- **Package manager**: pnpm 11.1.2 (Q12 resolved). `pnpm add <pkg>` is the canonical install path.

### Scope (this unit)

Install the **minimum-viable D3 surface** for Unit 4.2's DomainMap force-graph scaffold:

- `d3-force` (~10 KB gz) — force simulation primitives (`forceSimulation`, `forceLink`, `forceManyBody`, `forceCenter`, `forceX`, `forceY`).
- `d3-selection` (~5 KB gz) — declarative DOM updates inside the simulation tick handler.

Plus `@types/d3-force` + `@types/d3-selection` as devDependencies for the `tsc --noEmit` smoke.

### Scope deferred to Unit 4.2

- **`d3-scale`**: Unit 4.0 D-4 specified `radius = sqrt(composite) × k`. Plain `Math.sqrt` covers this without d3-scale. If Unit 4.2 ends up wanting `scaleLinear()` or `scaleSqrt()` for cleaner readability or range/domain clamping, install there. Avoid premature install (§14 minimalism).
- **`d3-zoom`**: Unit 4.0 D-6 scoped zoom OUT of Phase 4. Skip entirely.
- **Storybook story**: defers to 4.2 when DomainMap exists. A Storybook story for d3-force-without-DomainMap would be artificial — there's no rendered component to story.

### Constraints

- **Pure JS, no postinstall.** D3 sub-packages don't have native build steps; `pnpm approve-builds --all` is not needed.
- **Client deps, not dev.** Even though DomainMap will be `"use client"`, the imports show up in the build's client bundle — they belong in `dependencies`, not `devDependencies`. Mirrors how `fuse.js` (Unit 1.9's search) is a regular dep.
- **`@types/d3-*` are devDeps.** TS-only metadata; not shipped at runtime.
- **No app code changes.** This unit is install-only. Unit 4.2 owns the first import.
- **Lockfile collision.** pnpm-lock.yaml rewrites are global; this unit cannot run in parallel with anything else touching lockfile.

### Alternatives considered

- **Install all 4 sub-packages now** (force + selection + scale + zoom). Rejected — d3-zoom is out of Phase 4 scope per Unit 4.0 D-6; d3-scale is hypothetical until 4.2 confirms it's needed. Adding deps "for the future" inflates bundle and audit surface unnecessarily.
- **Install the umbrella `d3`**. Rejected — Unit 4.0 D-2; ~150 KB gz vs ~15 KB gz tree-shaken.
- **Use `@tinybase/force-graph` or `d3-force-3d` or similar**. Rejected — d3-force is the canonical force-layout library, well-typed, broadly understood, and at ~10 KB gz it's not the bottleneck.
- **Use the React-native wrappers** (`react-force-graph-2d` etc.). Rejected — those bundle Canvas-based renderers; Unit 4.0 D-1 chose SVG. Manual d3-force + custom SVG render gives full control over a11y plumbing.

### Tradeoffs / defensible defaults flagged

1. **Pinned vs. floating version**. Existing deps use `^` ranges with major-minor floors (e.g., `react ^19.0.0`). Same pattern for D3: `^3.0.0` floors (d3-force v3 is the current stable). lockfile pins the exact version at install time.
2. **No Velite config change.** D3 is client-runtime, not build-time. Velite continues to emit `.velite/*.json`; DomainMap reads those at SSR + hydrates with d3-force on the client.
3. **No tsconfig change.** `@types/d3-*` are auto-discovered by TS via `node_modules/@types/`. `compilerOptions.types` is not gated.

### Smoke gates

- `pnpm install` lockfile clean (no peer-dep warnings on the new deps; d3 v3 has no peer-deps).
- `pnpm typecheck` clean (newly-installed types should not trigger any `tsc` errors; no imports added yet).
- `pnpm test` 171/171 across 25 files (unchanged from Unit 4.0).
- `pnpm validate-content` 203 files (unchanged).
- `pnpm build` 198 routes (no app changes; bundle unchanged at this commit since nothing imports d3 yet — Unit 4.2 owns the first import + the bundle bump).
