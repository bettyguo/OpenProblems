### Decisions pinned (per Unit 15.0 D-10 anticipated shape)

**D-A. Editable field set.** Pins `displayName` (80 chars max) + `bio`
(280 chars max) inclusion; `users.image` override / `users.githubLogin`
mutation / `users.email` visibility / `users.name` mutation all
EXCLUDED. Length caps match GitHub / Bluesky / Twitter standards;
conservative; Phase 16+ may expand.

**D-B. Validation + sanitization model.** Length cap (client +
server) + trim whitespace + empty string after trim = clear (NULL)
+ React's default escape handles XSS + NO regex content filter (Q68
flagged) + NO uniqueness check on displayName (`githubLogin` remains
unique URL key).

**D-C. Edit surface route shape.** Extend existing
`/[locale]/profile` route with inline edit form. NOT separate
`/profile/edit`. Preserves ADR-0015 D-F's "two surfaces per identity"
pattern verbatim.

**D-D. Server-action vs API route.** Server actions (mirrors Phase
10/11/12 precedent for signed-in own-state mutations). NOT REST
endpoint; no third-party demand; CSRF + cookies handled for free.

**D-E. Public consumption fallback chain.** Extends Phase 14's chain
to `displayName ‚Üí name ‚Üí githubLogin ‚Üí translated fallback`. Applies
to `/u/{handle}` (Phase 14), `/profile` (Phase 10), AuthControl pill
(Phase 9). SiteHeader "Your profile" link UNCHANGED (uses `@login`
URL key, not displayName).

**D-F. Bio display formatting.** Plain text via `whitespace-pre-wrap`;
280-char cap; NO markdown / NO HTML / NO link-detection (Q66 flagged).
Bio placement on `/u/{handle}`: between header card and activity
section. Omitted entirely when null (no placeholder).

**D-G. Image override Phase-16+ deferral.** Requires storage ADR
(**ADR-0017 candidate**) ‚Äî Vercel Blob vs S3 / R2 vs external URL
allowlist. Image upload pipeline + cropping + EXIF stripping +
content moderation each their own concern. Phase 15 ships text-only
foundation; Phase 16+ extends.

### Why an ADR at all?

Two reasons:

1. **First user-controlled writes surface for `users` table** crosses
   ADR-0012 D-E + ADR-0013 D-F + ADR-0014 D-E + ADR-0015 D-A/C/D
   boundaries. Pinning the editable-field contract + validation model
   + edit-surface shape + server-action choice + public consumption
   fallback chain + bio formatting + image override deferral in ONE
   document prevents downstream drift when Phase 16+ image override /
   markdown / privacy opt-out surfaces land.
2. **Q63 promotion was explicit Phase 15+ deferral** (ADR-0015 D-C):
   "Phase 14 ships READ-ONLY public profile ‚Ä¶ Phase 15+ writes can
   extend the contract incrementally." Phase 15 is the deferral-honor;
   not landing an ADR would be a documentation regression.

Contrast Phase 13: Q58 promoted to resolved WITHOUT new ADR because
the lean was documented across three prior units (11.6 + 12.0 D-10
+ 13.0 D-3). Phase 15's editable-field surface spans more architectural
ground (writes API + validation + sanitization + length caps + public
consumption updates + image override Phase-16+ deferral); ADR-worthy.

### Options + chosen rationale (mirrors ADR body)

Four options considered:

1. **`displayName` + `bio` + plain-text + server-action + `/profile`
   extension** (CHOSEN). Honors ADR-0015 D-C deferral path; preserves
   public-data invariant; scope cap discipline (9 units); reversible
   image / markdown / moderation promotions Phase 16+.
2. **Full editable Phase 15 (incl. image override + markdown + edit
   history)**. REJECTED. Triples scope; needs storage ADR + sanitization
   library + audit-log migration. Phase 16+ is the natural home.
3. **Markdown bio in Phase 15**. REJECTED. New dependency (remark +
   rehype + sanitize plugin); XSS audit surface; conservative
   plain-text default beats premature richness. Q66 candidate Phase
   16+.
4. **Separate `/profile/edit` route**. REJECTED. Splits editing
   affordance from context (watchlist + challenges); lifts Phase-9
   Class B item 12 middleware threshold unnecessarily; ADR-0015 D-F
   "two surfaces per identity" already sufficient.

### Architectural relationships pinned

- **Inherits ADR-0015 D-A** public-data invariant (no NEW public-data
  category introduced; editable fields are user-controlled overrides
  of fields already public elsewhere).
- **Inherits ADR-0015 D-C** Phase-15+ deferral path verbatim.
- **Extends ADR-0015 D-F** "two surfaces per identity" pattern ‚Äî
  `/profile` becomes the edit-mode surface with inline edit form;
  `/u/{login}` stays public canonical.
- **Inherits ADR-0014 D-E** ALTER migration discipline ‚Äî Phase 15's
  `0004_user_profile_fields` is the second ALTER migration in project
  history; same additive nullable column pattern.
- **Reuses ADR-0012 D-E** `users.githubLogin` joining (excluded from
  editable fields; URL key immutable).
- **Reuses ADR-0011** i18n strategy ‚Äî new `messages.profile_edit.*`
  namespace follows sibling-file convention.
- **Establishes user-controlled writes pattern** that Phase 16+
  identity-edit surfaces (image override; opt-out toggle; markdown
  bio) inherit verbatim.
