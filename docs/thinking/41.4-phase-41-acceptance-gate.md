## Architectural firsts in Phase 41 (13 enumerated)

1. **Third concrete Phase-37-framework consumer** —
   `ArxivExtensionRegistry` ships; **completes 3-of-3 framework
   slot demonstration via real consumer**. After Phase 41
   every framework slot has at least one real-consumer
   existence proof: Phase 38 `rehypePlugins` + Phase 39
   `schemaOverrides` + Phase 41 `remarkPlugins`.
2. **First mdast-level synthesis-emitting remark plugin** in
   project history — `remark-gfm` (Phase 17) is a syntax-
   parser hook; arxiv autolinks splice new `link` nodes from
   inside pre-existing `text` nodes via regex pattern match.
   Different shape from `remark-gfm`'s parser-hook style.
3. **First absolute-URL emission** by a framework extension
   — Phase-38 wikilinks emit relative `/problems/{slug}`;
   Phase-39 tables emit no URLs; Phase-41 arxiv emits absolute
   `https://arxiv.org/abs/<id>`. First framework consumer to
   require ZERO sanitize-schema cooperation (`<a>` + `href` +
   `https:` all already in Phase-17 base allow-list).
4. **First content-aligned framework extension** — project
   mission is "open AI research problems platform"; paper-
   citation auto-linking is mission-aligned in a way the
   format-oriented earlier consumers (wikilinks navigation +
   tables formatting) were not.
5. **First `rationale`-surface extension** — the Phase-27
   challenge-resolution-rationale surface gains an extension
   for the first time. After Phase 41 ship, **only `bio`
   remains un-enabled-for-any-consumer** among the four wired
   markdown surfaces.
6. **First 3-way composition feasibility VALIDATED end-to-end**
   — `MARKDOWN_EXTENSIONS=wikilinks,tables,arxiv` flows
   through Phase-40 multi-value parsing + composite-registry
   wrapping with zero rework; 3 distinct slots × 3 disjoint
   surfaces × the standard composition pathway = conflict-
   free under APPEND-D-R rules. Validates the "arbitrary
   disjoint-surface multi-consumer composition" claim from
   Phase-40 boundary statement with a 3-way real-consumer
   case end-to-end through the full markdown pipeline.
7. **Eighth APPEND on ADR-0018 D-G** — extends the **first-
   ADR-D-clause-with-most-APPENDs record** from 7 → 8.
   Cumulative APPENDs: Phase 18 + 27 + 29 + 37 + 38 + 39 +
   40 + 41.
