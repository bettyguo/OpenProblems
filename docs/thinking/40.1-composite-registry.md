## THINK

### Problem restatement

Implement `CompositeExtensionRegistry` per Phase-40-prep D-3.
The class implements `MarkdownExtensionRegistry`, takes a
`ReadonlyArray<MarkdownExtensionRegistry>` of components, and
returns merged extension sets per surface:

- `remarkPlugins` + `rehypePlugins`: concatenate across all
  components in registration order.
- `schemaOverrides`: at most one component per surface; throw
  on conflict.

### Inputs

- HEAD `80f458f` (Unit 40.0 ship; Phase 40 prep).
- `lib/markdown/extensions/types.ts` — `MarkdownExtensionRegistry`
  + `MarkdownExtensionSet` interfaces.
- `lib/markdown/extensions/wikilinks.ts` + `tables.ts` —
  existing 2 consumers for end-to-end test fixtures.

### Constraints

- **Server-side only**.
- **No new runtime deps**.
- **TS strict + `exactOptionalPropertyTypes: true`**.
- **APPEND-D-C override-replace semantics preserved**:
  schema-override conflict is loud-failure (throw), not
  silent merge.
- **Empty components array allowed** — returns `{}` for all
  surfaces (degenerate case; would be returned only via
  factory misconfiguration which Unit 40.2 prevents).

### Edge cases & failure modes

1. **Empty components array**: registry returns `{}` for every
   surface (each iteration adds nothing). Documented but
   unreachable via Unit-40.2 factory.
2. **Single-component composite**: returns the same set the
   component would return; equivalent to using the component
   directly. Factory short-circuits to direct registry for
   single-value env-vars per Phase-40-prep D-E.
3. **Plugin-array concatenation preserves order**: registration
   order is the same as the env-var order. Tests verify.
4. **Schema-override conflict**: two components both providing
   `schemaOverrides` (any field) on the same surface → throws
   with message naming both component indices.
5. **Mutation safety**: components' returned extension sets
   must not be mutated by the composite (would leak across
   surfaces if a component returned a shared mutable
   reference). The composite copies the plugin arrays it
   builds; does NOT clone `schemaOverrides` (the first
   component's reference is returned as-is, mirroring direct-
   registry behavior).
6. **Empty extension sets**: components returning `{}` for a
   surface contribute nothing to the merged result.
