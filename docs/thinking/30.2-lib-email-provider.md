# 30.2 — `lib/email/` provider client + verification + welcome templates + 19 new tests (646/59)

Third Phase-30 unit; code unit. Realizes ADR-0021 D-A (provider
choice = Resend) + D-F (Phase-30 email content = verify + welcome
templates). Establishes the provider-isolated `lib/email/` module
pattern. Mirrors Phase-17 `lib/markdown/` + Phase-19 `lib/storage/`
module-introduction units.

## THINK

### Scope realized

Five new files + 1 dep install:

1. **`lib/email/index.ts`** (new; 90 lines): Resend SDK wrapper.
   Lazy-init `Resend` client behind `getResendClient()`; cached
   across calls. `sendEmail(to, subject, html, options?)` returns
   structured `SendEmailResult` discriminated union (`{ok: true, id}`
   or `{ok: false, error}`). Graceful degradation when
   `RESEND_API_KEY` unset (returns `{ok: false, error: "missing_api_key"}`);
   mirrors `safeAuth()` Phase-9 pattern. Test-only
   `__resetResendClientForTests()` clears cache between vitest
   suites.
2. **`lib/email/templates/verification.tsx`** (new; ~100 lines):
   `VerificationEmail` TSX server component + `renderVerificationEmail`
   helper. Renders to static HTML via `renderToStaticMarkup` from
   `react-dom/server`. Inputs: verificationUrl + domainTitles +
   supportEmail (all server-trusted; no user-input echo per
   ADR-0021 §15.5 reviewer-mode mindset (f)). Mentions 24h expiry.
3. **`lib/email/templates/welcome.tsx`** (new; ~80 lines):
   `WelcomeEmail` TSX server component + `renderWelcomeEmail`
   helper. Mirrors verification's render strategy. Mentions
   "scheduler launches in a future release" per ADR-0021
   Consequences foundation-only weak-UX mitigation. Includes
   unsubscribe URL.
4. **`lib/email/index.test.ts`** (new; 9 tests): Resend SDK
   stubbed via `vi.hoisted` + `vi.mock`. Tests structured failure
   cases (missing API key + missing from + provider error +
   SDK throw + string-only throw fallback) + happy path + client
   caching across calls (constructor invoked exactly once across
   3 sends).
5. **`lib/email/templates/verification.test.tsx`** (new; 5 tests):
   verification email render validation (URL in href + plain-text
   fallback; domain titles; mailto support email; `<!DOCTYPE html>`
   prefix; 24h expiry mention).
6. **`lib/email/templates/welcome.test.tsx`** (new; 5 tests):
   welcome email render validation (unsubscribe URL in href;
   bold domain titles; mailto support email; `<!DOCTYPE html>`
   prefix; "future release" mention).

Total: **19 new tests across 3 new vitest files**.

**Dependency added**: `resend@6.12.3` (Resend SDK; latest stable
at Phase 30; satisfies ADR-0021 D-A `resend@4+` minimum). Server-
only; bundled into the server runtime; client bundle unaffected.

### Architectural firsts shipped in this unit

- **First `lib/email/` infrastructure** in project history.
- **First Resend SDK integration**.
- **First TSX server component rendered to HTML string via
  `renderToStaticMarkup`** in project history (Phase-17 markdown
  rendering used `rehype-stringify` instead; Phase-19 image
  processing produces binary output). Distinct from React's SSR
  page rendering (which returns a tree, not a string).

  project history. Phase-28 `lib/auth/link-account.test.ts` used
  plain `vi.fn()` mocks; Phase-30 needs class-shape because
  Resend is a class. Establishes hoisted-class pattern for
  future SDK-class mocks.
- **First test-only export hook** (`__resetResendClientForTests`)
  in project history. Pattern matches the lazily-initialized
  singleton + test-isolation requirement.
