# 2.5b — Author backfill for Unit-2.5 papers (batch 2)

## THINK

### Problem restatement

Unit 2.5 ([29e01fb](https://github.com/anthropics/llm-openproblems/commit/29e01fb)) landed
10 papers with `authors: []` — a deliberate deferral flagged in
[2.5-seed-papers-batch-2.md](./2.5-seed-papers-batch-2.md):

> Authors stay `[]`. … the authors of these 10 papers don't have slugs yet
> … Backfilling lands in a later Phase-2 unit when more authors are minted.

The session handoff at HEAD ≈ `80cf802` named this as one of the three
non-blocking Phase-2 follow-ons. Unit 2.6c then shipped the ROR-ID backfill
for the 5 Unit-2.6b institutions; Unit 2.6d extended that to the original
9 institutions. This unit lands the **batch-2 half** of the remaining
author backfill; a sibling unit (2.6e) follows for batch 3.

### Inputs

- **10 paper YAML files in `content/papers/`.** Their arXiv IDs:
  `1805.00899`, `2005.11401`, `2211.00593`, `2211.03540`, `2304.14997`,
  `2308.00352`, `2308.11432`, `2309.08600`, `2311.12022`, `2404.06654`.
  All carry `authors: []`.
- **`PaperSchema.authors: z.array(slug)`.** Each entry must match
  `/^[a-z0-9-]+$/`. Names with diacritics are slugified ASCII-only.
- **`AuthorSchema` (`lib/schemas/author.ts`).** Required: `slug`,
  `display_name`. Optional and routinely absent in existing files:
  `affiliations[]`, `homepage`, `scholar_id`, `orcid`. Established
  convention (e.g. [`percy-liang.yaml`](../../content/authors/percy-liang.yaml),
  [`shunyu-yao.yaml`](../../content/authors/shunyu-yao.yaml)) is
  three lines: `slug` + `display_name` + `affiliations: []`.
- **12 existing author slugs (post-Unit-2.4).** From this batch's
  ~130 author-references, exactly two intersect: `dario-amodei`
  (on `1805.00899` and `2211.03540`) and `jacob-steinhardt`
  (on `2211.00593`). Reused, not re-minted.

### Constraints

- **§15.6 — no fabrication.** Author lists must be attested from a
  primary source. The arXiv abstract page for each ID is the canonical
  attestation (`https://arxiv.org/abs/<id>`). WebFetch the page; copy
  the author list verbatim; slugify; mint missing authors. Do not guess
  middle initials, full names, or affiliations.
- **HF MCP unavailable.** Same constraint as Unit 2.5 itself
  (`Server not found` on `paper_search`). WebFetch on `arxiv.org` is
  the fallback attestation channel and is allowed under the CURATION
  contract's "primary source" rule.
- **`exactOptionalPropertyTypes: true`** in tsconfig. Author YAMLs
  omit `homepage`, `scholar_id`, `orcid` rather than writing them as
  `null` — matches existing pattern.
- **No invented ORCIDs, scholar IDs, homepages.** Same rule that kept
  ROR IDs off Unit 2.6b's institutions until Unit 2.6c / 2.6d had
  verified pulls from ror.org. A future per-author hygiene unit can
  backfill these from authoritative sources.
- **`affiliations: []` for all minted authors.** The schema requires
  an array; the existing 12 authors all carry `affiliations: []`. A
  proper backfill (matching paper-time institution + ISO dates) is its
  own curation pass — out of scope here, and out of scope for the
  existing 12 too, so consistency wins.
- **ASCII slug only.** `lib/schemas/_primitives.slug` enforces
  `/^[a-z0-9-]+$/`. Diacritics are stripped (e.g. *Küttler* →
  `heinrich-kuttler`, *Rocktäschel* → `tim-rocktaschel`,
  *Garriga-Alonso* → `adria-garriga-alonso`,
  *Lukošiūtė* → `kamile-lukosiute`, *Schmidhuber* → `jurgen-schmidhuber`).
  Display names preserve the published spelling, including diacritics
  (file encoding is UTF-8, no BOM, LF).
- **Cross-link audit must stay green.** The audit's
  `danglingPaperAuthorRefs` counter currently shows 0 because the
  arrays are empty. After this unit it should still be 0 — every slug
  referenced in a paper's `authors[]` must have a matching
  `content/authors/<slug>.yaml`.

### Alternatives considered

1. **Backfill both batches 2 and 3 in one unit.** Rejected — that's
   20 papers and ~80+ unique authors, well past the "small commit"
   rhythm. Splits cleanly: 2.5b for batch 2 (this unit), 2.6e for
   batch 3 (next unit).
2. **Mint only the first author per paper to keep the diff small.**
   Rejected — the cross-link audit's value comes from being able to
   answer "what papers did author X co-author?", which collapses to
   first-author-only with this shortcut. Faithful author lists are
   cheap once we have the WebFetch attestation in hand.
3. **Mint authors with full affiliations[] populated.** Rejected —
   affiliation history requires `from`/`to` ISO dates that are
   curator-attested per-employment-spell. None of the 12 existing
   authors carry this and adding it now creates inconsistent depth.
   A later "author hygiene" unit handles this uniformly.
4. **Skip surname-disambiguation (e.g. two authors named "Mike Lewis").**
   Rejected for safety — but this batch doesn't have a known clash,
   so the deferral costs nothing. If a future paper introduces a
   collision, the slug for the later-arriving author gets a
   middle-initial or institution disambiguator and that's documented
   in their YAML.
5. **Use AI-search inference instead of WebFetch attestation.** Rejected
   — §15.6 cardinal rule. The arXiv abstract page is the primary
   source. WebFetch supplies the bytes; AI just extracts the author
   list verbatim from those bytes.

### Edge cases & failure modes

1. **Anthropic's *Measuring Progress on Scalable Oversight* (`2211.03540`)
   has 46 authors.** Long author lists are expected for Anthropic
   team papers. Mint all of them; the audit doesn't care about list
   length.
2. **Author slug collision with an existing entity.** Each newly-minted
   slug is checked against `content/authors/` before writing — if a
   match exists, reuse it (this is how `dario-amodei` and
   `jacob-steinhardt` get reused on `1805.00899` and `2211.00593`
   respectively).
3. **Diacritic-only names producing an empty slug.** Handled
   defensively by slug-creation logic (lowercase + ASCII-fold
   diacritics + hyphenate-separators + strip non-alphanumerics).
   Verified with a manual eyeball pass of the 6 diacritic names in
   this batch.
4. **arXiv version-bump / withdrawn / replaced paper.** If WebFetch
   on `https://arxiv.org/abs/<id>` redirects or 404s, fall back to
   the published-venue page (NeurIPS / ICLR / OpenReview) for the
   author list. Don't proceed with an unattested list. (Not triggered
   in this unit — all 10 fetches succeeded.)
5. **Paper has a corporate author (e.g. "OpenAI" listed instead of
   individuals).** Not encountered in any of these 10 papers; all 10
   list individual authors on arXiv.
6. **Slug clashes between author and institution namespaces.** Both
   collections key by slug; a paper-time accident where an author's
   slug equals an institution's slug would be caught by the audit's
   cross-collection scan. Not expected with this batch, and verified
   none of the 114 minted slugs collide with the 14 institution slugs.
7. **Parallel-curator collision on `content/authors/`.** A sibling
   session could be minting overlapping authors. Mitigation: the mint
   script checks `fs.existsSync(target)` before writing, so re-runs are
   idempotent. After the parallel session's Unit 2.6d commit, the
   state was: their commit touched institutions + CHANGELOG + their
   THINK doc; not authors. No collision.

### Decision

For each of the 10 batch-2 papers:

1. WebFetch `https://arxiv.org/abs/<id>` and extract the author list.
2. Slugify each author name (`first-last`, ASCII, no diacritics).
3. For each slug not already in `content/authors/`, write a 3-line
   author YAML (slug + display_name + `affiliations: []`).
4. Update the paper's YAML so `authors:` lists the slugs in
   publication order.
5. Run `pnpm validate-content`, `pnpm audit-content`, `pnpm test`,
   `pnpm typecheck`. The audit's `danglingPaperAuthorRefs` line must
   remain `0` (now meaningful — non-empty arrays exist).
6. Update `CHANGELOG.md` and commit as `chore(phase-2): unit 2.5b`.
