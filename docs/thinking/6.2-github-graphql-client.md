# 6.2 — `lib/discussions/github-graphql.ts` (first-party GitHub GraphQL read-side client)

First code unit of Phase 6. Implements ADR-0010 D-B (first-party GraphQL read-side) + D-D (token discipline) + D-E (filesystem cache). Mirrors the Phase-5 `lib/curate/arxiv-client.ts` + `lib/curate/anthropic.ts` precedent shape — env-driven secrets, filesystem cache with hash key, test-injection points for mock-the-network.

## THINK

### Inputs

- **ADR-0010** (`docs/adr/0010-discussions-backend.md`, `300cde3`) pins:
  - **D-B**: read-side metadata = first-party GitHub GraphQL via `lib/discussions/github-graphql.ts`; build-time only.
  - **D-C**: per-problem mapping = pathname-based (`mapping: "pathname"`); lazy discussion creation.
  - **D-D**: `GITHUB_TOKEN` env, `public_repo` minimum scope, never write; mirror ADR-0008 D-C env-token discipline (fail loud when unset).
  - **D-E**: `.github-cache/<query-hash>.json` (gitignored); per-build TTL.
  - **D-F**: defer moderation to GitHub native (not this unit's surface).
- **`lib/curate/arxiv-client.ts`** (Phase-5 Unit 5.2) — the cache + fetch precedent. Patterns to mirror: `interface FetchXOptions` with optional `noCache` / `cacheDir` / `fetchImpl`/`clientFactory`; `__testing` export for internal helpers; `readCache` / `writeCache` helpers reading/writing JSON sidecars.
- **`lib/curate/anthropic.ts`** (Phase-5 Unit 5.2) — the env-token + `clientFactory` precedent. Pattern: read env at call time (not module load); throw with a specific error string citing the ADR; expose `clientFactory` option so tests fully mock the SDK.
- **GitHub GraphQL API** ([docs.github.com/graphql](https://docs.github.com/en/graphql)) — `repository.discussions` returns a paginated list of discussions; `search(type: DISCUSSION, query: "...")` finds discussions by query. Both shapes are documented + stable.
- **Rate limit**: GitHub primary rate limit for authenticated GraphQL = 5000 points/hour (each query costs 1+ depending on node count). Build-time SSG with 10 problems × 1 query each = 10 points per build; comfortably under.
- **Q47 (open)**: Discussions must be enabled in the `bettyguo/OpenProblems` repo settings. Without it, queries return empty. Tests use mocks; production-deploy validation pends Q47 resolution.

### Scope (this unit)

A single new module + its test file + a dependency install + a `.gitignore` entry:

1. **Install `@octokit/graphql`** (pure JS; no postinstall — `pnpm-workspace.yaml` `allowBuilds` not affected). Added to `dependencies` (not `devDependencies`) because SSG pages in Units 6.3-6.6 import it at build time, paralleling how `@anthropic-ai/sdk` sits in `dependencies` despite being script-only — ANY import path from `lib/` or `scripts/` or `app/` qualifies for `dependencies`.
2. **`.gitignore`** addition: `.github-cache/` under a new Phase-6 comment block.
3. **`lib/discussions/github-graphql.ts`** exports:
   - `interface DiscussionMetadata` — `{ discussionId, url, commentCount, lastActivityAt, categoryName, title }`.
   - `interface RecentActivityItem` — discussion + latest-comment metadata for the digest pipeline.
   - `interface GraphqlClientOptions` — `noCache?`, `cacheDir?`, `clientFactory?`, `now?`, `repoOwner?`, `repoName?`.
   - `async function queryGitHub<T>(document, variables, options)` — the core cache+auth+call wrapper.
   - `async function getDiscussionByPath(pathname, options)` — returns `DiscussionMetadata | null` for the discussion matching the pathname-mapping; null if none exists yet (Giscus pathname mapping creates discussions lazily per ADR-0010 D-C).
   - `async function getRecentDiscussionActivity(since, options)` — returns `RecentActivityItem[]` for the digest pipeline window.
   - `__testing` export with `sha256`, `cacheKeyFor`, `readCache`, `writeCache`, `DEFAULT_CACHE_DIR`.
4. **`lib/discussions/github-graphql.test.ts`** with at least 8 tests covering:
   - `sha256` stability + uniqueness.
   - `cacheKeyFor` shape (sha256 of `document + JSON.stringify(variables)`).
   - Cache hit returns cached payload without calling the client factory.
   - Cache miss calls the client factory + writes cache + returns result.
   - `noCache: true` skips read but still writes.
   - Missing `GITHUB_TOKEN` env throws a clear ADR-0010-citing error (when no `clientFactory` is provided AND a real call would be made).
   - `clientFactory` override fully replaces the network path (mock-the-network).
   - `getDiscussionByPath` returns `null` when the search returns 0 nodes.
   - `getDiscussionByPath` parses metadata correctly when the search returns a discussion.
   - `getRecentDiscussionActivity` filters by `since` and returns items in expected shape.

### Constraints

- **ADR-0004 (file-first)** holds. Cache is per-build JSON sidecars; no DB.
- **ADR-0010 D-D** — `GITHUB_TOKEN` env-only; no fallback; fail loud (mirror ADR-0008 D-C error-string framing).
- **§14.1 TS strict + `exactOptionalPropertyTypes`** — `clientFactory?: () => GraphqlClient` style with no `undefined` default. The cache option for `cacheDir` follows the Phase-5 precedent (`options.cacheDir ?? DEFAULT_CACHE_DIR`).
- **§14.2 testing** — minimum 8 tests; mirror the arxiv-client test density (13 tests in Phase-5 5.2; aim for similar coverage proportional to the smaller surface).
- **`exactOptionalPropertyTypes`** care with the GraphqlClient option — assign-only-when-defined pattern.
- **No CLI in this unit.** The lib is consumed by SSG pages in subsequent units; no `package.json` script entry.
- **No write-paths.** Read-only client; `public_repo` token scope is sufficient.

### Alternatives considered

- **Use `@octokit/rest` (REST API) instead of `@octokit/graphql`.** Rejected — Discussions are NOT exposed via the REST API; only GraphQL. `@octokit/rest` would not help.
- **Use `graphql-request` instead of `@octokit/graphql`.** Rejected — `@octokit/graphql` is the GitHub-canonical wrapper; handles auth + rate-limit headers natively. `graphql-request` is provider-agnostic but adds a thin abstraction layer for no gain here.
- **Build a fetch-based client without an SDK.** Rejected — GitHub GraphQL has auth header conventions (`Authorization: Bearer <token>`) + a specific endpoint URL; `@octokit/graphql` handles these correctly. ~50 lines of indirection saved.
- **Module-level fetch of GITHUB_TOKEN.** Rejected — mirror Phase-5 anthropic precedent of reading env at *call time*, not at module load. Avoids test-pollution + lets tests override via `clientFactory`.
- **Different cache strategies** (TTL-based vs build-keyed). Rejected — per-build TTL is the simplest defensible behaviour; matches the SSG build cadence. A future `pnpm clean-cache` script handles invalidation.
- **Different cache key shapes** (e.g., per-discussion-id JSON vs per-query-hash JSON). Rejected — query-hash is provider-agnostic (matches the ADR-0009 D-E `.pdf-cache/<id>.json` precedent semantically while being parameterised on `document+variables` instead of a single id).

### Tradeoffs / defensible defaults flagged

1. **`@octokit/graphql` becomes a `dependencies` entry** (not `devDependencies`). Justified: SSG-time imports → ships as part of the build pipeline → `dependencies` is the conventional home. Same as `@anthropic-ai/sdk` (script-only but in `dependencies`).
2. **No retry/backoff on GraphQL errors.** Build-time fetches that fail surface as a build error, not a silent fallback. Acceptable for v1; if rate limits become real (>100 problems × multiple queries), revisit.
3. **Cache files are per-build-ephemeral** (no TTL inside the cache; CI builds fresh; local devs delete `.github-cache/` to refresh). Matches the `.arxiv-cache/` lifecycle from Phase-5.
4. **Tests fully mock the GraphQL endpoint via `clientFactory`.** No real-network tests in this unit — those land in the acceptance-gate Unit 6.10 (or earlier in a Q47-resolution follow-on).
5. **`getDiscussionByPath` returns `null` for "no discussion yet"** (not throw). Pathname-based lazy creation means missing discussions are the normal happy-path before any visitor comments. Throwing would be wrong semantics.
6. **Repo owner/name defaults to `bettyguo/OpenProblems`** read from git remote convention (already used in Phase-4 issue templates + `/contributing` MDX links). `options.repoOwner` / `options.repoName` override in tests + for forks.
