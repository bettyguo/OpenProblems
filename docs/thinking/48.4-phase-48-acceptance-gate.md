## Third realization of Phase-46 phase-shape pattern + second dual-form regex + first triple-alias surface

- `DOI_PATTERN` regex evolved in-place from
  `/\bdoi:(10\.\d{4,9}\/[-._;()/:A-Za-z0-9]+?)(?=[\s,;)]|\.(?:\s|$)|$)/gi`
  (Phase 45 ship) to
  `/\[\[doi:(10\.\d{4,9}\/[-._;()/:A-Za-z0-9]+?)(?:\|([^\]\n]+))?\]\]|\bdoi:(10\.\d{4,9}\/[-._;()/:A-Za-z0-9]+?)(?=[\s,;)]|\.(?:\s|$)|$)/gi`
  (Phase 48 ship). **Second dual-form regex** in the framework
  — alternation between bracketed form (priority) and bare form
  (fallback). **First "selectively-applied lookahead in dual-
  form regex"**: bracketed alternative has explicit `]]`
  terminator → no lookahead needed (full Crossref suffix class
  permissive inside brackets); bare alternative preserves the
  Phase-45 prose-friendly trailing-punctuation lookahead.
- `remarkLinkDoiIds` plugin body updated to branch on
  `isBracketed`. Three display rules mirror Phase-47 arxiv body
  verbatim (alias defined → display alias; bracketed without
  alias → display `matched.slice(2, -2)`; bare form → display
  `matched` verbatim).
- `DoiExtensionRegistry` class + factory dispatch arm +
  `PHASE_45_DEFAULT_ENABLED_SURFACES` constant **UNCHANGED**.
- 13 NEW tests in `doi.test.ts` covering plugin-level alias
  behavior (Unit 48.1).
- 18 NEW end-to-end tests in `lib/markdown/index.test.ts`
  covering alias rendering on rationale + dual-alias on
  non-rationale surfaces + first triple-alias surface under
  4-way composite (Unit 48.2).
- ADR-0018 D-G APPEND adds Phase-48 EXTENDED block + **APPEND-
  D-AF** (Unit 48.1; sixth two-letter APPEND letter).
- ADR-0018 APPEND-D-AC item 2 deferral CLOSED at **3-phase
  carryover** (Phase 45 → Phase 48; Unit 48.3 documents).
  **Ties Phase-44 fastest-closure record of 3-phase carryover**
  (Phase 41 → 44 D-Y item 1 cross-surface arxiv). **Fastest
  alias-syntax closure ever observed**: accelerating cadence
  Phase-46 wikilinks 8-phase → Phase-47 arxiv 6-phase → Phase-
  48 doi 3-phase. Each alias-syntax realization halves the
  architectural risk for the next.
- **Third realization of the Phase-46 plugin-regex-extension
  phase-shape pattern**. Phase 46 wikilinks (rehype); Phase 47
  arxiv (remark); Phase 48 doi (remark, second). Pattern's
  **within-slot reusability** validated.
- **Second plugin-regex-extension on a `remarkPlugins` consumer
  in project history**. **First "two-consecutive-`remarkPlugins`-
  regex-extension phases" pair** (Phase 47 + Phase 48).
- **First "triple-alias surface"** in project history:
  rationale under 4-way `wikilinks,tables,arxiv,doi` default
  carries wikilinks alias (Phase 46) + arxiv alias (Phase 47)
  + doi alias (Phase 48) simultaneously. **First surface with
  3 alias-syntax consumers active** in project history. Other
  3 surfaces (bio + reviewNotes + actionRationale) remain
  **dual-alias** (wikilinks + arxiv) because doi is NOT
  enabled on them per Phase-45 default.
- **Regex-disjointness-as-sole-defense discipline** for same-
  slot composition established at Unit 48.2 ship. Phase-47's
  collision-freedom discipline had two layers (distinct
  pipeline stages + distinct regex character classes) for
  cross-slot pairs. For same-slot pairs (arxiv-vs-doi both in
  `remarkPlugins`), the staged-execution-order layer is moot;
  regex character classes become sole defense. Validated end-
  to-end through the 4-way composite test.
- **Backwards-compatibility discipline extended**: every
  existing bare `doi:10.NNNN/xxx` match preserved via the
  second alternation arm verbatim.
