### Constraints

- **`translation_source: "machine-assisted"`** required on `*.fr.mdx` per ADR-0011 D-G + Velite refine (in `velite.config.ts`). FR file must declare it; EN canonicals must NOT declare it. Both held.
- **Heading anchors**: section anchors derive from the heading text (e.g., `#1-qui-peut-contribuer` for `## 1. Qui peut contribuer`); FR translation changes the anchor slug, but no internal page links reference these anchors. Cross-page links (e.g., methodology references) point at routes (`/methodology`), not at anchors within contributing. Safe.
- **Markdown table rendering**: FR table cells preserve the markdown pipe layout; column widths recompute per content. Tested via `pnpm build` SSG output.
- **Sitemap test invariant (Unit 8.5)**: "every entry carries locale alternates" — held; `/contributing/v1.0.0` and `/contributing/v1.1.0` carry alternates universally per the `entryWithAlternates` helper.

### Alternatives considered

- **Translate v1.0 instead of v1.1**: Unit 8.0 prep's D-4 lean. Rejected after re-evaluation: v1.0 is a historical snapshot; users hitting the default `/fr/contributing` land on the latest version (v1.1). Translating v1.0 wouldn't expose FR content on the default route — the FR experience would stay 100% English unless the user explicitly navigated to `/fr/contributing/v1.0.0`. Translating v1.1 puts FR content on the most-visited surface immediately.
- **Translate both v1.0 AND v1.1**: rejected for scope discipline. v1.0 is a curator-track follow-on per Q51; translating both in one unit doubles the prose-translation work without proportional benefit. v1.0 translation can land later.
- **Skip the route refactor (keep `lang === "en"` filter)**: rejected. Without `resolveLocalized`, `/fr/contributing` would serve EN content despite the FR file existing. The pilot would be content-only with no route-side consumption — defeats the unit's purpose.
- **Add fallback-hint UI in this unit**: rejected. The methodology FR pilot (Unit 7.5) also deferred this; cross-unit consistency. Hint UI lands as a focused Class B follow-on once a "this page is not yet translated" copy + design is approved.
- **Translate FR variants of the v1.1 changelog block ("v1.1 changelog (over v1.0): added §3.6...")**: included. The §3.6 / §7 references are version-history facts; translating them keeps the FR page self-consistent.

### Edge cases & failure modes

1. **`resolveLocalized` returns null** when both FR and EN are missing: `notFound()` fires. Today this is unreachable (every version has an EN canonical).
2. **FR file lacks `translation_source` field**: Velite's post-transform refine in `velite.config.ts` rejects the file at build time. Caught at `pnpm build`.
3. **FR file has malformed frontmatter** (e.g., `version` mismatch with filename): Velite Zod schema rejects. Caught at build.
4. **Markdown rendering**: French quotation marks (`«»`), em-dashes, and accented characters survive rendering. Verified via SSG output inspection during build.
5. **Internal link to `/contributing/v1.0.0` from `[Rating challenge]` template** (none today, but possible): would render the EN fallback content under FR layout. Acceptable; documented as Class B follow-on.
