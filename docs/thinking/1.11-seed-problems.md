# 1.11 — Seed problems 2–10

## THINK

### Inputs

- **MASTER_PROMPT §16.** Lists 10 seed problems with title, domain/subdomain,
  why-now framing, and benchmark anchors. §13 says "Six is the minimum for
  Phase 1 acceptance; ten is the target." Unit 1.4 authored seed #1
  (hallucination-reduction). This unit authors seeds 2–10.
- **Unit 1.4 folder shape.** Each problem is one directory under
  `content/problems/<slug>/` containing: `problem.yaml`, `background.mdx`,
  `definition.mdx`, `history.mdx`, `ratings/<YYYY-MM-DD>-initial.yaml`. The
  Velite collections (`problems`, `ratings`, `problemPages`) and the §9
  ten-block detail page already consume this shape with no per-problem
  routing changes — `app/problems/[slug]/page.tsx` uses
  `generateStaticParams` over all problem slugs.
- **Schemas.** `OpenProblemSchema` ([lib/schemas/problem.ts](../../lib/schemas/problem.ts))
  and `RatingActionSchema` ([lib/schemas/rating-action.ts](../../lib/schemas/rating-action.ts))
  are the source of truth; `velite.config.ts` mirrors them per Q31.
- **MASTER_PROMPT §15.6.** "Never invent leaderboard numbers, paper titles,
  or rating scores — write `TODO(curate)`." Unit 1.4 followed this for
  benchmark scores (`metric_direction` set; `value` / `upper_bound` omitted).
  Same pattern here.

### Constraints

- **Scope.** Author the 9 remaining seeds in one unit. Hits the §13 target
  (10) and sets up Unit 1.12's acceptance gate cleanly. Tradeoff: each
  problem gets tight prose (2–3 short paragraphs per MDX file), in the
  same register as the Unit 1.4 baseline. Phase 2's curation pass will
  deepen the prose and fill in benchmark numbers.
- **Citations and named entities.** §15.6 forbids fabricating paper titles
  or numerical scores. Where §16 names a specific anchor (Yang et al. 2025
  inverse scaling, Snell et al. compute-optimal scaling, Han et al. 2024–26
  multi-agent survey, Berkeley RDI 2026 benchmark-integrity finding,
  9.3T-nucleotide foundation models), use it. Otherwise reference benchmark
  families and waves of work generically ("the 2023–2025 wave of …",
  "early SAE work in 2024–25").
- **Slugs.** Kebab-case; matches the routing (`/problems/[slug]`) and the
  search index. The slug `long-context-rag` is already referenced from
  `hallucination-reduction.related_problems` (Unit 1.4) so it must match.
- **Related-problem graph.** Wire each problem's `related_problems` to
  realistic neighbours in this seed set. Cross-link audit (Q33) is deferred
  to Phase 2, so non-bidirectional edges are acceptable; do not retro-edit
  Unit 1.4's `hallucination-reduction.problem.yaml` to keep this commit's
  diff scoped to net-new files.
- **`posed_year`.** Use the year the *open problem in this framing* became
  legible to the field, not the earliest precedent. For most LLM-era
  problems that's 2020–2024.
- **Rating confidence.** Lower than seed #1 (which had 0.55–0.80 across
  dimensions). For these first-pass seeds use a conservative 0.5–0.7
  range. Watchlist defaults to `false`; flip per-problem when the literal
  §8.4 criteria apply.
- **ADR-0005.** Each rating action lands as a net-new file (`A` status) so
  the pre-commit immutability hook is satisfied.

### Alternatives considered

- **Author only the 5 minimum needed to clear the §13 floor of 6.** Cleaner
  diff; defers 4 problems to a follow-up unit. Rejected — the handoff for
  this unit explicitly named "seed problems 2–10", and Unit 1.12's
  Lighthouse / Playwright sweep benefits from a fuller route surface.
- **Author full editorial-quality prose per problem.** Would push this
  unit to 3000+ lines and risk diluting the per-problem care. Rejected —
  Phase 1's job is to prove the 10-block layout works at scale; Phase 2 /
  Phase 3 do the curation pass with real numbers.
- **Skip `definition.mdx` for non-mathematical problems** (e.g.
  benchmark-integrity, multi-agent-coordination). Rejected — the §9 ten-
  block layout assumes all three MDX files exist; renaming/skipping the
  file means the detail page renders an empty section. Use prose-only
  definitions where the formalism is genuinely linguistic.

### Edge cases

- **Cross-domain subdomain `multi-agent`.** §16 puts seed #6 under
  `reinforcement-learning / multi-agent`. The taxonomy file
  ([content/taxonomy.yaml](../../content/taxonomy.yaml)) has
  `reinforcement-learning` with subdomain `multi-agent` — confirmed it
  exists; no schema-FK breakage.
- **`representation-learning` ambiguity.** Not used in this seed set, so
  no Q11 risk.
- **`large-language-models` collisions.** Three of the 10 seeds (1, 2, 10)
  are under `deep-learning / large-language-models`. The
  `/domains/deep-learning/large-language-models` page will show all three
  in its problems list — desirable density for the acceptance-gate sweep.
- **Empty `editorial.last_curated`.** All 9 use `2026-05-14` (today, per
  the conversation date). Matches Unit 1.4's seed.

### Tradeoffs / defensible defaults flagged

1. **Coverage over depth.** Tight 10–15-line MDX files, conservative
   confidences, TODO(curate) on every benchmark number. Phase 1 has the
   IA stress-tested at 10 problems; Phase 3's "Ratings dynamics" unit
   re-rates everything on the basis of curated SOTA.
2. **`posed_year` is a curatorial call.** For "Faithful Hallucination
   Reduction" Unit 1.4 chose 2022 (HaluEval / TruthfulQA era). I'm using
   the same era-anchor convention: the year a benchmark made the failure
   mode legible, not the earliest precedent.
3. **No new ADR.** No deviations from existing architecture. Q34 (cross-
   link audit / `entries.json`) remains the right Phase-2 home for
   bidirectional `related_problems` enforcement.
4. **No new tests.** Vitest covers schemas + helpers; the seeds are pure
   content and run through `pnpm validate-content` plus `pnpm build`'s
   Velite parse. A schema regression on these files would surface in CI.
