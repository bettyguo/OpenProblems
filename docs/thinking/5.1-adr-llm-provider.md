# 5.1 — ADR-0008 (LLM provider selection + cost-governance pact)

## THINK

### Decision drivers (will be in the ADR proper)

1. **§15.6 primary-source rule.** LLM-assisted curation must remain auditable; the provider must support deterministic seeds OR structured-output modes that the curator can verify against the source.
2. **§3 brand alignment.** LLM OpenProblems indexes Claude-class models among the systems whose limitations the encyclopedia tracks. Using Anthropic to draft entries about its own + competitors' work is a conflict-adjacent choice — must be disclosed in the curator workflow.
3. **Cost discipline.** Phase 5 is the first paid-API surface in the project. Defensible-from-day-zero cost governance.
4. **Operator familiarity.** The `claude-api` skill in the project's command palette presupposes Anthropic SDK usage; mismatch would be a productivity hit.
5. **Prompt caching.** Multi-paper ingest runs reuse the same MASTER_PROMPT.md context block; Anthropic's `cache_control: { type: "ephemeral" }` cuts cost by ~50%.

### Decisions (will be in the ADR proper)

**D-A. Provider = Anthropic (`@anthropic-ai/sdk`).**

**D-B. Model defaults**:
- `ingest-arxiv` → **Sonnet 4.6** (fast + cheap + good enough for metadata + abstract → YAML transformation).
- `extract-leaderboard` → **Opus 4.7** (multi-table PDF parsing benefits from frontier capability).
- `build-digest` → **Sonnet 4.6** (text summarisation; not benefit-bound).
- `--model` CLI flag overrides on every script.

**D-C. Cost-governance pact**:
- `ANTHROPIC_API_KEY` from env. No fallback. No default key.
- `--dry-run` flag on every script — prints prompts without API calls.
- `--verbose` flag prints per-call token estimate before the call.
- `LLM_OPENPROBLEMS_DAILY_BUDGET_USD` env var, optional; when set, each script reads usage from a `.llm-spend.log` (gitignored, JSONL) and aborts if the day's spend would exceed it.
- No default daily-cap (Q42 lean). Trade-off documented.

**D-D. Prompt caching**:
- The MASTER_PROMPT.md context block (when needed for a script's prompt) goes in a `cache_control: { type: "ephemeral" }` block — saves cost on multi-paper runs.
- 5-minute TTL on the cache is acceptable (curator-run sessions are typically < 5 min).

**D-E. Auditability**:
- Every script writes a `drafts/<unit>-<timestamp>-<slug>.diff.meta.json` alongside its diff containing: model used, full request token-count, response token-count, cost estimate, prompt hash (SHA256 of the rendered prompt text). Makes spot-audit + reproduction tractable.

**D-F. Conflict-of-interest disclosure**:
- The `/contributing` workflow page (Unit 4.5 / 4.6) gets a follow-on note in v1.x or v2 surfacing: "This encyclopedia uses Anthropic Claude to draft curator suggestions, including for papers from Anthropic itself. Curators are expected to apply equal scrutiny regardless of authorship." Lands as a future content unit; not in scope for ADR-0008.

### Considered options (rejected — will be in the ADR proper)

1. **OpenAI GPT-4-class** (`gpt-4o` / `o1` / `o3` / etc., via `openai` SDK).
2. **Google Gemini** (`@google/generative-ai`).
3. **Local model via Ollama** (Llama 3.x / Mistral / Qwen).
4. **Multi-provider abstraction** (`ai-sdk` / Vercel AI SDK or LangChain).
5. **No LLM** — curator drafts everything manually, Phase 5 ships only the arXiv metadata fetcher + RSS digest.

### Consequences (will be in the ADR proper)

- **Positive.** Single vendor → predictable SDK ergonomics, structured-output via tool-use, prompt caching, well-typed TS bindings.
- **Positive.** `@anthropic-ai/sdk` is mature; the `claude-api` skill exists to assist development.
- **Positive.** Decision is firm but reversible — `lib/curate/` modules abstract the provider call into a thin wrapper; a future ADR could swap providers with one-file edits.
- **Negative.** Single-vendor risk — Anthropic API outages take Phase-5 scripts offline. Mitigation: `--dry-run` lets curators iterate prompts during outages; `.arxiv-cache/` + `.pdf-cache/` let metadata + PDF work proceed.
- **Negative.** Cost. Per-paper ingest is bounded (~1k input + 1k output Sonnet 4.6 tokens ≈ $0.005 per paper); per-PDF leaderboard extraction can hit 50k+ input tokens with Opus 4.7 ≈ $0.75 per paper. 100-paper batches are $5–$75. Manageable but real.
- **Negative.** Conflict-of-interest framing (D-F) needs explicit disclosure once write-paths land.
