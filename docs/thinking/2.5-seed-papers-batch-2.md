# 2.5 — Seed papers batch 2

## THINK

### Inputs

- **§13 Phase 2 deliverable.** 30–50 seed papers covering the seed problems'
  SOTA history. Unit 2.4 landed batch 1 (10 papers across 3 problems:
  hallucination-reduction, long-horizon-agent-reliability,
  compute-optimal-test-time-reasoning). This unit lands batch 2 for the
  next four problems per the [Unit 2.0 plan](./2.0-phase-2-prep.md):
  `mechanistic-interpretability`, `scalable-oversight`,
  `multi-agent-llm-coordination`, `long-context-rag`.
- **PaperSchema + Velite collection.** `lib/schemas/paper.ts` is the
  canonical Zod-4 source; `velite.config.ts` mirrors via `s.object({...})`
  per Q31. Filename convention from Unit 2.4: `id == arxiv_id` for
  arXiv-anchored papers. `evidence` per contribution must be a real URL.
- **Anchor inventory.** Each of the four problem.yamls already names its
  anchor benchmarks and signals in `signals_considered:` and
  `benchmarks:`. The 10 papers in this batch are picked to back those
  anchors (IOI / SAE / ACDC for mech-interp; the 2018 debate paper +
  the 2022 sandwich-experiment paper + GPQA for scalable-oversight;
  MetaGPT + the autonomous-agents survey for multi-agent; the original
  RAG paper + RULER for long-context-rag).
- **Existing institution slugs.** 8 from Unit 2.1: `openai`, `anthropic`,
  `google-deepmind`, `meta-fair`, `microsoft-research`,
  `stanford-university`, `mit`, `uc-berkeley`. Each paper's
  `institutions[]` references at most one of these where the paper's
  primary affiliation matches; otherwise the field is left empty (the
  cross-link audit in Unit 2.11 will surface dangling references for
  curator follow-up).

### Constraints

- **§15.6 — never invent.** Titles are verbatim from the published
  papers. TLDRs are 3-line summaries of the public abstract; no
  fabricated numbers. arXiv IDs are well-established pre-2026 IDs from
  my training set. The HF MCP `paper_search` tool was unavailable when
  this unit ran (`Server not found`) — proceeding under the CURATION
  contract's "MCP unavailable — passive review only" rule, using only
  prior signals already named in the problem.yamls.
- **Authors stay `[]`.** Unit 2.1 only authored 5 author YAMLs; the
  authors of these 10 papers don't have slugs yet (Yejin Choi, Percy
  Liang, Owain Evans, Jacob Steinhardt, Dario Amodei are the only ones
  on disk). The PaperSchema accepts an empty array; the cross-link audit
  surfaces the gap. Backfilling lands in a later Phase-2 unit when more
  authors are minted.
- **`contributions[]` benchmark_id.** Where a paper releases or
  centrally evaluates a benchmark named in the parent problem's
  `benchmarks[]`, the contribution's `benchmark_id` matches the problem.yaml id
  (e.g. `gpqa-diamond` on the GPQA paper for `scalable-oversight`).
  Otherwise the contribution carries only `problem_slug` + `evidence` —
  signalling "this paper informs the problem" without claiming a
  benchmark assertion.
- **One contribution per paper in this batch.** Several of these papers
  legitimately contribute to multiple problems (RAG to long-context-rag
  AND hallucination-reduction; the autonomous-agents survey to multi-
  agent AND long-horizon-agent-reliability). Following Unit 2.4's
  convention, list the primary contribution only; Phase-2's cross-link
  audit can flag secondary contributions for back-fill.
- **No `score:` field.** §15.6 forbids fabricated numbers; the
  PaperSchema makes `score` optional precisely for this case. None of
  these 10 papers have a published score we can cite without curation,
  so all 10 contributions carry `evidence` + (optionally) `metric` and
  nothing else.

### Alternatives considered

- **Pick newer 2024–2025 papers over the foundational 2018–2023
  papers.** Rejected — the §13 deliverable is "the seed problems' SOTA
  history", which is the chronology naming the foundational results.
  Recent papers can land via PAPER-INGEST runs once the workflow is
  stood up (see `docs/PAPER_INGEST_RUNBOOK.md`).
- **3 + 3 + 2 + 2 = 10 split vs. 3 papers per problem.** Stuck with
  3 + 3 + 2 + 2 because the mech-interp / scalable-oversight literatures
  are denser than the multi-agent / long-context-rag literatures for the
  foundational set; the §13 floor (30 papers total) reaches at Unit 2.6.
- **Author authors[] + institutions[] in this same unit.** Rejected —
  the 2.0 plan dedicates a future unit to backfilling authors as papers
  anchor publication dates, and trying to land that here doubles the
  scope. Cross-link audit (Unit 2.11) is the right home for the
  dangling-reference detector.

### Edge cases

- **MetaGPT (`2308.00352`).** Lead institution is DeepWisdom (not in
  the 8 seed institutions). Empty `institutions[]` is valid.
- **AI Safety via Debate (`1805.00899`).** Old enough that the venue
  field uses `arXiv 2018`; no peer-reviewed venue exists.
- **Autonomous-agents survey (`2308.11432`).** Renmin University-led;
  no institution slug available. Empty array.
- **RULER (`2404.06654`).** NVIDIA-led; not in the seed institution
  list. Empty array. Named in `long-context-rag.benchmarks[]` as
  `id: ruler`; cite it.
- **GPQA (`2311.12022`).** Bowman/NYU+Anthropic. Using `anthropic`
  as the institution (matches the Anthropic-funded data construction).
  Named in `scalable-oversight.benchmarks[]` as `id: gpqa-diamond`.
- **RAG (`2005.11401`).** Originally Facebook AI Research; using
  `meta-fair` slug.

### Tradeoffs / defensible defaults flagged

1. **Foundational over recent.** This batch is the chronological
   spine; PAPER-INGEST runs handle the post-2024 wave (one paper per
   run, parallel-safe).
2. **`institutions[]` partially populated.** 4 of 10 papers have a
   slug match; 6 don't. Phase-2 cross-link audit (Unit 2.11) will
   either backfill or flag.
3. **`authors[]` uniformly empty.** Same — the 5 author YAMLs from
   Unit 2.1 don't cover any of these 10 papers' author lists. Backfill
   is a later unit per the 2.0 plan.
4. **Single contribution per paper.** Some papers earn multi-problem
   contributions; this batch lists the primary one and lets Unit 2.11
   flag the rest.
