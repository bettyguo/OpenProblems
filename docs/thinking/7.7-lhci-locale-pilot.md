# 7.7 — `lighthouserc.json` enrolment for the `[locale]/about` pilot URLs

Fifth code unit of Phase 7. Per [Unit 7.0 §F](7.0-phase-7-prep.md) original scope: "`lighthouserc.json` enrolment for the FR pilot URL". The "FR pilot" landed in Unit 7.3 as `/en/about` + `/fr/about` (not `/methodology` as originally planned — Unit 7.3 deviated). This unit enrols **both** locale variants in `lighthouserc.json` so the CI Lighthouse cohort runs the canonical a11y ≥ 95 / perf ≥ 95 / SEO ≥ 95 gates on the bilingual surfaces on the first PR with this commit.

## THINK

### Inputs

- **[`lighthouserc.json`](../../lighthouserc.json)** — 14 URLs at HEAD; targets: `localhost:3000` (LHCI starts `pnpm start` and probes); thresholds: `perf:0.95 error / a11y:0.95 error / seo:0.95 error / best-practices:0.95 warn`.
- **[Unit 5.13](5.13-phase-5-acceptance-gate.md)** — added `/digest` (the most recent LHCI enrolment; same pattern).
- **[Unit 6.7](6.7-lighthouseci-talk-enrolment.md)** — added `/problems/<slug>/talk`; precedent for a 13→14 URL bump.
- **[Unit 7.3](7.3-locale-layout-wiring-proof.md)** — created `/en/about` + `/fr/about` (the two SSG pilot routes); both render via `getTranslations`; no client-side interactivity beyond the SiteHeader controls.
- **[ADR-0011 D-A](../adr/0011-i18n-strategy.md)** — `NextIntlClientProvider` ships ~5-8 kB to client when `useTranslations` is called on a client component. The about-page is server-rendered (`getTranslations` from `next-intl/server`); no NextIntlClientProvider serialization actually hits the wire for `/en/about` + `/fr/about`. Confirmed by Unit 7.3's "Per-route chunk 150 B" measurement.

### Scope (this unit)

Two file edits:

1. **`lighthouserc.json`** — add two URLs to the `ci.collect.url` array:
   - `http://localhost:3000/en/about`
   - `http://localhost:3000/fr/about`
   - Insertion order: append to the tail (same convention as Unit 5.13 + Unit 6.7); CI runs all 16 URLs in the order listed.
   - No threshold change; both routes inherit the same `0.95 error` gates as every other URL.
2. **`docs/thinking/7.7-lhci-locale-pilot.md`** (this doc).
3. **`CHANGELOG.md`** — Phase-7 Unit 7.7 entry.

### Scope NOT in this unit

- **No new locale-specific thresholds**. Both pilot routes inherit the canonical project gates. If FR rendering introduces a measurable a11y regression (e.g., `lang="fr"` not properly emitted causing screen-reader misidentification), CI catches it on the first PR. No pre-emptive special-casing.
- **No middleware-redirect URLs**. The toggle-target URLs (`/en/about` from a `/about` hit) aren't separately enrolled — LHCI runs against direct URLs only; redirect chains test under different conditions.
- **No `/en/` or `/fr/` (locale root)** — these 404 today (`app/[locale]/page.tsx` doesn't exist yet; Unit 7.3 didn't add a home page under `[locale]`). LHCI would fail on the SSG-route check. Skipped.
- **No methodology FR pilot URL** — Unit 7.5 is the methodology FR pilot; the parallel-curator session likely owns it. When `/en/methodology` + `/fr/methodology` ship (or `/methodology/[locale]/v1.0.0` shape), a follow-on unit enrols them. Out of scope here.
- **No mobile preset** — `lighthouserc.json` uses `desktop` preset throughout; no mobile-specific gate. Phase-4 DomainMap had a deliberate `< 640px` table fallback (Q39) but no mobile-LHCI gate. Stay consistent.

### Constraints

- **CI behavior preservation**. `lighthouserc.json` is consumed by `@lhci/cli` in CI; adding URLs extends the matrix but doesn't change how existing URLs are evaluated. Backwards-compatible.
- **No `lhci` install change**. Existing dependency at HEAD (`@lhci/cli` per `package.json` `devDependencies`); no version bump.
- **`pnpm build` + `pnpm start` are LHCI's startup invariants** — both URLs are SSG (`/en/about` is statically generated via `app/[locale]/layout.tsx`'s `generateStaticParams`; `/fr/about` the same). Once `pnpm build` produces them, `pnpm start` serves them statically. LHCI's `startServerReadyPattern: "Ready"` matches Next.js's startup banner; no changes needed.
- **Number-of-runs = 3**. Each new URL adds 3 runs to the CI matrix (= 6 additional Lighthouse passes total). Net cost: roughly one extra CI-minute. Acceptable.

### Alternatives considered

- **Enrol only `/fr/about` (skip `/en/about`)** — Rejected. Both locales need the gate. `/en/about` has the same `[locale]` routing pipeline + same NextIntlClientProvider wrapper as `/fr/about`; testing only one would miss EN-side regressions (e.g., a language-tagging bug that swaps `lang="en"` with `lang="fr"`).
- **Enrol `/en/about` and `/fr/about` as separate `assertMatrix` blocks with per-locale thresholds** — Rejected. No data yet to justify per-locale thresholds; both should hit ≥ 95 the same way. If FR-specific regressions emerge, escalate to `assertMatrix`.
- **Defer LHCI enrolment until Unit 7.5 ships more FR content** — Rejected. The pilot routes exist at HEAD and are part of the bilingual surface; not gating them in CI means a regression can land silently. Phase-precedent (Unit 5.13 enrolled `/digest` immediately upon creation) supports immediate enrolment.

### Tradeoffs / defensible defaults flagged

1. **Adding 2 URLs = +6 LHCI runs in CI**. Small cost; matches Unit 6.7's precedent of +3 runs (the talk-page enrolment was `numberOfRuns: 3` × 1 URL).
2. **`/en/about` and `/fr/about` will both render the same EN HTML** until the FR translations land — actually false: Unit 7.3 already shipped `messages/fr.json` with the about-page's FR strings (`title = "À propos"` / `description = "Description du projet, gouvernance et contact. Phase 1."`). So `/fr/about` renders FR content today. LHCI's a11y check will see the FR text + the `lang="en"` on `<html>` (set by root layout) — potential a11y mismatch that LHCI will flag on the first PR. **Documented risk**: if LHCI's a11y rule for `html-has-lang` accounts for content-language mismatch, this could push the FR pilot below 95. The fix (set `lang={locale}` on `<html>` in `app/[locale]/layout.tsx`) is out of scope for this unit (touches the layout) but is the obvious follow-on if CI flags it. Flagging here rather than pre-emptively fixing — it may not regress in practice (axe's lang rule is forgiving of locale-vs-content drift).
3. **Pilot-URL count stays low** — 2 URLs now; expands organically as more `[locale]/...` routes ship. Don't pre-emptively enrol the bulk of `/en/<route>` until Unit 7.3a creates them.
