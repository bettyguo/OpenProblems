### D-3. `CompositeExtensionRegistry` shape anticipated (Unit 40.1)

D-A. **Registry class**:

```ts
export class CompositeExtensionRegistry implements MarkdownExtensionRegistry {
  constructor(private readonly components: ReadonlyArray<MarkdownExtensionRegistry>) {}

  getExtensions(surface: MarkdownSurface): MarkdownExtensionSet {
    const remarkPlugins: Pluggable[] = [];
    const rehypePlugins: Pluggable[] = [];
    let schemaOverrides: Partial<Schema> | undefined;
    let schemaOverridesSource: number | undefined;

    for (let i = 0; i < this.components.length; i++) {
      const set = this.components[i].getExtensions(surface);
      if (set.remarkPlugins) remarkPlugins.push(...set.remarkPlugins);
      if (set.rehypePlugins) rehypePlugins.push(...set.rehypePlugins);
      if (set.schemaOverrides) {
        if (schemaOverrides !== undefined) {
          // Two components both override schema for the same surface.
          // Throw — composition cannot resolve this safely under
          // override-replace semantics (APPEND-D-C).
          throw new Error(
            `CompositeExtensionRegistry: schema override conflict on surface ` +
              `"${surface}": both component ${schemaOverridesSource} and ` +
              `component ${i} provide schemaOverrides; only one may.`,
          );
        }
        schemaOverrides = set.schemaOverrides;
        schemaOverridesSource = i;
      }
    }

    const result: MarkdownExtensionSet = {};
    if (remarkPlugins.length > 0) result.remarkPlugins = remarkPlugins;
    if (rehypePlugins.length > 0) result.rehypePlugins = rehypePlugins;
    if (schemaOverrides !== undefined) result.schemaOverrides = schemaOverrides;
    return result;
  }
}
```

D-B. **Composition rules** (documented APPEND-D-R):

| Slot | Composition semantics | Order |
|---|---|---|
| `remarkPlugins` | **Concatenate** across all components | Component registration order |
| `rehypePlugins` | **Concatenate** across all components | Component registration order |
| `schemaOverrides` | **At most one component per surface**; throw on conflict | N/A |

The schema-override conflict rule is deliberate: under APPEND-
D-C override-replace semantics, two components both providing
`schemaOverrides.tagNames` (or any field) cannot be safely
merged — the framework explicitly rejects deep-merge. Throwing
on conflict is the loud-failure analog to `getModerator()`'s
throw-on-unknown.

For the Phase 38 + 39 consumers + the most likely Phase 41+
third consumer, **conflicts are unlikely**: wikilinks is
rehypePlugins-only; tables is schemaOverrides-only; any
remarkPlugins consumer would also be plugins-only. Only when
two schemaOverrides-using consumers target the same surface
does the conflict arise.

D-C. **Multi-value env-var parsing**: `MARKDOWN_EXTENSIONS=wikilinks,tables`
splits on comma (no whitespace tolerance Phase 40; trim each
value); each value must be a recognized non-default arm
(`"wikilinks"` or `"tables"`); duplicates rejected; empty
values rejected; `"default"` is the single-value-only default
(can't combine with others).
