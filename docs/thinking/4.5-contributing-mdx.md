### Issue-template URL pattern

GitHub's "new issue from template" URL pattern is:
```
https://github.com/<owner>/<repo>/issues/new?template=<filename>.yml
```

Concrete URLs for this project:
- `https://github.com/bettyguo/OpenProblems/issues/new?template=new-problem.yml`
- `https://github.com/bettyguo/OpenProblems/issues/new?template=new-paper.yml`
- `https://github.com/bettyguo/OpenProblems/issues/new?template=leaderboard-entry.yml`
- `https://github.com/bettyguo/OpenProblems/issues/new?template=rating-challenge.yml`

### Alternatives considered

- **Inline the curator workflow verbatim from CURATION_PROMPT.md.** Rejected — CURATION_PROMPT.md is written for Claude (parallel-safety contract, prompt-engineering register). External-curator MDX needs a different voice + scope.
- **Single combined "contributing" + "methodology" doc.** Rejected — different audiences, different update cadences (contributing evolves with the issue templates; methodology evolves with rating dimensions).
- **Skip versioning for v1.** Rejected — methodology is versioned (v1 / v2 / …) per §8.1, and contributing should follow the same convention so Phase-5+ workflow changes don't silently re-grade the contributor pact.
- **Static prose without table formatting**. Considered. Rejected — the 4 contribution types map naturally to a table (Type / When to use / Template), and tables scan faster than prose lists.

### Tradeoffs / defensible defaults flagged

1. **MDX file unreferenced until Unit 4.6 ships.** Acceptable: Velite glob doesn't match; pre-commit hook doesn't gate on MDX outside known collections.
2. **No external screenshots / GIFs.** External-contributor docs sometimes lean visual. Following the MASTER_PROMPT.md text-first convention here; revisit if user-research signals demand visual aids.
3. **No "Code of Conduct" section.** Out of scope; if needed, a separate `content/code-of-conduct/v1.mdx` lands as its own unit. Not Phase-4-blocking.
4. **`supersedes: undefined`** for v1 — methodology v1 has the same pattern. v2 will set `supersedes: 1.0.0`.
