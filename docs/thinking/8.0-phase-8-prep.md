### D-3. Migration strategy for Unit 8.1 (deferred to Unit 8.1 THINK doc)

Two viable approaches for the bulk page migration (22 routes + `middleware.ts`):

1. **Big bang** — single commit migrates all 22 routes + adds `middleware.ts` with `localePrefix: "always"` per ADR-0011 D-B. Mechanically simple. Trust smoke gates (typecheck + tests + build). Risk: any missed call site breaks the build at commit time, surfacing late.
2. **Staged with `localePrefix: "as-needed"` transient intermediate** — 8.1.i ships middleware-only with `as-needed`; 8.1.ii through 8.1.N migrate routes 2-4 at a time per commit; 8.1.final flips to `always` and deletes bare-route shadows. Decomposed risk surface. Requires the intermediate `as-needed` state to be transient (not load-bearing) so no ADR-0011 amendment is needed; documented in Unit 8.1's THINK doc.

**Lean** (subject to Unit 8.1 THINK doc): **Big bang**. Reasoning: 22 routes is mechanically tractable; the smoke gates (typecheck + 383 tests + build) catch missed call sites before commit; the staged approach inflates the unit count by 4-7 (one commit per route batch) without proportional risk reduction. Pin the choice in Unit 8.1 after a per-route inventory.

**Alternative consideration**: if Unit 8.1's per-route inventory surfaces non-trivial call-site breakage (e.g., `lib/digest/build-digest.ts` references hardcoded `/problems/<slug>` URLs that need locale-prefixing), staging may be re-evaluated.
