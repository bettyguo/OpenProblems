# 7.5 — Methodology FR pilot: first end-to-end sibling-file consumer

Fourth code unit of Phase 7. Renumbered from the [Unit 7.0 §F](7.0-phase-7-prep.md) plan (was originally 7.4); Unit 7.3's deviation note swapped the 7.4 ↔ 7.5 order to land Velite plumbing (Unit 7.4) before its first consumer. This unit ships the first end-to-end consumer: an FR translation of `content/methodology/v1.mdx`, a locale-aware loader, locale-aware methodology routes, and a backward-compatible filter on the existing bare routes so they keep serving EN-only after the FR record joins the Velite collection.

## THINK

### Inputs

- **[ADR-0011 D-D](../adr/0011-i18n-strategy.md)** — locale fallback chain: missing FR file falls back to EN canonical with a switch hint. Wired in `lib/i18n/load-localized.ts` so the route can render the hint when `didFallback === true`. The methodology pilot has both EN and FR variants, so the hint UI isn't *exercised* in this unit — but the loader return shape is the contract.
- **[ADR-0011 D-G](../adr/0011-i18n-strategy.md)** — `translation_source: "human" | "machine-assisted"`. The FR pilot is drafted by an LLM (Claude inside Claude Code) with a curator-reviewer (the session author); honest label is `"machine-assisted"`.
- **[Unit 7.4](7.4-velite-sibling-file.md)** — Velite sibling-file plumbing. This unit's FR file flows through the unchanged Velite pipeline; the `methodology` collection now produces 2 records (EN + FR) with the same canonical slug.
- **[Unit 7.3](7.3-locale-layout-wiring-proof.md)** — `app/[locale]/layout.tsx` + the about-page pilot. The methodology routes in this unit follow the same `setRequestLocale(locale)` pattern.
- **`app/methodology/page.tsx`** + **`app/methodology/[version]/page.tsx`** — existing bare routes that must be filtered to `lang === "en"` to preserve current behavior once the FR record enters the collection (otherwise `generateStaticParams` errors on duplicate `v1.0.0` and `methodology.find(...)` returns whichever record sorts first).
- **`content/methodology/v1.mdx`** — the EN source document (132 lines, 7 sections, KaTeX math + tables + markdown links).

### Scope (this unit)

Six file changes:

1. **`lib/i18n/load-localized.ts`** (new) — generic locale resolver:
   - `resolveLocalized<T extends { lang: Locale }>(records, locale, matches)` → `{ record: T; didFallback: boolean } | null`.
   - `matches` is a `(record: T) => boolean` predicate; the consumer-supplied key (e.g., canonical slug equality).
   - Returns the record matching `lang === locale` if any; otherwise the record matching `lang === defaultLocale` and `didFallback: true`; otherwise `null` (no record matches at all → caller calls `notFound()`).
   - Pure function; no next-intl import; reusable across methodology / contributing / problem-pages / problems / papers consumers as `.fr` siblings land.

2. **`lib/i18n/load-localized.test.ts`** (new) — vitest unit tests:
   - Returns the FR record when both EN + FR present and locale = "fr".
   - Returns the EN record + `didFallback: false` when locale = "en" and both present.
   - Returns the EN record + `didFallback: true` when locale = "fr" and only EN present.
   - Returns `null` when no record matches the predicate at all.
   - Works with multiple matching slugs (correctly filters by predicate first, then by lang).

3. **`content/methodology/v1.fr.mdx`** (new) — full FR translation:
   - Frontmatter: `version: "1.0.0"` (matches EN), `title: "Méthodologie de notation v1.0"`, `summary` (FR), `date: 2026-05-14` (matches EN), `translation_source: machine-assisted` (per ADR-0011 D-G).
   - Body: prose-translated EN content; **link targets unchanged** (per ADR-0011 D-E English-canonical slug strategy); KaTeX formulas unchanged (math is locale-neutral); table grades (S/A/B/C/D/E) unchanged; section numbering preserved.
   - All seven sections translated: First principles, Dimensions, Composite, Confidence, Rating actions, COI policy, Methodology versioning.

4. **`app/[locale]/methodology/page.tsx`** (new) — locale-aware index:
   - `generateStaticParams()` returns `locales.map(l => ({ locale: l }))`.
   - `params` → `{ locale }`; validates via `isLocale()`; `setRequestLocale(locale)`.
   - Selects the **latest version per locale** (sorts by version desc; applies `resolveLocalized` against the predicate that matches the latest version's canonical slug at the requested locale).
   - Renders title + summary + version metadata + body via `MDXContent`.
   - Includes "Other versions" nav (links to `/<locale>/methodology/v<version>`).
   - **No fallback-hint UI in this unit** — methodology v1.0 has FR; the loader's `didFallback` is `false`. UI hint deferred to whichever unit ships content that triggers fallback.

5. **`app/[locale]/methodology/[version]/page.tsx`** (new) — locale-aware version page:
   - `params` → `{ locale; version }`; same locale validation + `setRequestLocale`.
   - `generateStaticParams()` returns the cartesian product of locales × distinct versions (currently `{en, fr} × {v1.0.0} = 2 entries`; would scale as more versions / locales land).
   - Resolves locale-aware record via `resolveLocalized`; `notFound()` if neither EN nor FR exists at that version.
   - Renders the same shape as the bare version page, with the version label + body in the requested locale.

6. **`app/methodology/page.tsx`** + **`app/methodology/[version]/page.tsx`** (edits) — backward-compat filter:
   - Both routes add `.filter(m => m.lang === "en")` at the top of the methodology-collection usage.
   - **Why required**: `methodology` now contains 2 records (`v1` EN + `v1` FR); without the filter:
     - `/methodology` would render whichever record sorts first by `(version)` — non-deterministic on ties.
     - `/methodology/[version]`'s `generateStaticParams` would emit `{ version: "v1.0.0" }` twice → Next.js build error: "duplicate path".
     - `methodology.find(m => m.version === requested)` would return whichever record happens to sort first.
   - Filter preserves the bare-route current behavior: EN content at bare paths. Locale-aware versions live under `/[locale]/`.

### Scope NOT in this unit

- **Fallback-hint UI**. The loader returns `didFallback: boolean`; route consumers render the hint when true. Methodology v1.0 has FR — no fallback triggered. Hint UI lands alongside content that triggers it (e.g., a future EN-only problem page or methodology v1.1).
- **Other surfaces' FR translations** (`contributing/v1.mdx` + `contributing/v1.1.mdx` + per-problem MDX + problem.yaml etc.). Defer to curator-track follow-ons per Q51 lean.
- **Middleware** (`localePrefix: "always"` redirect from bare to `/en/`). Still deferred from Unit 7.3a.
- **`scripts/validate-content.ts` extension** for `.fr.mdx` detection. MDX files aren't currently validated by `validate-content.ts` (the script handles YAML + JSON only); methodology is Velite-only. So no extension needed in this unit. Will be needed when `problem.fr.yaml` lands.
- **Locale toggle UI** (Unit 7.6 original). The current state lets users navigate to `/fr/methodology` by URL; in-app toggle ships when 7.6 lands.
- **SEO `<link rel="alternate" hreflang="...">`** (ADR-0011 D-G companion). Could land here cheaply (the route knows the locales); deferring to keep this unit focused on the rendering proof. Tracked.
- **`lighthouserc.json` enrolment** for the new URLs. Existing LHCI cohort runs on the bare routes; locale-aware routes are a new enrolment opportunity. Defer to Unit 7.7 per the §F plan (matches the precedent of 5.13b for `/digest`).

### Constraints

- **`exactOptionalPropertyTypes: true`**. `resolveLocalized` returns either a populated `{ record, didFallback }` object or `null` — never a partial. Route consumers check `if (!resolved) notFound()`.
- **Velite collection shape** (post-Unit 7.4): every methodology record carries `lang: "en" | "fr"`. The FR record also carries `translation_source: "machine-assisted"` (Velite's post-transform `.refine` enforces this on non-EN).
- **ADR-0005 immutability** unaffected. No new rating-action files; the FR methodology is a new MDX, not an existing rating.
- **§5.5 perf budget**. The FR translation adds ~6 KB to `.velite/methodology.json`; routes are SSG; no client-bundle weight. Shared chunk stays 103 kB.
- **Q31 dual-schema contract**. Velite's `methodology` schema (extended in Unit 7.4) accepts the FR frontmatter; no Zod-4 source-of-truth schema for methodology (it's MDX-only, not in `lib/schemas/*`).
- **English-canonical slugs (ADR-0011 D-E)**. `/fr/methodology/v1.0.0` (English-version segment) is correct; **not** `/fr/methodologie/v1.0.0`. The route folder name `methodology/` stays English.
- **`generateStaticParams` cartesian product**. The `[locale]/methodology/[version]/page.tsx` route emits `locales.length × distinct_versions = 2` entries today (`{en, v1.0.0}` and `{fr, v1.0.0}`). Scales linearly as versions / locales grow.

### Alternatives considered

- **Embed the FR translation inline in `messages/fr.json` instead of an `.fr.mdx` sibling** — Rejected. Messages JSON is for short ICU-formatted UI strings; long-form prose with markdown / KaTeX / tables would explode the messages bundle and break next-intl's per-route tree-shaking. ADR-0011 D-C explicitly chose sibling-file storage for content; this unit is the demonstration.
- **Author a partial FR translation (e.g., sections 1-2 only) and use the fallback hint for the rest** — Rejected. ADR-0011 D-D says "No partial-translation rendering — a page either fully uses its locale's content or fully uses the EN fallback." Sibling files are whole-doc or absent.
- **Don't filter the bare routes; let `lang` overlap silently** — Rejected. `generateStaticParams` errors on duplicate `v1.0.0` would break `pnpm build`; the failure mode is loud, not subtle. Filter is non-negotiable when the FR record lands.
- **Move the bare `/methodology` routes under `/en/methodology` now and 308-redirect** — Rejected. That's Unit 7.3a scope (bulk migration + middleware). Out of scope for the FR pilot. Coexistence is the documented intermediate state.
- **Make `resolveLocalized` accept a key function `(record) => string` instead of a predicate** — Rejected for now. Predicate is more flexible (consumer can use any matching logic — version, slug, problem_slug × kind, etc.) and the predicate API is what every Velite consumer in this project already uses (`.find((m) => m.version === requested)`).

### Tradeoffs / defensible defaults flagged

1. **`translation_source: "machine-assisted"` chosen** for the FR draft. I'm an LLM authoring the translation; honest label is "machine-assisted" per ADR-0011 D-G ("curator-reviewed LLM-drafted translation"). A pure-human curator translating from EN would use `"human"`. Curators who edit `v1.fr.mdx` afterwards can flip the field to `"human"` once they've reviewed it line-by-line; this unit ships with the LLM label so the provenance is honest from day zero.
2. **No fallback-hint UI** in this unit. The loader contract is in place (`didFallback: boolean` in the return type); the route doesn't render a hint because methodology v1.0 has its FR sibling — `didFallback` is always `false` here. Hint UI lands when first-triggered content surfaces. Trade-off: a curator visiting `/fr/methodology/v1.0.0` today sees the FR translation with no visible "this is a translation" indicator. Acceptable for the demo; the locale prefix in the URL is the indicator.
3. **English-canonical route folder name `methodology/`** preserved under `/[locale]/`. A French-only reader sees `/fr/methodology/v1.0.0` which is mildly awkward; the alternative (slug aliases) is the larger ADR-0011 D-E rejection.
4. **The locale-aware routes coexist with bare routes** for 3 reasons: (a) Unit 7.3a deferral means no middleware redirect yet; (b) deep links to `/methodology/v1.0.0` from external sites + RSS + earlier digests must not break; (c) the FR pilot validates the routing layer without forcing a 22-file restructure. Trade-off: visitors see 3 paths render the same EN content (`/methodology`, `/en/methodology`, `/methodology/v1.0.0` vs `/en/methodology/v1.0.0`). Documented intermediate state.
5. **Page count delta**: +4 routes (`/en/methodology`, `/fr/methodology`, `/en/methodology/v1.0.0`, `/fr/methodology/v1.0.0`). Was 2 (bare only); now 6. 335 → 339.
6. **Translation provenance honesty**. The FR draft is going to be visible — readers can verify the translation themselves. The `translation_source: "machine-assisted"` field is editorial signal that this isn't a native-speaker editorial pass. A future curator-track unit could re-author with `translation_source: "human"`.
