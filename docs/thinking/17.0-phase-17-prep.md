### D-3. Markdown subset (allowed elements + sanitization rules)

Per ADR-0018 D-? (anticipated):

| Element / attribute | Phase 17 allowed? | Notes |
|---|---|---|
| **Bold** (`**text**` / `__text__`) | YES | Wraps in `<strong>`. |
| **Italic** (`*text*` / `_text_`) | YES | Wraps in `<em>`. |
| **Inline code** (`` `code` ``) | YES | Wraps in `<code>`; no syntax highlighting. |
| **Code blocks** (fenced ```` ``` ````) | YES | Wraps in `<pre><code>`; no syntax highlighting; no language hint enforcement. |
| **Links** (`[text](url)`) | YES | Allowed schemes: `https:` + `mailto:` ONLY; `http:` denied (require TLS); `javascript:` + `data:` + `file:` denied (XSS surface); link-detection on bare URLs YES via `remark-gfm` autolink. |
| **Lists** (ordered + unordered) | YES | Standard `<ol>` + `<ul>` + `<li>`. |
| **Headings** (`# h1` through `###### h6`) | YES but **demoted** — `# h1` rendered as `<h3>` (avoid colliding with profile page's `<h1>` outline). |
| **Blockquotes** (`> text`) | YES | Standard `<blockquote>`. |
| **Horizontal rules** (`---`) | YES | Standard `<hr>`. |
| **Tables** (GFM) | **NO** (Phase 17+) | `remark-gfm` table extension off; Phase 18+ if signal demands. |
| **Footnotes** (GFM) | **NO** (Phase 17+) | Phase 18+ if signal demands. |
| **Strikethrough** (`~~text~~`) | YES via `remark-gfm` | Standard `<del>`. |
| **Task lists** (`- [x] item`) | YES via `remark-gfm` | Standard checkbox; rendered as read-only. |
| **Raw HTML** | **NO** | rehype-sanitize denies all raw HTML; markdown source only. |
| **Class attributes / inline styles** | **NO** | rehype-sanitize strips. |
| **Images** (`![alt](url)`) | **NO** (Phase 17+) | Bio is text-context; image uploads belong to Phase 16's `imageOverride` surface. Q66+1 candidate if signal demands. |

### D-4. Library choice

Per ADR-0018 D-? (anticipated):

**Decision**: `unified@11+` + `remark@15+` + `remark-parse@11+` +
`remark-gfm@4+` + `remark-rehype@11+` + `rehype-sanitize@7+` +
`rehype-stringify@10+`. The **`unified` pipeline** is the standard
markdown processing stack; widely adopted; battle-tested
sanitization (`rehype-sanitize` ships with a conservative default
schema that we'll narrow per D-3).

**Why this stack** (vs alternatives):
- `marked` — single library; fast; but **no built-in
  sanitization** (couples to external sanitizer).
- `markdown-it` — single library; fast; sanitization via plugin
  ecosystem; **less standardized** schema for what's allowed.
- `unified + remark + rehype` — **multi-stage pipeline**;
  sanitization stage is explicit + audit-friendly; community
  schema (`rehype-sanitize/lib/schema.js`) is the de-facto safe
  default; **best for security-conscious project**.
- `react-markdown` — wraps `unified` for React; couples render to
  JSX; for SSR'd `/u/{handle}` + `/profile` we can use the
  underlying `unified` pipeline directly and render to HTML
  string.

**Bundle impact**: ~120 kB transitive (server-side only; markdown
renders at SSR time; rendered HTML in response body; no client-
side markdown work). First Load JS shared chunk: expected
**UNCHANGED** at 103 kB.

### D-6. Server vs client rendering

Per ADR-0018 D-? (anticipated):

**Decision**: **server-side rendering ONLY** (in the SSR render
pass for `/u/{handle}` + `/profile`). NOT client-side.

- The `unified` pipeline runs inside the async server component;
  bio markdown rendered to HTML string at request time; HTML
  inlined via `dangerouslySetInnerHTML`.
- No client-side markdown processing; no `react-markdown`
  rendering; no client-side `unified` invocation.
- **Why**: minimizes bundle size; sanitization runs in trusted
  server context; no race between client + server render.
- **First Load JS unchanged at 103 kB** (markdown deps are
  server-only).

### D-7. Validation + sanitization model

Per ADR-0018 D-? (anticipated):

1. **`renderBioMarkdown(bio)` helper** in `lib/markdown/index.ts`
   — wraps the `unified` pipeline with the Phase-17 sanitization
   schema (per D-3). Returns sanitized HTML string. Idempotent +
   side-effect-free.
2. **Sanitization schema** (`lib/markdown/sanitize-schema.ts`) —
   exports a `rehype-sanitize` schema narrowed per D-3's
   allow-list. ADR-0018 documents.
3. **URL scheme allow-list** — `https:` + `mailto:` ONLY.
   Defense-in-depth: bare URLs in markdown autolinks pass through
   the same sanitization.
4. **No regex content filter on bio** — Q68 (content moderation)
   stays deferred. Phase 17 trusts user content; XSS surface
   minimized by sanitization.
5. **Null bio** — `renderBioMarkdown(null)` returns null; caller
   omits the section (Phase-15 D-F empty-state behavior
   preserved).
