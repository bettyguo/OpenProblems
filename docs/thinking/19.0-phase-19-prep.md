### D-3. EXIF stripping scope

Per ADR-0019 D-? (anticipated):

| EXIF concern | Phase 19 stripped? | Notes |
|---|---|---|
| **GPS coordinates** (`GPSLatitude` / `GPSLongitude` / `GPSAltitude`) | **YES** | Primary privacy concern; mobile camera defaults to embedding GPS. |
| **Camera serial / model** | **YES** | Device fingerprinting surface. |
| **Datetime metadata** (`DateTimeOriginal` / `DateTime`) | **YES** | Schedule / location-history inference surface. |
| **Software signatures** (`Software` / `ProcessingSoftware`) | **YES** | OS / app fingerprinting. |
| **Author / Copyright tags** | **YES** | PII surface (user may have set Lightroom / Photoshop name). |
| **Comment / Description tags** | **YES** | Free-text PII surface. |
| **Color profile** (ICC) | **NO (preserved)** | Image rendering integrity. |
| **Image dimensions** | **NO (preserved)** | Required for layout. |
| **Compression / quality settings** | **NO (preserved)** | Required for re-encoding. |

**Approach**: strip ALL EXIF metadata except color profile +
dimensions + encoding settings. **Allow-list approach inverted**
— `sharp`'s default `.rotate()` removes all EXIF metadata; we
opt in to preserving color profile via `.withMetadata({ icc:
true })` if observation shows color shift on common image
profiles. Phase-19 conservative default: strip everything.

### D-4. Library choice (ADR-0019 keystone)

**Decision**: `sharp@0.34+` (Node.js image processing library;
widely-vetted; server-side).

**Rationale**:
- **Battle-tested**: `sharp` is the de-facto Node.js image
  processing standard (used by Vercel Image Optimization, Next.js
  `next/image`, Cloudinary, etc.).
- **Native libvips bindings**: 10x faster than `gm`/`imagemagick`;
  matches Vercel serverless cold-start budget.
- **EXIF stripping default**: `sharp`'s default behavior strips
  EXIF on `.toBuffer()` unless `.withMetadata()` explicitly opts
  in. Inverted-allow-list shape.
- **Server-only**: no client bundle impact; First Load JS
  UNCHANGED at 103 kB per ADR-0018 D-F invariant extension.
- **Already a transitive dep** of `next` (via `next/image`); may
  not require explicit install — TBD verify Unit 19.1.
- **Re-encoding support**: same library handles future Phase 20+
  resizing / format conversion / WebP transcoding if Q68
  expansion or cropping UI ships.

**Alternative options** (rejected):
- **`exifreader` / `exiftool` Node bindings**: read-only or
  external-binary; less efficient at re-encoding.
- **Browser-side stripping** (e.g., `piexifjs`): adds client bundle
  (~10 kB); user-tampering surface (trust boundary moves to
  client); regresses 103 kB invariant.
- **Vercel Image Optimization API**: read-side optimization only;
  doesn't strip EXIF on upload-path images.
- **External transcoding service** (Cloudinary / Imgix): heaviest;
  needs additional API key + per-image cost; over-engineered for
  Phase 19 scope.

### D-5. Pipeline placement

Per ADR-0019 D-? (anticipated):

**Decision**: EXIF stripping happens **server-side in
`lib/storage/putAvatar`** BEFORE Vercel Blob upload. Approach:
sharp reads buffer → re-encodes WITHOUT EXIF → uploads stripped
buffer to Vercel Blob.

```ts
import sharp from "sharp";

export async function putAvatar(file: File, userId: string): Promise<string> {
  const inputBuffer = Buffer.from(await file.arrayBuffer());
  const strippedBuffer = await sharp(inputBuffer)
    .rotate()                                          // auto-rotate based on EXIF Orientation BEFORE strip
    .toBuffer();                                       // re-encode without EXIF (sharp default)
  // upload strippedBuffer to Vercel Blob as before
}
```

**Auto-rotation rationale** (`.rotate()` with no args): EXIF
Orientation tag tells the image how it should be displayed; iOS
cameras default to landscape sensor orientation with portrait
display Orientation metadata. If we strip EXIF without first
rotating, the image renders sideways. `.rotate()` with no args
reads the Orientation tag, applies the rotation to pixel data,
then resets Orientation to 1 (default) — preserves visual
correctness across all clients.
