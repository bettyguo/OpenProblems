- **First "regex-extension on a most-sophisticated-prior-regex
  consumer"**: pre-Phase-48 the doi plugin already had the
  framework's most-complex regex — `\bdoi:(10\.\d{4,9}\/[-._;()/:A-Za-z0-9]+?)(?=[\s,;)]|\.(?:\s|$)|$)/gi`
  — with the period-in-suffix disambiguation lookahead added
  Phase 45. Phase 48 extends it with bracketed alternation while
  preserving the lookahead semantics on the bare alternative.
  Maximum-prior-regex-complexity validation of the dual-form
  pattern.
- **Bracket-wrapping disambiguates display terminator + selectively
  drops lookahead constraints**: per APPEND-D-AC item 2's
  deferral text, the alias syntax is
  `[[doi:10.NNNN/xxx|display]]` — wrapped in wikilinks-style
  `[[ ]]` brackets. The brackets serve dual roles: (a) explicit
  `|display` terminator (same as Phase-47 arxiv alias rationale);
  (b) explicit `]]` suffix terminator that obviates the bare-
  form's prose-friendly lookahead. Inside brackets, the full
  Crossref DOI character class `[-._;()/:A-Za-z0-9]` is
  permissive (`;`, `(`, `)`, `.` all allowed) because `]]` is
  the explicit terminator. **Second dual-form regex** in the
  framework — alternation between bracketed (priority) and bare
  (fallback). Backwards-compatible: every existing bare
  `doi:10.NNNN/xxx` match preserved via the second alternation
  arm verbatim.
- **No collision with wikilinks plugin**: wikilinks regex
  `\[\[([a-z0-9-]+)(?:\|([^\]\n]+))?\]\]` requires slug to be
  pure kebab-case `[a-z0-9-]+`. The doi ID inside
  `[[doi:10.NNNN/xxx]]` contains `:`, `.`, and `/` — all
  excluded from wikilinks' slug class. Wikilinks plugin would
  NOT match `[[doi:10.1234/abc]]`. Distinct regexes → no
  collision. AND wikilinks runs in `rehypePlugins` stage AFTER
  doi runs in `remarkPlugins` stage; even if there were a regex
  collision, the staged execution order would resolve it in
  favor of doi. **No regex-ambiguity audit required.**
- **No collision with arxiv plugin** (the same-slot sibling
  consumer Phase 45): arxiv ID class `\d{4}\.\d{4,5}` requires
  4-digit prefix + `.` + 4-5-digit suffix; doi ID class
  `10\.\d{4,9}\/...` requires `10.` literal prefix + 4-9-digit
  registrant + `/` + suffix. The two regexes literally cannot
  match the same string (arxiv lacks the `/`; doi requires
  4-9-digit registrant where arxiv requires 4-digit prefix
  exactly + `.` not `/`). Same `remarkPlugins` slot; distinct
  regex character classes; collision-free via regex-disjointness
  alone (the staged-execution layer is moot for same-slot pair).
  Establishes the **regex-disjointness-as-sole-defense
  discipline** for same-slot composition.

### D-3. DOI alias regex shape anticipated (Unit 48.1)

D-A. **`DOI_PATTERN` regex evolution** (in-place edit of
`lib/markdown/extensions/doi.ts`):

```ts
// Before (Phase 45 ship through Phase-47 close):
const DOI_PATTERN =
  /\bdoi:(10\.\d{4,9}\/[-._;()/:A-Za-z0-9]+?)(?=[\s,;)]|\.(?:\s|$)|$)/gi;

// After (Phase 48 ship):
const DOI_PATTERN =
  /\[\[doi:(10\.\d{4,9}\/[-._;()/:A-Za-z0-9]+?)(?:\|([^\]\n]+))?\]\]|\bdoi:(10\.\d{4,9}\/[-._;()/:A-Za-z0-9]+?)(?=[\s,;)]|\.(?:\s|$)|$)/gi;
```

**Second dual-form regex** in the framework: alternation between
the bracketed form (priority) and the bare form (fallback).
Engine tries the bracketed alternative first; if it fails at a
given position, tries the bare alternative.

- **Bracketed form** `\[\[doi:(10\.\d{4,9}\/[-._;()/:A-Za-z0-9]+?)(?:\|([^\]\n]+))?\]\]`:
  - Group 1 = DOI ID (4-9-digit registrant + `/` + Crossref
    suffix class; same character class as Phase-45 baseline).
  - Group 2 = optional display text (`[^\]\n]+`; same shape as
    Phase-46 wikilinks alias + Phase-47 arxiv alias).
  - **No trailing lookahead** — `]]` is the explicit terminator.
    The full Crossref suffix character class is permissive
    inside brackets (`;`, `(`, `)`, `.` all allowed); the bare-
    form's prose-friendly truncation does NOT apply.
- **Bare form** `\bdoi:(10\.\d{4,9}\/[-._;()/:A-Za-z0-9]+?)(?=[\s,;)]|\.(?:\s|$)|$)`:
  - Group 3 = DOI ID (Phase-45 baseline preserved verbatim).
  - Trailing lookahead `(?=[\s,;)]|\.(?:\s|$)|$)` PRESERVED —
    prose-friendly truncation of trailing punctuation
    discipline is the Phase-45 ship tradeoff unchanged.
  - No alias support in bare form (alias requires bracket-
    wrapping per APPEND-D-AC item 2 deferral text).

D-B. **`remarkLinkDoiIds` plugin body update**:

```ts
const matched = match[0];
const isBracketed = matched.startsWith("[[");

const id = isBracketed ? match[1] : match[3];
const alias = match[2]; // only defined for bracketed form
if (id === undefined) continue;

let display: string;
if (alias !== undefined) {
  display = alias;
} else if (isBracketed) {
  // Bracketed without alias: drop brackets while preserving
  // source casing (e.g., `[[DOI:10.1234/abc]]` →
  // `<a>DOI:10.1234/abc</a>`).
  display = matched.slice(2, -2);
} else {
  // Bare form (Phase-45 baseline): display source casing
  // verbatim (e.g., `Doi:10.1234/abc.def`).
  display = matched;
}

newNodes.push({
  type: "link",
  url: `https://doi.org/${id}`,
  children: [{ type: "text", value: display }],
});
```

Mirrors Phase-47 `remarkLinkArxivIds` body shape verbatim with
`arxiv:` → `doi:` substitution + Phase-45 `https://doi.org/${id}`
URL construction preserved + version-suffix branch removed (doi
IDs have no version-suffix concept; the Phase-47 arxiv body's
`version` handling is doi-irrelevant).
