### Decisions pinned (per Unit 17.0 D-8 anticipated shape)

**D-A. Library choice.** `unified@11+` + `remark-parse@11+` +
`remark-gfm@4+` + `remark-rehype@11+` + `rehype-sanitize@7+` +
`rehype-stringify@10+`. Multi-stage pipeline; explicit
sanitization boundary. ~120 kB transitive (server-only).
Pipeline order: parse → GFM transforms → HAST conversion (with
`allowDangerousHtml: false`) → sanitize → stringify.

**D-B. Allowed markdown subset.** Bold / italic / inline-code /
fenced-code-blocks (no syntax highlighting) / links / bare-URL
autolinks / unordered + ordered + task lists / headings (demoted
per D-C) / blockquotes / horizontal-rules / GFM strikethrough /
paragraphs. EXCLUDES: tables (Phase 18+) / footnotes (Phase 18+)
/ images (Phase 18+) / raw HTML / class attributes / inline
styles / event handlers / `<iframe>` / `<object>` / `<embed>` /
`<script>` / `<style>` / `<base>` / `<a target>` / `<a rel>` /
`<a title>`. Length cap UNCHANGED at 280 chars.

**D-C. Heading demotion.** Source `#` (h1) → `<h3>`; `##` →
`<h4>`; `###` → `<h5>`; `####+` → `<h6>` (max). Page-level `<h1>`
is user display name; `<h2>` is section heading; bio nests within
page outline. Accessibility-tool semantic order preserved.

**D-D. URL scheme allow-list.** `https:` + `mailto:` ONLY.
`http:` / `javascript:` / `data:` / `file:` / `vbscript:` /
relative URLs all DENIED. Defense-in-depth: bare-URL GFM
autolinks pass through the same `protocols.href` filter.

**D-E. Render path placement.** New `lib/markdown/` module —
`renderBioMarkdown(text: string | null): string | null` returns
sanitized HTML string; null in / null out. Two consumer
integration points: `/[locale]/u/[handle]/page.tsx` bio section
(replace `<p whitespace-pre-wrap>` with `<div
dangerouslySetInnerHTML>`); `/[locale]/profile/page.tsx`
read-mode preview (same swap). NOT in AuthControl pill / NOT in
`/u/{handle}/challenges` / NOT in SiteHeader.

**D-F. Server-side only.** Pipeline runs in async server component
render pass at request time; rendered HTML inlined via
`dangerouslySetInnerHTML`. NO client-side markdown processing.
**First Load JS shared chunk MUST stay UNCHANGED at 103 kB**
end-to-end through every Phase 9-17 unit. No caching Phase 17;
~280-char bio renders cheap; `force-dynamic` re-renders on each
request.

**D-G. Couples to markdown-rendered curator review notes.** Phase
17 ships **bio-specific** helper `renderBioMarkdown`. Phase-18+
markdown surfaces (curator review notes per Phase-15 Class B B.2
item 5; possibly rating-action `rationale`) inherit via sibling
helpers (`renderReviewNotesMarkdown`, etc.) — each with its own
sanitization schema (`reviewNotesSchema`, `rationaleSchema`).
Schema-per-surface keeps audit boundary explicit + customization
local. The shared base schema is named `bioSchema` (NOT
`defaultSchema`) to signal Phase-17 scope.

**D-H. Phase 18+ deferrals.** Tables / footnotes / images /
syntax-highlighting / bio length expansion / markdown reviewNotes
inheritance / markdown rating-action rationale / `@mentions` /
task-list interactivity / live preview / `<a target>` / `<a rel>`
/ `<a title>` / Tel + FTP URL schemes / per-user schema
configurability / DOMPurify defense-in-depth on client (all
deferred to Phase 18+ if signal demands).

### Why an ADR at all?

Three reasons:

1. **First markdown rendering pipeline + first XSS-audit surface
   + first `dangerouslySetInnerHTML` surface in project history**.
   Three architectural firsts converging in one unit. Pinning the
   library choice + sanitization subset + URL allow-list +
   render path + server-vs-client boundary + heading demotion +
   Phase-18+ deferrals in ONE document prevents downstream drift
   when Phase 18+ markdown surfaces land (curator review notes;
   rating-action rationale; possibly methodology-page markdown).
2. **Q66 promotion was explicit Phase 16+ deferral** (ADR-0016
   D-F): *"NO markdown / NO HTML / NO link-detection Phase 15.
   Q66 candidate flagged for Phase 16+."* Phase 17 is the
   deferral-honor; not landing an ADR would be a documentation
   regression.
3. **XSS audit needs an audit-friendly boundary**. The
   `dangerouslySetInnerHTML` surface is the XSS-attack-vector by
   name; the sanitization layer must be documented + verifiable +
   referenceable from Unit 17.2's ~10-vector XSS test suite.
   ADR-0018 is that document.

Contrast Phase 13: Q58 promoted to resolved WITHOUT new ADR
because the lean was documented across three prior units (11.6 +
12.0 D-10 + 13.0 D-3). Phase 17's markdown-rendering surface
spans more architectural ground (library choice + sanitization
+ URL allow-list + outline preservation + bundle invariant +
Phase-18+ inheritance contract); ADR-worthy.

### Options + chosen rationale (mirrors ADR body)

Four options considered:

1. **`unified` + `remark` + `rehype-sanitize` pipeline (CHOSEN)**.
   De-facto standard markdown stack; community-audited
   sanitization; explicit multi-stage pipeline boundary;
   server-only deps; preserves 103 kB invariant; reversible
   Phase-18+ extensions via schema additions.
2. **`marked` library**. REJECTED. No built-in sanitization;
   couples to external sanitizer (`DOMPurify` / `sanitize-html`)
   with bespoke config; less standardized audit surface.
3. **`markdown-it` library**. REJECTED. Sanitization via plugin
   is less standardized; multiple sanitizer plugins with
   different allowlist shapes; less community auditing than
   rehype-sanitize.
4. **`react-markdown` component**. REJECTED. Adds ~50 kB to
   client bundle; First Load JS would regress 103 → ~150 kB
   (violates §10.4 perf budget invariant carried through every
   Phase 9-16 unit); requires `"use client"` boundary; defeats
   SSR optimization.

### Test surface (for downstream Unit 17.2)

This unit is docs-only — no code shipped; no tests run. Unit
17.2's test surface is anticipated:

- `renderBioMarkdown("**bold**")` → `<p><strong>bold</strong></p>`.
- `renderBioMarkdown("\`code\`")` → `<p><code>code</code></p>`.
- `renderBioMarkdown("```\ncode block\n```")` → `<pre><code>code
  block\n</code></pre>`.
- `renderBioMarkdown("[link](https://example.com)")` →
  `<p><a href="https://example.com">link</a></p>`.
- `renderBioMarkdown("[bad](javascript:alert(1))")` →
  href stripped to nothing (or `<a>bad</a>`).
- `renderBioMarkdown("[bad](data:text/html,xss)")` → href stripped.
- `renderBioMarkdown("[bad](http://insecure.example)")` → href
  stripped (require https).
- `renderBioMarkdown("[bad](/local/path)")` → href stripped
  (require absolute).
- `renderBioMarkdown("<script>alert(1)</script>")` → escaped or
  stripped (no raw HTML pass-through).
- `renderBioMarkdown("<img onerror=\"alert(1)\" src=x>")` → stripped.
- `renderBioMarkdown("<iframe src=evil></iframe>")` → stripped.
- `renderBioMarkdown("<style>body{display:none}</style>")` →
  stripped.
- `renderBioMarkdown("# Heading")` → `<h3>Heading</h3>` (per
  D-C demotion).
- `renderBioMarkdown("## H2")` → `<h4>H2</h4>`.
- `renderBioMarkdown("- a\n- b")` → `<ul><li>a</li><li>b</li></ul>`.
- `renderBioMarkdown("- [x] done")` → checkbox rendered read-only.
- `renderBioMarkdown("~~strike~~")` → `<del>strike</del>`.
- `renderBioMarkdown("plain text only")` → `<p>plain text only</p>`
  (backward-compat Phase-15 path).
- `renderBioMarkdown(null)` → null.
- `renderBioMarkdown("")` → null or empty (decided code-time).
- Title attribute removal: `[link](url "title")` → `<a
  href="url">link</a>` (no `title` attr).

~15-20 new tests anticipated, organized into:
- ~6 "happy path" formatting tests.
- ~10 XSS-vector defense tests.
- ~3 outline / heading-demotion tests.
- ~2 null / empty edge cases.
