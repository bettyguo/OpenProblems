## THINK

### Inputs

- **Problem**: [`hallucination-reduction`](../../content/problems/hallucination-reduction/problem.yaml).
  Declares 4 benchmarks: `halueval`, `truthfulqa-2026`, `simpleqa`,
  `facts-grounding`.
- **Contributing papers** (per Unit 2.4, batch 1):
  - [2109.07958](../../content/papers/2109.07958.yaml) — TruthfulQA
    (Lin, Hilton, Evans; ACL 2022). Contributes to `truthfulqa-2026`.
  - [2305.11747](../../content/papers/2305.11747.yaml) — HaluEval
    (Li et al.; EMNLP 2023). Contributes to `halueval`.
  - [2411.04368](../../content/papers/2411.04368.yaml) — SimpleQA
    (Wei et al.; OpenAI 2024). Contributes to `simpleqa`.
  - [2305.14251](../../content/papers/2305.14251.yaml) — FActScore
    (Min et al.; EMNLP 2023). Contributes to `hallucination-reduction`
    generally (no specific benchmark_id).
- **LeaderboardEntrySchema** (`lib/schemas/entry.ts`): `paper_id`,
  `benchmark_id`, `score` (number), `date` (ISO YYYY-MM-DD), `verified`
  (boolean), optional `protocol_notes` (string).
- **Verified-flag semantics**: per
  [`app/problems/[slug]/leaderboard/page.tsx`](../../app/problems/[slug]/leaderboard/page.tsx)
  L62-64 — "Verified entries have been replicated or have explicit
  protocol notes; unverified entries are author-claimed and pending
  review." OR is inclusive: explicit protocol notes alone qualify.

### Sourcing decision per benchmark

#### `simpleqa` ✓ landed

- **Primary source**: [openai/simple-evals](https://github.com/openai/simple-evals)
  README, fetched this session. OpenAI maintains this as the canonical
  evaluation harness for the SimpleQA benchmark they introduced in
  paper 2411.04368. The repo's results table is the rolling reference
  for SimpleQA F-scores.
- **Models selected** (3 of 18 reported, for range coverage):
  - `gpt-4o-2024-08-06` → **40.1** — well-known multimodal flagship,
    Aug 2024 snapshot
  - `o1` → **42.6** — flagship reasoning model, public availability
    Dec 5 2024
  - `gpt-4.5-preview-2025-02-27` → **62.5** — current top scorer in
    the simple-evals table, Feb 2025 preview snapshot
- **Score scale**: percentages (0–100), preserving OpenAI's reported
  format verbatim. The problem.yaml benchmark declaration doesn't
  pin a scale; preserving primary-source format is the §15.6
  defensible default.
- **Date semantics**: model snapshot date (when that specific model
  weights/version became publicly available). For `gpt-4o-2024-08-06`
  and `gpt-4.5-preview-2025-02-27` the date is embedded in the model
  name; for `o1`, the date is OpenAI's public-availability date
  (Dec 5, 2024).
- **`verified: true`**: each entry has explicit `protocol_notes`
  pointing to the source repo, qualifying per the leaderboard page's
  inclusive-OR definition. The scores are not independently
  replicated — they are OpenAI-attested — but the protocol-notes
  branch of the definition is met.
- **`paper_id: "2411.04368"`**: the SimpleQA paper. Semantic note:
  these scores come from OpenAI's separately-maintained eval harness,
  not the paper's tables. The schema has no `model_name` field and no
  way to distinguish "paper that defined the benchmark" from "paper
  that reports the score". Future schema refinement may split this;
  for now `paper_id` follows the convention that it identifies the
  benchmark-defining paper, with `protocol_notes` carrying source
  attribution.

#### `truthfulqa-2026` — deferred

- TruthfulQA paper (2109.07958) abstract reports "best model was
  truthful on 58% of questions". This is the simple-truthfulness
  metric, NOT the benchmark's declared `truthful+informative` metric
  (which is a stricter conjunction). Minting an entry with score 0.58
  on the `truthful+informative` benchmark would be metric-mismatched.
- The headline `truthful+informative` figures (GPT-3 175B ≈ 21%,
  per the paper's body) are in the paper's tables, not the abstract.
  WebFetch on `https://arxiv.org/abs/2109.07958` returned the abstract
  only; full-PDF access is needed for verifiable table reads.
- Additionally, the benchmark is `truthfulqa-2026` (a hypothetical
  2026 refresh per `problem.yaml`'s `notes`), distinct from the
  original 2021 TruthfulQA. No 2026 refresh paper exists in the repo.
  Whether the original 2021 TruthfulQA scores migrate to the
  `truthfulqa-2026` benchmark slot is a curatorial decision; deferring
  is the §15.6-defensible default.

#### `halueval` — deferred

- HaluEval paper (2305.11747) abstract is methodological only — no
  baseline numbers in the abstract. Headline ChatGPT-on-HaluEval-QA
  figures are in the paper's tables (commonly cited around 62-75%
  accuracy depending on subset, but I do NOT have a verified
  primary-source fetch of the specific table number this session).
- WebFetch did not return enough table content for a §15.6-grade
  attestation. Full-PDF access or an alternative primary source
  (e.g., Papers With Code leaderboard, the HaluEval GitHub repo's
  README) would unblock this.

#### `facts-grounding` — empty by design

- FACTS Grounding is a 2024 DeepMind benchmark, NOT one of the 4
  papers in the hallucination-reduction problem. The benchmark is
  declared on `problem.yaml` as a forward-looking entry the problem
  *should* track, but no contributing paper has been added to the
  repo yet. A future paper-ingest commit that adds the FACTS
  Grounding paper (`arxiv.org/abs/2501.03200` per the DeepMind
  release) would attach this benchmark its first entries.

### Constraints

- **§15.6 — no fabricated scores.** Every score in the entries.json
  file must trace to a primary source. The 3 SimpleQA scores are
  from OpenAI's official simple-evals repo; no other entries are
  landed.
- **`exactOptionalPropertyTypes: true`** in tsconfig. `protocol_notes`
  is required on every entry (not omitted), because each is set to
  a real string — no key-vs-undefined ambiguity arises.
- **Date format**: schema requires `YYYY-MM-DD`. All 3 dates comply.
- **JSON, not YAML**: `content/problems/<slug>/entries.json` is JSON
  per the load-entries.ts loader (`JSON.parse`). No YAML.
- **Cross-link audit must stay green.** No paper_id refers to a slug
  outside `content/papers/`; benchmark_id matches the declared
  `simpleqa` benchmark in `problem.yaml`. Audit's
  `entries-contributions-agreement` (mentioned in CHANGELOG but not
  yet implemented as a hard check per `scripts/cross-link-audit.ts`
  inspection this session) is satisfied: paper 2411.04368's
  `contributions[]` declares `benchmark_id: simpleqa`, and the
  entries here reference the same.
- **No collisions with parallel session.** Parallel session is
  working on `content/papers/2306.15794.yaml` (Unit 2.6e). This
  unit creates `content/problems/hallucination-reduction/entries.json` —
  a different directory entirely.

### Alternatives considered

1. **Mint all 18 SimpleQA model scores from simple-evals.** Rejected
   — too many entries for the leaderboard page's table to render
   without scrolling, and the value-add tapers (the top 3 + 1 mid-tier
   span the score range adequately). Trim to 3 for readability.
2. **Mint scores from the SimpleQA paper's own tables (instead of
   the rolling simple-evals repo).** Rejected — WebFetch on the
   arXiv abstract page returned only the abstract; the paper's
   tables aren't accessible without full-PDF retrieval, which
   WebFetch handles inconsistently. The simple-evals repo is
   primary-source-grade for the same authors' organization.
3. **Skip entries.json entirely (preserve Unit 2.9's empty-by-default
   pattern).** Rejected — the user explicitly picked entries.json
   curation as the next unit; shipping at least one verified entry
   set is the deliverable. Empty across all 4 benchmarks would defer
   the §13 deliverable indefinitely.
4. **Mint TruthfulQA / HaluEval entries from abstract-only data with
   `verified: false`.** Rejected — `verified: false` doesn't lower
   the §15.6 bar; it only encodes "pending review". An unverified
   entry still needs an attestable source. Metric mismatches
   (TruthfulQA abstract reports "truthful" but the declared metric
   is "truthful+informative") can't be papered over with
   `verified: false`.
5. **Add a 4th SimpleQA entry for `o1-mini` as a low-end anchor.**
   Considered but rejected — three entries (40, 42.6, 62.5) span the
   score range adequately for a first-pass leaderboard. The
   `simple-evals` repo's full table can be ingested in a future
   leaderboard-ingest unit if the schema gets a `model_name` field.

### Surfaced findings (for future curators)

1. **The other 3 benchmarks' entries are blocked on full-PDF access.**
   For `halueval` and `truthfulqa-2026`, the rate-limit-tolerant path
   is to ingest the GitHub repo's leaderboard (HaluEval has
   [`RUCAIBox/HaluEval`](https://github.com/RUCAIBox/HaluEval); the
   TruthfulQA repo is
   [`sylinrl/TruthfulQA`](https://github.com/sylinrl/TruthfulQA))
   instead of the paper PDFs.
2. **`truthfulqa-2026` is a hypothetical 2026 refresh.** The
   `problem.yaml` `notes` describe a deduplicated 2026 refresh
   distinct from the 2021 TruthfulQA. No 2026 refresh paper exists
   in the repo. A curatorial decision is needed: either retire the
   `-2026` suffix and back-attach the original-TruthfulQA scores
   (with a curator-attestable mapping), or add a real 2026 refresh
   paper if/when one is published.
3. **`facts-grounding` benchmark has no contributing paper.** A
   future paper-ingest unit can attach `arxiv.org/abs/2501.03200`
   (DeepMind, 2025) or similar.
4. **Schema gap**: `LeaderboardEntrySchema` lacks `model_name` and
   `score_scale` (percentage vs decimal vs other). Phase-3 work
   could split or extend.
