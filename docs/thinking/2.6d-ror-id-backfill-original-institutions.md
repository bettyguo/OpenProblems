### Constraints

- **7 YAML edits**: append `ror_id:` after `homepage:`, matching the
  schema field order. Don't reorder existing fields.
- **`meta-fair` and `microsoft-research` exceptions**. Their parent
  corporations (Meta, Microsoft) DO have ROR IDs, but the slug names
  refer to the research divisions, which ROR doesn't separately
  register. Attaching a parent-corporate ROR to a research-division
  slug is misattribution. Per the InstaDeep precedent in Unit 2.6c
  (omit when no faithful authoritative record exists), omit. If a
  future schema admits a `parent_ror_id` field for cases like this,
  re-visit.
- **No collisions.** This unit only touches `content/institutions/`.
  The parallel curator session is working on **Unit 2.5b** (author
  backfill for batch 2 papers — THINK doc exists at
  `docs/thinking/2.5b-author-backfill.md`, untracked). Author work
  touches `content/authors/` and `content/papers/*.yaml`; no overlap
  with this unit.

### Alternatives considered

- **Attach parent ROR (Meta, Microsoft) to the research-division
  slugs.** Rejected — `ror_id` semantically identifies "this
  organization's ROR record". Using Meta's ROR for Meta FAIR is a
  category error that future tooling (anything that walks the ROR
  graph) would treat as authoritative and propagate the mistake.
- **Add a separate `parent_ror_id` schema field for these two cases.**
  Rejected — schema change is out of scope; this is a metadata
  backfill unit. If the broader curatorial need persists, a separate
  ADR + schema unit can introduce the new field.
- **Re-curate the institution slugs.** E.g. fold `meta-fair` into a
  `meta` slug pointing at the parent. Rejected — the research
  division distinction is semantically valuable for paper attribution
  ("paper X is from Meta's research arm, not from Meta's product or
  ads orgs"). Keep the slugs; live with the missing ROR.

### Tradeoffs / defensible defaults flagged

1. **2 of 9 institutions omit `ror_id`.** Documented above. Pattern
   matches Unit 2.6c's InstaDeep omission.
2. **No collateral changes.** Build surface, route count, bundle
   envelope unaffected — same as Unit 2.6c. This is pure metadata.
3. **Combined with Unit 2.6c**, 11 of 14 institutions now carry
   `ror_id` (3 omitted: `instadeep`, `meta-fair`, `microsoft-research`).
   The 3 omissions form a clean, documented exception set.
