### D-3. ORCID consumer shape anticipated (Unit 54.1)

D-A. **ORCID regex pattern**:

```ts
const ORCID_PATTERN = /\borcid:(\d{4}-\d{4}-\d{4}-\d{3}[\dX])\b/gi;
```

- **Prefix** `orcid:` (single canonical prefix; no alternation
  unlike pubmed's `pubmed|pmid`).
- **Identifier class** `\d{4}-\d{4}-\d{4}-\d{3}[\dX]` matches
  the canonical ORCID format: 16 characters total, 4 groups of
  4 digits separated by hyphens, last character is a digit or
  uppercase `X` (the ISO/IEC 7064 MOD 11-2 checksum). E.g.,
  `0000-0002-1825-0097` or `0000-0001-5109-3700` or
  `0000-0002-9079-593X`.
- **Word-boundary anchors** `\b...\b` prevent mid-word matches.
  No trailing-lookahead constraint: the trailing character is
  `[\dX]` (word char), and `\b` triggers at non-word boundary.
- **Case-insensitive flag** `i` allows mixed-case prefix
  (`ORCID:`, `Orcid:`); the checksum `X` matches via `/i` →
  `[xX]` which is fine because ORCID checksum convention is
  uppercase `X` but case-insensitive matches are acceptable.

**Emitted host**: canonical `https://orcid.org/${id}` (no
trailing slash; matches official ORCID URL form).
