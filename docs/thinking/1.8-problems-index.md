# Unit 1.8 — Problems index `/problems` with client-side filter + sort

> Stage 1 (THINK) artifact.

## Problem restatement

§9 spec: "All problems, filterable, sortable." A scannable table of every problem, with filter chips for domain / status / tag and sort options (title, last-curated, composite). The composite is the §8.3 advisory weighted scalar — "sorting on Composite must always show the underlying dimensions" (§8.3) — so the table always renders the dimension breakdown alongside.

## Inputs

- §9 IA — `/problems` route.
- §8.3 — Composite formula; §8.3 last sentence: "The composite is never displayed alone." Show the five-dimension chips next to it.
- Unit 1.5 — `lib/ratings/normalize.ts` exposes `composite(points)` and `dimensionsToRadar(...)`.
- Velite `problems`, `ratings`, `taxonomy`.
- §10.4 — perf budget; the filter UI is the first interactive surface outside the dark-mode toggle. Keep the client component small.

## Constraints

- **Server-render the data; client component owns the filtering + sorting.** Server fetches every problem + its latest rating + composite + tags; client mounts them and manages state.
- **No external state library.** `useState` + `useMemo` for filter/sort. URL-encoded state is a Phase 2 polish (`?domain=…&sort=…`).
- **Static page.** Server data is determined at build time from Velite. The page itself is `○` static; only the client island re-renders on filter change.
- **Reading width.** `/problems` is dashboard-y, not prose. Use a wider container than `max-w-prose`.
- **Sort by composite shows underlying dimensions.** Each row carries the five chart-hue chips with the dimension's normalized value as a width.

## Alternatives considered

1. **One server component loads, one client component owns UI** (chosen).
2. URL-state filters via Next searchParams. Better deep-linking; Phase 2 work, not Phase 1.5.
3. Server-driven filtering via `?domain=…` reroute. Adds round-trips for trivial state.

## Edge cases & failure modes

1. **Problem has no rating yet.** Composite shown as `—`, dimension chips omitted.
2. **Tag list is huge.** Phase 1 has ≤ 30 tags across 10 seed problems. Render as a flat chip strip; defer collapse-to-popover to later.
3. **All filters off.** Show every problem; the empty-filter state is the default.
