### Prose styling

Markdown-rendered HTML elements (`<ul>` / `<ol>` / `<blockquote>` /
`<pre>` / `<code>` / `<h3>`-`<h6>` / `<hr>` / `<a>`) need styling
because Tailwind v4's preflight resets default browser styles.
Phase 17 ships **minimal inline arbitrary-variant utilities** in
the container `className`:

- `<a>` — accent color + underline-on-hover (matches existing
  link styling on page).
- `<code>` — muted background + rounded + monospace + xs size.
- `<pre>` — muted background + rounded + padding + overflow-x;
  nested `<code>` strips its own bg + padding for readability.
- `<ul>` / `<ol>` — list-disc / list-decimal + left padding.
- `<blockquote>` — left border + padding + italic.
- `<hr>` — border + vertical margin.
- `<h3>` / `<h4>` / `<h5>` / `<h6>` — serif font + size descent +
  font weight descent (matches existing heading hierarchy).
- `<p+p>` — paragraph spacing.

**NOT using `@tailwindcss/typography`** Phase 17 — adds ~30 kB
to the bundle + needs config tuning. The arbitrary-variant
utility list is self-contained + tight (~14 lines in the
className). Phase-18+ may swap to `@tailwindcss/typography` if
signal demands richer prose styling (e.g., curator review notes
or rating-action `rationale` rendering inheriting markdown).
