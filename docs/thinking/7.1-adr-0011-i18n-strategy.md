### Alternatives considered

The actual options-comparison work happens in the ADR itself. Here in the THINK doc, the meta-question: **should this ADR land BEFORE or AFTER Unit 7.2 (runtime install + lib/i18n)?**

- **Before** (chosen). Pins the choice + cost-governance posture before any code lands. Mirrors the ADR-0008 (LLM provider) precedent in Unit 5.1 — first paid surface in the project got its ADR before any script shipped.
- **After**. Would let the realized API shape inform the ADR. Mirrors ADR-0009 (human-review diff format) which landed at Unit 5.6 after Units 5.3 + 5.5 shipped the realized format.

The first pattern (ADR before code) is correct here because:
1. The runtime choice is binary (next-intl wins or doesn't); not a "shape emerged after implementation" case.
2. Q50 needs to close before Unit 7.2 imports `next-intl` to keep the audit trail tight.
3. ADR-0011 also pins the **non-code** decisions (content-storage shape, slug strategy, fallback chain) that span units beyond the runtime install — those need ADR-grade pinning before any unit can rely on them.

### Tradeoffs / defensible defaults flagged

(These are headlines; the ADR itself has the full options-comparison.)

1. **`next-intl` is a third-party runtime dependency**. Adds ~30 KB to the bundle. Acceptable per §5.5; the App Router-canonical i18n surface justifies the cost. Lighter alternatives (Paraglide.js) exist but require more glue.
2. **Sub-path routing (`/en/...`, `/fr/...`) over cookie-based locale**. Sub-path is crawler-friendly, bookmarkable, and works with SSG. Cookie-based serves the same content from one URL — incompatible with the project's SSG-first posture.
3. **Sibling files over sub-tree mirror over `lang:` frontmatter**. Each option has its own audit-trail / loader-complexity tradeoff. Sibling files preserve directory layout, which matches curator workflow.
4. **Slugs stay English-canonical**. Means `/fr/problems/hallucination-reduction` (not `/fr/problemes/reduction-hallucinations`). Trade-off: French speakers click an English-slug URL. Mitigated: titles + descriptions + body translate; URL slugs are technical identifiers (akin to ROR / ORCID IDs — non-translated).
5. **Locale fallback = `fr` falls back to `en`**. Untranslated FR pages render the EN content with a locale-switch hint, not garbled mix. Acceptable per the Q51 lean (infrastructure ships in Phase 7; backfill is curator-track).
6. **`translation_source` frontmatter field** (Q52 lean). Records human vs machine-assisted provenance per file. Defers full schema to Unit 7.5 but the ADR-level decision is "yes, provenance gets recorded" (not "no provenance tracking").
