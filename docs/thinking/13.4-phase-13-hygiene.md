## Class B — 8 Phase-13-specific follow-ons

Architectural follow-ons surfaced by Phase 13's surface. Each is
overridable as a Phase-14+ thread. Smaller catalog than Phase 12's 14
items because Phase 13 ships a read-only surface; fewer new
architectural questions.

1. **Public profile route at `/[locale]/u/[handle]`** — Q58 lean #3 +
   Phase-10 Class B item 1 + Phase-12 Class B item 12. The largest
   remaining Q58-adjacent surface. Phase 14+; needs per-user privacy
   model ADR (which fields are public — `name` + `image` + `githubLogin`
   + counts? user-editable signed-in?). **Potential ADR-0015 candidate**
   (per-user privacy model) IF Phase 13's per-status visibility policy
   doesn't fully cover the per-user case.
2. **Per-problem listing URL search-param sort** — allow re-sort by
   status / dimension / createdAt / submitter via URL `?sort=...&order=...`.
   Phase 14+ when volume justifies. Default `createdAt DESC` (newest
   first per Unit 13.0 D-5) sufficient for Phase-13 volume.
3. **Per-challenge public detail page** at
   `/[locale]/problems/[slug]/challenges/[id]` — submitter login + full
   rationale (not truncated) + status + reviewer notes (if accepted)
   + acceptedActionId link to rating-action page. Phase 14+; from
   Phase-11 Class B item 6 + Phase-12 Class B item 8.
4. **Counter section RSS** — emit per-problem digest RSS that includes
   "X pending / Y accepted challenges" line. Couples to Phase-5 digest
   thread. Phase 14+ if subscriber-list thread lands.
5. **Submitter anonymity option** — schema flag allowing submitter to
   opt out of public attribution. Per Unit 13.0 D-9 risk-surface +
   Unit 13.0 anticipated **Q61 candidate**. Phase 14+; needs schema
   change + ADR. Privacy note added in Unit 13.3 mitigates UX surprise.
6. **Rejection-rationale public visibility** — opposite of current
   Unit 13.0 D-3 lean. Editorial value case: showing rejections
   publicly creates "no-shadow editorial decisions" surface. Per Unit
   13.0 anticipated **Q62 candidate**. Phase 14+; policy decision.
7. **Per-problem listing pagination** — currently LIMIT 50 (matches
   `CHALLENGES_LIMIT`). When a single problem accumulates > 50
   challenges, oldest fall off the listing. Phase 14+ pagination via
   URL search-param `?offset=N`.
8. **Linked `acceptedActionId` reference** — currently rendered as
   mono-text filename (Unit 13.3). When Phase 14+ rating-action history
   route (`/problems/[slug]/ratings/[filename]` or similar) lands,
   upgrade to clickable Link. Couples to Phase-3 rating-action history
   surface.
