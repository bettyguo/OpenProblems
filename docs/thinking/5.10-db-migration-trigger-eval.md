## Re-evaluation triggers carried forward

Unchanged from Unit 4.12 + Unit 5.0 with one refinement:

1. **Content scale 3× / > 600 validated files / > 1 MB gzipped (internal alarm)** — gives ~5× headroom before the 5 MB hard trigger.
2. **First Phase-6+ write-path lands** — auth trigger flips. Candidates: email-subscription opt-in flow (Q44-adjacent), user-side "watch this problem" toggle, GitHub-OAuth-gated submission UI (§5.8).
3. **Phase 6 kickoff regardless** — mandatory per §12. Unit 6.0 (future) re-runs this measurement.
4. **Rating-action volume reaches 200** — currently 20.
5. **NEW** (post-Unit-5.5 observation): **drafts directory consistently > 100 stale files in any curator's working tree**. Operational signal, not a migration trigger — points at a `pnpm clean-drafts` script need (low-effort hygiene unit candidate for Phase 6).

## Forward-looking signals to watch (carried from Unit 4.12)

- **Storage-shape stress, not just size**: cross-cutting queries that don't map cleanly to per-page JSON slices.
- **Real-time data freshness**: if Phase 6+ adds a daily ingest cadence, every ingest is a re-deploy. A DB lets the cadence decouple from deploy cycles.
- **Multi-curator coordination**: ADR-0005 immutability + git PR review handles current concurrency. If 5+ active curators per week becomes the norm, optimistic-concurrency primitives from a DB become attractive.

None fires today.

## Phase-5-specific observation

The LLM-drafting pipeline (Units 5.3 + 5.5) writes to `drafts/` only — **NEVER** to `content/`. This means Phase 5's data-production model is **curator-mediated**, not **direct-LLM-write**. The size trigger is therefore latent: even with 100 ingest runs, no `content/` growth happens until curators `git apply` the diffs. The size trigger fires on the **applied content**, not on the **drafted content**. Drafts are gitignored; they don't affect deploy snapshots.

This is by design (§13 no-auto-merge + ADR-0009). A curator-bottleneck on the growth rate is the intended safety property.

## Consequences

- **Positive.** Same as Units 4.12 + 5.0: file-first stack remains performant + auditable + git-versioned.
- **Positive.** Phase 5's intelligence-layer deliverables shipped without any content-side growth — the trigger metric is stable across the boundary.
- **Negative.** Same Phase 5 → Phase 6 punt risk: if a Phase-6 write-path lands, the migration plan must come together quickly. Mitigated by the ADR-0004 anchor + the cumulative re-eval notes (4.12 → 5.0 → 5.10) establishing the durable framing.
