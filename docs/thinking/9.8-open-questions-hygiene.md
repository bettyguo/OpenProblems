#### ADR-0012 unit-level exercise

- **D-A. Runtime = NextAuth.js v5** → Unit 9.4 installed `next-auth@5.0.0-beta.31`. Prose says "`^5.x`"; actual pin is `5.0.0-beta.31` (exact, no caret) per beta-tag convention. **Prose-shift reconciliation**: edit D-A to acknowledge "exact-pin during the beta tag; switches to `^5.x` after v5 GA" — keeps the architectural intent intact while documenting the pin reality.
- **D-B. Identity provider = GitHub OAuth (only)** → Unit 9.4's `providers: [GitHub]` matches. Multi-provider expansion stays forbidden in Phase 9 (Phase-10+ Q-promotion required).
- **D-C. DB-backed sessions via Drizzle adapter** → Unit 9.4's `session: { strategy: "database" }` + `adapter: DrizzleAdapter(db, { usersTable, accountsTable, sessionsTable, verificationTokensTable })` matches.
- **D-D. Redirect-to-provider UX (no modal)** → Unit 9.5's `AuthControl` uses server-action `<form action={async () => { "use server"; await signIn("github", { redirectTo: "/" }); }}>` matches. Unit 9.6's `WatchlistToggle` signed-out branch links to `/api/auth/signin/github` — same architectural shape.
- **D-E. `users.githubLogin` text-unique column joining to file-system `editorial.primary_curator`** → Unit 9.3 landed the column + unique index; Unit 9.6 added `events.linkAccount` callback populating it from GitHub OAuth profile's `login` field on first sign-in. Architectural intent realized.

#### ADR-0013 unit-level exercise

- **D-A. DB engine = libSQL (Turso)** → Unit 9.3 installed `@libsql/client@0.17.3`. Connection string format honored (`libsql://` production / `file:` local-dev). No other DB driver landed.
- **D-B. ORM = Drizzle (migration immutability)** → Unit 9.3 installed `drizzle-orm@0.45.2` + `drizzle-kit@0.31.10`. Schema in `lib/db/schema.ts`; migrations in `lib/db/migrations/`. Two migrations applied: `0000_initial_auth` (Unit 9.3) + `0001_watchlist` (Unit 9.6). Both committed, neither edited post-generation.
- **D-C. Local-dev DB = `file:./local.db`** → Unit 9.3's env-fallback in `lib/db/index.ts` matches. `local.db` + sidecar `.gitignore`'d in Unit 9.3.
- **D-D. Production DB = single Turso database (single-tenant)** → Q55 operational; architectural intent intact.
- **D-E. Migration cadence: `0001_initial_auth` + `0002_watchlist`** → **Prose-shift**: ADR-0013 D-E prose used 1-indexed migration names. Actual landed names are 0-indexed (`0000_initial_auth` + `0001_watchlist`) per drizzle-kit's monotonic convention. Off-by-one noted in Unit 9.3 + 9.6 CHANGELOG entries; **this unit corrects D-E inline** to match drizzle-kit's 0-indexed reality.
- **D-F. No write-paths against content tables** → Unit 9.6's `watchlist` table is USER-STATE (`(userId, problemSlug)` per Q56); `problemSlug` is plain text with no FK to content. Content stays file-first per ADR-0004.
