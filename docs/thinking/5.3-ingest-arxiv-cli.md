### Scope (this unit)

Three new files:

1. **`lib/curate/anthropic.ts`** ÔÇö thin wrapper over `@anthropic-ai/sdk`. Per ADR-0008 D-D: handles `cache_control` for shared prompt blocks. Per D-C: reads `ANTHROPIC_API_KEY`, supports `--dry-run` semantics by accepting a `dryRun: boolean` option, reports estimated cost, appends to `.llm-spend.log` when budget env is set. 2. **`lib/curate/paper-draft.ts`** ÔÇö pure functions: `buildIngestPrompt(metadata, slugs)`, `parseLLMResponse(text)`, `renderPaperYaml(draft)`, `buildUnifiedDiff(draftPath, draftBody)`. Testable in isolation.
3. **`scripts/ingest-arxiv.ts`** ÔÇö CLI entry. Args via `process.argv` parsing (lightweight; no `commander` / `yargs` dep). Orchestrates: fetch metadata Ôćĺ build prompt Ôćĺ call LLM (or dry-run) Ôćĺ render YAML Ôćĺ build diff Ôćĺ write `drafts/<unit>-<ts>-<slug>.diff` + `.meta.json` sidecar.

Plus:

- **`package.json`** ÔÇö add `"ingest-arxiv": "tsx scripts/ingest-arxiv.ts"` to scripts.
- **New runtime dep**: `@anthropic-ai/sdk` (per ADR-0008 D-A).
- **Tests** for `paper-draft.ts` (pure functions); `anthropic.ts` wrapper tests use dry-run mode + injected fake client.

### Constraints

- **`--dry-run` produces a draft diff WITHOUT calling the API**. Stub the LLM response with a "DRY RUN" comment block. Lets curators preview the prompt + see the diff shape without burning tokens.
- **`ANTHROPIC_API_KEY` required for non-dry-run**. Loud failure when unset.
- **Diff format = unified diff vs. an empty file** at `content/papers/<arxiv-id>.yaml`. If the file already exists (paper already ingested), the script aborts loudly with a "paper already ingested; edit by hand" message ÔÇö does NOT overwrite existing content.
- **No auto-apply**. Script never edits `content/papers/`. Curator runs `git apply drafts/<file>.diff` manually.
- **Slug-existence advisory**: the prompt instructs the LLM to use existing author/institution slugs from a list passed in. When the LLM proposes a new slug, the YAML draft emits a `# NEW SLUG ÔÇö curator must create content/authors/<slug>.yaml` comment line. Curator decides.
- **No `content/` writes** at runtime ÔÇö the script is in `scripts/`, not `app/`; safe to keep Velite reads + filesystem reads of `content/` but never writes.

### CLI surface

```
pnpm ingest-arxiv <arxiv-id> [options]

Options:
  --model <id>             Override the Anthropic model (default: claude-sonnet-4-6)
  --dry-run                Don't call the API; emit the prompt + placeholder diff
  --verbose                Print prompt + estimated cost before each API call
  --no-cache               Bypass the .arxiv-cache/ on metadata fetch
  --out <dir>              Override the drafts directory (default: drafts/)
  --help                   Show this help
```

Examples:
```
pnpm ingest-arxiv 2310.06770
pnpm ingest-arxiv 2310.06770 --dry-run --verbose
pnpm ingest-arxiv 2310.06770 --model claude-opus-4-7
```

### Prompt design

System block (cached via `cache_control: { type: "ephemeral" }`):
- Excerpt of `MASTER_PROMPT.md` ┬ž7 (paper-object framing) + ┬ž15.6 (primary-source rule).
- `PaperSchema` description in TypeScript-like prose.
- List of available `problem_slug`s.
- List of available `author` slugs (just identifiers; curator-side resolution).
- List of available `institution` slugs.

User block (per-paper):
- arXiv metadata (id, title, authors[], abstract, primaryCategory, dates, URLs).

Expected response: YAML body **only** (no fenced code block, no commentary). The wrapper parses it; if it's wrapped in ``` blocks, strip them.

Cost estimate (Sonnet 4.6 published prices):
- System block ~3000 tokens (cached after first call).
- User block ~500 tokens.
- Response ~400 tokens.
- Per-call: ~$0.007 (cold) / ~$0.003 (cache hit).

### Alternatives considered

- **Apply the YAML directly into `content/papers/` (no diff)**. Rejected per ┬ž13 ("must produce a human-review diff; no auto-merge").
- **Use Anthropic's structured-output / tool-use mode** instead of free-form YAML response. Considered. Rejected for Unit 5.3 ÔÇö adds complexity (defining a tool schema, parsing tool_use blocks); free-form YAML parsing is sufficient at this scale. Revisit if YAML parse errors are common; ADR-0008's `lib/curate/anthropic.ts` wrapper is the right place to gate the upgrade.
- **CLI lib (commander / yargs / sade)**. Rejected ÔÇö lightweight `process.argv` parsing is < 30 lines for this surface. Adding a dep for it is over-engineering at this scale.
- **Combine 5.3 + 5.4 + 5.5 into a single mega-CLI** with subcommands. Rejected ÔÇö separation makes each script's prompt + audit shape simpler. Phase-5 acceptance gate (5.13) is easier to smoke-test per-script.
- **Test the CLI end-to-end via subprocess**. Considered. Rejected ÔÇö CLI subprocess tests in vitest are brittle on Windows; the testable surface is the pure functions in `paper-draft.ts`. CLI smoke is part of Unit 5.13.

### Tradeoffs / defensible defaults flagged

1. **Free-form YAML response.** No structured-output guard. If the LLM emits a `tldr:` over 400 chars, the curator catches it at `pnpm validate-content` time (the existing Zod schema gates the eventual PR). Trade-off: silent generation of invalid YAML in the diff. Acceptable because the diff is human-reviewed.
2. **Audit sidecar is on disk, not committed.** Per `.gitignore` (Unit 5.2). Future Phase-6 might want a curator-visible audit log; revisit then.
3. **No retry on Anthropic API failures.** First call fails Ôćĺ loud error Ôćĺ curator decides. Don't burn retries on the curator's behalf.
4. **No automatic slug resolution for authors/institutions.** The LLM proposes slugs in kebab-case; curator validates against `content/authors/*.yaml` and `content/institutions/*.yaml`. Proposed-but-non-existent slugs get a `# NEW SLUG` comment.
5. **System block as a single cached chunk.** Even if ┬ž7 + ┬ž15.6 + schema + slug-lists drift in different units, they cache together. Trade-off: a single MASTER_PROMPT.md edit invalidates the cache for all in-flight runs. Acceptable.
