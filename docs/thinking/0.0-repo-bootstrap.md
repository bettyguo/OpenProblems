# Unit 0.0 — Repo Bootstrap

> Stage 1 (THINK) artifact for Unit 0.0. Per MASTER_PROMPT §15.1.

## Problem restatement

Establish the minimum repo hygiene that every subsequent unit depends on: a project-facing README, a changelog convention, a place for Claude Code to surface ambiguity to the human (`OPEN_QUESTIONS.md`), correct ignore/editor configs, and the master prompt at the canonical filename it self-references (`MASTER_PROMPT.md`). No application code, no toolchain — that begins at Unit 0.2.

## Inputs

- `OpenProblems_MASTER_PROMPT.md` — the constitution (49 KB).
- §6 (Repository Structure) — required top-level files: `README.md`, `MASTER_PROMPT.md`, `OPEN_QUESTIONS.md`, `CHANGELOG.md`.
- §14.4 — PR template requires CHANGELOG updates; Conventional Commits required.
- §17 — nine open questions to seed.
- Appendix A — the doc references itself as `MASTER_PROMPT.md`; the repo currently has it under `OpenProblems_MASTER_PROMPT.md`.

## Constraints

- Critical-path units (schemas, IA, design tokens) need sign-off; **Unit 0.0 is not critical-path** (§15.4) — proceed through all four stages and post a summary before Unit 0.1.
- Do not pick a license — that is open question 17.4.
- Do not introduce toolchain files (`package.json`, `tsconfig.json`, etc.) — those belong to Unit 0.2/0.3.
- No node_modules, no build artifacts, no editor IDE folders should ever be committed; `.gitignore` must be comprehensive even before tooling lands.

## Alternatives considered

1. **Comprehensive bootstrap (chosen).** Land README, CHANGELOG, OPEN_QUESTIONS, .gitignore, .editorconfig, and the master-prompt rename in one unit.
   - Pro: every later unit can assume these files exist.
   - Con: slightly larger first commit; mitigated by zero code.
2. **Minimal bootstrap (rejected).** Land only README + rename now; defer CHANGELOG/.gitignore/.editorconfig to where they're first needed.
   - Pro: even smaller first commit.
   - Con: forces every later unit to also touch hygiene files, and risks committing build artifacts the moment Unit 0.2 introduces tooling.
3. **Defer the rename of `OpenProblems_MASTER_PROMPT.md` (rejected).** Keep the current filename.
   - Pro: zero churn.
   - Con: the doc self-references `MASTER_PROMPT.md` ~5×; Appendix A's "read `MASTER_PROMPT.md` end-to-end" instruction is wrong against the actual filename; future sessions and any tooling that greps for the filename will trip. Better to fix now while there's a single commit of history.

## Edge cases & failure modes

1. **Rename breaks an external reference.** Mitigated: there are no external references yet — this is the first commit. The rename is safe now and unsafe later.
2. **Premature LICENSE choice locks the project in.** Mitigated: do not commit a `LICENSE` file. Flag in OPEN_QUESTIONS as a *blocker* before any public release.
3. **`.gitignore` too narrow.** Once Unit 0.2 introduces Next.js / pnpm, generated artifacts (`node_modules/`, `.next/`, `.turbo/`, `coverage/`, `playwright-report/`) could leak into the next commit. Use a wide-net ignore now, tightened later if needed.
4. **`.gitignore` too wide.** Could mask content authors intend to commit (e.g., `.env.example`, design assets in `public/`). Explicitly allow-list those that are known to be tracked.
5. **CHANGELOG drift.** §14.4 says every PR updates CHANGELOG; without a stub, the first PR will skip it. Seed `[Unreleased]` and an entry for Unit 0.0 itself.
6. **Master prompt drift after rename.** If anyone later forgets the rename and recreates `OpenProblems_MASTER_PROMPT.md`, we get duplicate sources of truth. Mitigation belongs to a CI lint at Unit 0.11; for now, note it in OPEN_QUESTIONS so the human is aware.

## Open questions surfaced (→ OPEN_QUESTIONS.md)

In addition to the nine from MASTER §17, this unit surfaces:

- **Q10. "Eight phases" wording vs. seven rows.** §12 lists 7 rows (0–6+), Appendix A calls them "eight." Resolve by splitting Phase 6+ into 6 (Discussions / talk pages) and 7 (API auth + i18n + monetization), or by editing Appendix A.
- **Q11. Subdomain id collisions across domains.** `representation-learning` appears in both `deep-learning` and `general-ml`; `robustness` in both `deep-learning` and `social-aspects`; `theory` is both a subdomain and a top-level domain id. This affects URL design (`/domains/[domain]/[subdomain]` is safe — each id is scoped under a parent — but if we ever flatten to `/subdomains/[id]` it breaks). Confirm: keep ids domain-scoped (chosen default) or globally unique (would force renames).
- **Q12. Package manager.** §15.3 says `pnpm typecheck`. Confirm pnpm vs npm/yarn/bun before Unit 0.2.
- **Q13. Master prompt filename.** Renaming `OpenProblems_MASTER_PROMPT.md` → `MASTER_PROMPT.md` to match the doc's self-reference. Confirm.

## Decision

Land seven artifacts in one commit-shaped unit (no commit will be made until the human says so):

1. `MASTER_PROMPT.md` — renamed from `OpenProblems_MASTER_PROMPT.md` via `git mv`, content unchanged.
2. `README.md` — terse top-of-funnel: one-paragraph pitch, pointer to `MASTER_PROMPT.md`, current phase = 0, license = "pending (see OPEN_QUESTIONS Q4)".
3. `CHANGELOG.md` — Keep-a-Changelog format with `[Unreleased]` + an entry for Unit 0.0.
4. `OPEN_QUESTIONS.md` — Q1–Q9 verbatim from MASTER §17, plus Q10–Q13 from this THINK doc. Each item has a status field (`open` / `decided`).
5. `.gitignore` — Node + Next.js + Playwright + Storybook + Turbo + Vercel + common editor folders, wide-net.
6. `.editorconfig` — 2-space, UTF-8, LF, trim trailing whitespace, final newline. Matches Prettier defaults to come in Unit 0.8.
7. `docs/thinking/0.0-repo-bootstrap.md` — this file.

No LICENSE file is added (blocked on Q4). No package.json or toolchain files are added (Unit 0.2+).

The four-stage loop's DESIGN stage is collapsed into this THINK doc for this unit — there is no UI, no data schema, and no algorithm; the "design" is the file list above, which is fully specified here.
