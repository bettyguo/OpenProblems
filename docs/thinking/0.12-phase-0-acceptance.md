# Unit 0.12 — Phase 0 acceptance gate

> Stage 1 (THINK) artifact for the Phase 0 closeout.

## Problem restatement

Phase 0's acceptance gate (MASTER_PROMPT §13) requires:

1. `pnpm build` is green.
2. Preview deployment to Vercel renders the stub landing.
3. Lighthouse perf ≥ 95 on the empty landing.
4. All schemas have unit tests.

This unit confirms (1) and (4) locally, documents the current state vs the gate, updates the README to reflect Phase 0 closeout, and flags the steps (2 and 3) that require external systems (Vercel + LHCI run on a real Linux box) the user has to trigger.

It also walks the OPEN_QUESTIONS list and tags which entries block Phase 1 vs which can stay open.

## Inputs

- §13 Phase 0 acceptance criteria.
- The full commit history `0eb5b70..a3db1f2` (Units 0.0 → 0.11).
- Current `OPEN_QUESTIONS.md` (Q1–Q29 or so).

- **No new code in this unit beyond a README touch-up.** The acceptance gate is a milestone, not a feature.

- **Phase-1-ready hand-off.** After this commit, the next session should be able to read `MASTER_PROMPT.md` + `CHANGELOG.md` + `OPEN_QUESTIONS.md` and start Phase 1 unit 1.0 without confusion.
