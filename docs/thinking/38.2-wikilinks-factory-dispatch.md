### Edge cases & failure modes

1. **Unknown env-var value**: throws on first call with clear
   message listing recognized values. Test verifies the error
   contains both the invalid value AND the recognized list.
2. **Singleton-cached-across-env-changes**: once `getExtensionRegistry()`
   has been called and cached the registry, changing the env-var
   does NOT switch registries. Tests use `__resetRegistryForTests()`
   in `beforeEach` to force re-dispatch.
3. **Markdown processor caches**: end-to-end tests must call
   `__resetMarkdownCachesForTests()` after swapping the registry
   so the per-surface lazy processor caches rebuild against
   the swapped registry. Phase-37 framework's test-hook surface
   handles this.
4. **Test-suite ordering**: env-var state must be restored to
   `ORIGINAL_PROVIDER` in `afterEach` to prevent leaking into
   sibling test suites. Pattern mirrors `lib/moderation/index.test.ts`.
5. **Default arm receives empty string**: process.env lookups
   return `undefined` for unset; the `??  ""` coalesce treats
   both unset and empty as the default arm. Test verifies both.

## DESIGN

### Factory dispatch shape

```ts
let registryInstance: MarkdownExtensionRegistry | null = null;
let registryOverride: MarkdownExtensionRegistry | null = null;

export function getExtensionRegistry(): MarkdownExtensionRegistry {
  if (registryOverride) return registryOverride;
  if (registryInstance) return registryInstance;

  const provider = process.env["MARKDOWN_EXTENSIONS"] ?? "";

  switch (provider) {
    case "":
    case "default":
      registryInstance = new DefaultExtensionRegistry();
      return registryInstance;
    case "wikilinks":
      registryInstance = new WikilinkExtensionRegistry(
        PHASE_38_DEFAULT_ENABLED_SURFACES,
      );
      return registryInstance;
    default:
      throw new Error(
        `Unknown MARKDOWN_EXTENSIONS value: "${provider}". ...`,
      );
  }
}
```
