## THINK

### What this unit ships

Three new files in `lib/markdown/`:

1. **`lib/markdown/index.ts`** — exports
   `renderBioMarkdown(text: string | null): string | null`.
   Singleton `unified` processor instance with pipeline order per
   ADR-0018 D-A:
   - `remarkParse` → MDAST.
   - `remarkGfm` → GFM extensions parsed (tables / footnotes
     parsed but stripped at sanitize stage; strikethrough +
     autolinks + task-lists allowed per D-B).
   - `remarkRehype({ allowDangerousHtml: false })` → HAST; first
     line of defense (raw HTML in source stripped here).
   - **`rehypeDemoteHeadings`** (custom plugin) → outline
     preservation per D-C.
   - `rehypeSanitize` with `bioSchema` → tag + attribute + URL-
     protocol allow-list.
   - **`rehypeStripUnsafeHrefs`** (custom plugin) → defense-in-
     depth against schemeless relative URLs per D-D ("relative
     URLs DENIED" clause that rehype-sanitize's `protocols`
     filter doesn't catch — `protocols` only inspects URLs WITH
     a scheme; schemeless `/path` passes through it).
   - `rehypeStringify` → HTML string.

2. **`lib/markdown/sanitize-schema.ts`** — exports the
   `bioSchema: Schema` rehype-sanitize config narrowed per D-B +
   D-D. **Not named `defaultSchema`** to signal Phase-17 scope per
   ADR-0018 D-G; Phase-18+ markdown surfaces will declare sibling
   schemas (`reviewNotesSchema`, etc.) per surface-specific
   customization.

3. **`lib/markdown/index.test.ts`** — **33 new tests** organized
   into four sections:
   - **Happy path (D-B allowed subset)** — 14 tests covering
     bold / italic / inline code / fenced code blocks / https
     links / mailto links / unordered lists / ordered lists /
     blockquotes / horizontal rules / GFM strikethrough / plain
     text passthrough (Phase-15 backward compat) / GFM task lists
     read-only / GFM bare-URL autolinks.
   - **XSS-vector defense (D-D URL allow-list + D-B denials)** —
     12 tests covering `javascript:` / `data:` / `file:` /
     `http:` (require TLS) / `vbscript:` URL strips; raw HTML
     `<script>` / `<iframe>` / `<style>` strips; `onerror` event
     handler strip; markdown link `title` attribute strip
     (XSS-vector-by-name); schemeless relative URL strip; image
     embedding strip (Phase-18+ deferral per D-B).
   - **Heading demotion (D-C)** — 4 tests covering `#` → `<h3>`;
     `##` → `<h4>`; `###` → `<h5>`; `####` / `#####` / `######`
     all clamp to `<h6>` (single test, three assertions).
   - **Null + empty edge cases** — 3 tests covering null input
     (Phase-15 D-F empty-state preserved); empty-string input
     (treated as null); whitespace-only input (trim-collapses
     to null).

### Why two custom rehype plugins

Two small inline plugins in `lib/markdown/index.ts`:

1. **`rehypeDemoteHeadings`** (ADR-0018 D-C) — page outline
   preservation. Walks the HAST; demotes `<h1>`/`<h2>`/`<h3>` by
   2 levels; clamps `<h4>`/`<h5>`/`<h6>` to `<h6>`. Implements D-C
   exactly.
2. **`rehypeStripUnsafeHrefs`** (ADR-0018 D-D enforcement
   defense-in-depth) — rehype-sanitize's `protocols.href` filter
   only inspects URLs WITH a scheme; schemeless relative URLs
   (`/path`, `path`, `#anchor`) pass through. This plugin strips
   `<a href>` when it doesn't match the `^(https://|mailto:)`
   pattern. Runs AFTER rehype-sanitize so it operates on the
   already-sanitized tree shape.

Both are ~15 lines each; kept inline rather than separate files
because they're tightly coupled to the pipeline + don't need
independent reuse Phase 17. Phase-18+ markdown surfaces
inheriting via sibling helpers may extract them if reused.
