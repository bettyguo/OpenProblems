### Phase-5-blocking decisions resolved here

#### D-1. LLM provider = Anthropic Claude

**Decision**: Anthropic SDK (`@anthropic-ai/sdk`). Latest stable Claude model at the time the script runs (Sonnet 4.6 by default; Opus 4.7 for harder extraction tasks via `--model` flag).

Rationale:
- Project name + brand alignment ("LLM OpenProblems"; the encyclopedia *of* LLM open problems).
- Anthropic SDK has well-typed TS bindings + cost-cap headers + structured-output (`tool_use`) patterns suited to YAML drafting.
- Prompt caching halves the cost of multi-call ingest runs (relevant when the same MASTER_PROMPT.md context block is reused across paper drafts).
- ADR-0008 (Unit 5.1) records the realized choice + the dependency contract.

**Cost-governance pact** (also goes in ADR-0008):
- Every CLI accepts `--dry-run` that prints prompts without making API calls.
- Every CLI requires `ANTHROPIC_API_KEY` from env; no fallback / no default key.
- Every CLI prints estimated token cost before each call when `--verbose`.
- Daily-cap env var `LLM_OPENPROBLEMS_DAILY_BUDGET_USD` (default: unset → no cap, with WARNING).

#### D-2. PDF text extraction = `pdf-parse` (or `pdfjs-dist` if too lossy)

**Decision (lean)**: start with `pdf-parse` (lightweight, pure JS, ~50 KB gz, no native deps). If extraction quality on academic PDFs proves too lossy (citations getting munged into prose; equations dropped silently), upgrade to `pdfjs-dist` (Mozilla's reference parser, heavier but more faithful).

Empirical evaluation in Unit 5.4: run `pdf-parse` against 3 sample papers (one ML, one biology, one foundational-NLP) and compare against the arXiv abstract for noise floor. If the abstract text round-trips cleanly, ship `pdf-parse`. Otherwise escalate.

#### D-3. arXiv API rate limit = 3 req/s with filesystem cache

**Decision**: arXiv's published policy ([info.arxiv.org/help/api/index.html](https://info.arxiv.org/help/api/index.html)) is ≤ 3 req/s for the OAI-PMH + Atom endpoints. `lib/curate/arxiv-client.ts` exposes:
- `fetchMetadata(arxivId)` — returns title, abstract, authors, primary category, submitted-date, version, PDF URL.
- Internal rate limiter (token bucket; 3 req/s burst, 1 req/s sustained).
- Filesystem cache at `.arxiv-cache/<id>.json` (with `--no-cache` CLI flag to bypass).

`.arxiv-cache/` lives in `.gitignore` (added in Unit 5.2; not pre-emptively).

#### D-4. Email = NOT in Phase 5; RSS-only

**Decision**: §13 names "Email\RSS digest" but Phase 5 ships **RSS-only**. Email needs deploy infra (SES / Resend / Postmark account) + a subscriber-list store + double-opt-in flow — all auth-trigger-flipping per Unit 4.12. Defer to Phase 6+ alongside the auth migration.

RSS-only Phase 5: `/api/v1/digest/<domain>.xml` per-domain feeds; subscribers add to their RSS reader.

#### D-5. CLI invocation = `pnpm script <name>`

**Decision**: every Phase-5 CLI lives in `scripts/` and runs via `tsx`. New `package.json` scripts:

```json
{
  "ingest-arxiv": "tsx scripts/ingest-arxiv.ts",
  "extract-leaderboard": "tsx scripts/extract-leaderboard.ts",
  "build-digest": "tsx scripts/build-digest.ts"
}
```

Pattern mirrors `pnpm validate-content` / `pnpm audit-content` (Phase 0/2 tooling).

#### D-6. Human-review diff format = unified patch in `drafts/`

**Decision**: every LLM-assisted script writes its draft to `drafts/<unit>-<timestamp>-<slug>.diff` as a unified-format patch (compatible with `git apply`). The curator reviews the diff, edits if needed, and either:
1. `git apply drafts/...diff` to materialize the change in the working tree (then PRs as a normal commit), or
2. Edits the diff first, then applies.

Rationale:
- §13's "must produce a human-review diff; no auto-merge" explicitly forbids auto-merging.
- Unified diff is the lingua franca of git review. No new tooling needed.
- `drafts/` is in `.gitignore` (added in Unit 5.6's ADR-0009); diffs are ephemeral per-run.

ADR-0009 (Unit 5.6) records the realized diff format + the no-auto-merge contract.

#### D-7. Drafts directory = `drafts/`, gitignored

**Decision**: top-level `drafts/` (not under `content/` or `.velite/`). Gitignored. CLI flag `--out <path>` overrides the default location.

Why not `.drafts/` (hidden): unhidden makes the drafts discoverable from `ls`; curators may want to grep / search. Trade-off: unhidden + gitignored is the editor-friendly choice.

#### D-8. ADR-0004 (file-first) inherits + Phase 5 re-affirms

**Decision**: Phase 5's mandatory DB-migration trigger re-evaluation re-affirms ADR-0004. No new DB. **Re-eval at this commit**: `gzip(.velite snapshot) = 72,274 bytes` (~70.6 KB) — **~1.38% of the 5 MB §12 threshold** (was ~1.32% at Unit 4.12). Auth trigger also still negative — Phase-5 workflow is curator-side scripts + RSS subscription (no first-party auth). **DB migration remains deferred to Phase 6+**, per the §12 contemplated path.

Phase 5's first auth-trigger candidate is the RSS digest's subscriber list — but that's deferred to Phase 6+ per D-4.

### Phase-5-blocking decisions deferred to per-unit implementation

- **D-9. Prompt-caching shape for paper-draft prompts** (Unit 5.3). Anthropic's prompt caching with `cache_control: { type: "ephemeral" }` on the MASTER_PROMPT.md context block. Worth doing if the same script runs over 10+ papers in a session; skip for one-off ingests.
- **D-10. arXiv category filtering** (Unit 5.3). Default ingest scope: cs.AI / cs.CL / cs.LG / stat.ML — but a curator running `ingest-arxiv 2310.06770` on a paper outside those categories should still succeed. Implementation: filter only when no explicit ID is passed (i.e., `ingest-arxiv --recent` mode).
- **D-11. Leaderboard-extraction guardrails** (Unit 5.5). When the LLM extracts a score, the script verifies the score matches a `benchmarks[]` entry in the named problem's `problem.yaml`. If not, the diff includes a `# WARNING: benchmark_id "X" not declared on problem "Y"` comment.
- **D-12. Digest cadence** (Unit 5.7). Weekly summary's window = trailing-7-days OR most-recent-N-actions? Lean: trailing-7-days for cadence consistency; the digest's "items" are rating-actions + new-papers + new-entries in that window. Empty windows produce empty (but valid) RSS feeds with a `<description>` noting "no activity this week".

### Open questions newly surfaced

**Q41. LLM model choice per script** — Sonnet 4.6 default for paper ingest; Opus 4.7 for harder leaderboard-from-PDF extraction (multi-table parsing). Lean: **per-script default**; `--model` flag overrides. The `claude-api` skill in the project's command palette explicitly recommends "latest and most capable Claude models" — Phase 5's first runs land on Sonnet 4.6 + Opus 4.7.

**Q42. Cost-cap default** (no daily-cap env var = unbounded). Setting a default cap means CLIs fail on a fresh install; not setting one means curators discover cost surprises. Lean: no default cap, but `--verbose` shows estimated-cost-before-call so the curator sees the cost line before pressing Y. Re-evaluate after the first 100 ingest runs reveal the actual per-call distribution.

**Q43. PDF cache** — should `pdf-text.ts` cache extracted text to `.pdf-cache/<arxiv-id>.txt`? Lean: yes, mirrors `.arxiv-cache/`. Cuts dev-loop cost in half (PDF re-fetches are expensive).

**Q44. Digest signing** — the per-domain RSS feed should declare a `<managingEditor>` (Q33 resolution pending). Lean: defer until Q2 (DNS) lands so we have a real email.

### Acceptance gate plan (Unit 5.13)

Four checks + one soft:
1. **CLI smoke** for each Phase-5 script (`ingest-arxiv`, `extract-leaderboard`, `build-digest`) — each runs against a known fixture and produces a non-empty draft.
2. **RSS validates** per-domain feeds — W3C feed validator on each `/api/v1/digest/<domain>.xml`. Pass or block (same Q27-class pattern; CI on deploy).
3. **No-auto-merge contract verified**: every Phase-5 CLI run leaves the working tree clean (drafts go to `drafts/`; no direct edits of `content/`).
4. **Lighthouse a11y ≥ 95** on `/digest` (new hub page). Add to `lighthouserc.json` in the gate unit.
5. **Soft**: visual baselines for `/digest` × 2 themes × N viewports (same Q27-class follow-on).
